Ooyala to Video Cloud Migration: Backlot API

This topic provides guidance on moving from the Ooyala Backlot API to the Video Cloud platform APIs.

See other Ooyala to Brightcove Migration Guides.


Introduction

The Video Cloud architecture has an API generally for each module in Studio, and sometimes for specific functionality within a module. Therefore, the operations enabled by the Backlot API correspond to operations within different Brightcove API. This document will help you understand these mappings.

The sections below correspond to the different endpoint groups for the Backlot API:

Authentication

Authentication of Brightcove API requests is handled slightly differently than that for the Ooyala REST APIs. See Authentication for Brightcove API Requests for details.

Ad Sets

There are no real equivalents for the Ad Sets endpoints for client-side ad requests, because most of that ad setup for Video Cloud is done on players, through plugins, rather than attached to individual videos. The best general place to start for understanding how advertising is implemented for Video Cloud videos see the Learning Guide: Video Advertising.

For those using server-side advertising, the SSAI API does provide similar functionality for setting up ad configurations.

DRM Policies

There are no Video Cloud equivalents for the DRM Policy endpoints in the Backlot API. DRM is managed at the account level in Video Cloud by Support. The only relevant CMS API request is Update Video, which allows to set the drm_enabled flag to false, which removes DRM packaging from the specified video.

See the Overview: Digital Rights Management (DRM) in Video Cloud for more details on how DRM works.

Deleted Assets

There are no Video Cloud equivalents for the Deleted Assets endpoints in the Backlot API. Videos deleted within the past 10 days can be viewed by searchingfor ideos with state:DELETED.

Labels

There are no Video Cloud equivalents for the Labels endpoints in the Backlot API, because there is no exact equivalent of labels.. You can get a list of custom fields for an account using this CMS API endpoint, and set custom fields when you create or update a video. Currently custom fields must be defined in Studio.

Packages

There are no Video Cloud equivalents for the Packages endpoints in the Backlot API.

Playlists

The Backlot API Playlist endpoints correspond generally to the CMS API playlist endpoints. There is also a Playback API endpoint for retrieving a playlist for use in a player or video portal. Video Cloud playlists are somewhat more flexible in that you can define them with an array of video ids, or using a search string. See Managing Playlists Using the CMS API for more details.

Processing Profiles
