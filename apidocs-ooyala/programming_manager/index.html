Introduction

The Programming Manager REST API lets you query and update provider assets, asset offers, channels, and collections.

A­uthenticated Requests

User authentication and identity management is integrated with Gigya, a third-party vendor that handles identity management of our clients from all types of social networks such as Facebook, LinkedIn, Google Plus, etc. The client calls an authentication API that returns an account_token. The account_token must be appended as a query-string parameter to subsequent requests (e.g. for the eCommerce APIs). The account_token expires after 15 minutes (by default). When a request is sent with an expired account_token, the server responds with a 403 "Invalid account_token". The client is responsible for re-requesting the account_token on 403 responses.

Requests must be sent via HTTPS using the host player.ooyala.com (click the POST button below to see examples of usage).

Unauthenticated Requests
If you do not include your account token to authenticate a query, you must include the pcode=<pcode> parameter. See http://help-ooyala.brightcove.com/video-platform/concepts/api_keys.html for details on finding your pcode.

/authentication/v1/providers/{pcode}/gigya post

post

Retrieve your account_token by making the following request. The uid, timestamp, and signature are sent from Gigya once the consumer facing client authenticates with it. The server will return an account token along with an expiration.

Sample POST request:
https://player.ooyala.com/authentication/v1/providers/{pcode}/gigya?uid=johndoe&signatureTimestamp=1930294539&UIDSignature=p3RfmwGk%2FjwIup%’

Using your Account Token:
You may optionally include your account token to authenticate each query. Make sure to replace <account_token> with the token you retrieve by making the above call. For example:

curl 'http://player.ooyala.com/opm/v1/assets?s=<search_text>&account_token=<account_token>'

Error Responses:
If an error occurs, the server returns a 4xx response. The body will contain an error message, such as 400, “Invalid Signature”.

Assets

The asset library consists of videos, as well as custom content groups of video assets used to organize episodic content, movies, sports events, and channels.

Use the q query parameter with a query phrase to search all metadata based on the provided phrase.

/assets get

get

Return a list of all published assets in the asset library.

get /assets

Return a list of all published assets in the asset library.

  • Request
  • Response

Query Parameters

  • q: (string)

    Performs a full query search. See Elasticsearch query string syntax for details on how to format your query.

  • details: (string)

    Performs a simple full-text search against the currently available asset library. For partial match search or "Instant Search" functionality, use the q query parameter and specify a wildcard match.

    • summary: (Default) Returns high-level information about each asset, including id, name, description, and type. Ideal for auto-complete and quick searches.
    • images: Returns a list of images associated with an asset.
    • offers: Returns a list of offers that are directly associated with an asset (each offer is a direct parent of the asset). Note that if an offer is not enabled, it is not included in the list that is returned. Also, if an offer is configured with an availability window and the current date is not within that availability window, the offer is not included in the list that is returned.
    • collections: Returns a list of collections associated with the asset (parent relation only, no grandparents).
    • collections_with_offers: Returns a list of only those collections having offers to which the asset belongs.
    • channels: Returns a list of channels associated with an asset.
    • metadata: Returns an asset's complete metadata.
    • labels: Returns the labels associated with an asset.
    • parents: Returns labels, offers, implicit offers, collections, and channels with which the asset is associated.
    • children: Returns summary data for child assets (e.g. episodes in a season). Only immediate children are returned.
    • renditions: Returns the available renditions for an asset (HD, SD, etc.). Note that renditions only exist for the lowest content group, such as episodes.
    • all: Returns all of the above.

    Logical Operations. The following examples illustrate how to apply AND, OR, and NOT operations within your queries. Consider these two separate requests:
    curl -X GET 'https://player.ooyala.com/opm/v1/assets?q=metadata.display.director:"Josh"' curl -X GET 'https://player.ooyala.com/opm/v1/assets?q=metadata.display.actor:"Marc"'

    To combine the requests with a logical AND, retrieving assets with both the director Josh AND the actor Marc, use the + symbol:
    curl -X GET 'https://player.ooyala.com/opm/v1/assets?
    q=(metadata.display.director:"Josh" +metadata.display.actors:"marc")'

    To combine them with a logical OR, retrieving assets with the director Josh OR the actor Marc, use a space between the two values (an implicit OR):
    curl -X GET 'https://player.ooyala.com/opm/v1/assets?
    q=(metadata.display.director:"Josh" metadata.display.actors:"marc")'

    To combine them with a logical NOT, retrieving assets with the director Josh, excluding those with the actor Marc, use the - symbol:
    curl -X GET 'https://player.ooyala.com/opm/v1/assets?
    q=(metadata.display.director:"Josh" -metadata.display.actors:"marc")'

    More Query Examples. This example illustrates how to query movies from the 80's:
    curl -X GET 'https://player.ooyala.com/opm/v1/assets?type=movie&q=metadata.year:[1980 TO 1989]'

    This example illustrates how to query all assets rated PG-13:
    curl -X GET 'https://player.ooyala.com/opm/v1/assets?
    pcode=FhN2MyOlEAUn2NZ0OM94L7XNcL0f&details=all&q=metadata.rating:"PG13"'

    You can implement predictive search using partial matching using * in your elastic search queries.

    For example, after a user types in "ab", you could make the following type of call:
    curl -X GET 'https://player.ooyala.com/opm/v1/assets?q=metadata.display-name:ab*&pcode=&details=metadata'

    This will give you all assets where the display name has a word that starts with "ab". Then, when the user types in an additional character, you would create another API call:
    curl -X GET 'https://player.ooyala.com/opm/v1/assets?q=metadata.display-name:abr*&pcode=&details=metadata'

    Now you can expect fewer results, since it will only match content where a word in the display name starts with "abr". You can also combine search parameters if you would like to search for other data, such as actor names.

    Specify the field by which to order the results. You may optionally add asc (the default) or desc to specify ascending order or descending order, respectively. You can also add created to specify ordering by the date of creation: in such cases specify created+asc (date of creation in ascending order) or created+desc (date of creation in descending order). Note that this parameter only works on the properties of the requested asset, and is not applied to any other content groups that may be included in the response.

/assets/{embed_code}/offers get

get

Return a list of all offers that are applicable to an asset having the specified embed code. Offers are restricted to types HD and SD, and the embed code is associated with a rendition having a rendition type specified in the metadata. This endpoint only returns those offers that either do not specify any rendition types or specify the rendition type contained in the metadata (HD or SD). An embed code refers to exactly one rendition.

Note: Offers that are different from the rendition type associated with the embed code will not be returned.

    • images: Returns a list of images associated with an asset.
    • offers: Returns a list of offers associated with an asset. Note that if an offer is not enabled, it is not included in the list that is returned. Also, if an offer is configured with an availability window and the current date is not within that availability window, the offer is not included in the list that is returned.
    • collections: Returns a list of collections to which the content group belongs.
    • collections_with_offers: Returns a list of only those collections having offers to which the asset belongs.
    • channels: Returns a list of channels associated with an asset.
    • metadata: Returns an asset's complete metadata.
    • labels: Returns the labels associated with an asset.
    • children: Returns summary data for child assets (e.g. episodes in a season). Only immediate children are returned.
    • renditions: Returns the available renditions for an asset (HD, SD, etc.). Note that renditions only exist for the lowest content group, such as episodes.
    • all: Returns all of the above.

  • page: (integer)

    Zero-based page index to return. Default value: 0

  • page_size: (integer)

    Maximum number of items to return at a time. Default value: 10

  • orderby <field> [asc | desc | created]: (string)

    Specify the field by which to order the results. You may optionally add asc (the default) or desc to specify ascending order or descending order, respectively. You can also add created to specify ordering by the date of creation: in such cases specify created+asc (date of creation in ascending order) or created+desc (date of creation in descending order).
