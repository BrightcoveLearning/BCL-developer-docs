Unique user tracking

  • If the player/client application wants to track the unique viewer they should send a unique Id for the user as the user parameter to the collector.
  • If the user is not provided or is blank, we use the fallback method of using the (Source IP address + the User-Agent String) as the unique identifier.
  • The value of the user parameter is never stored in the logs or database. Only a hash (using SHA-256) is stored.
  • No cookies are set by the collector.
