Overview: Brightcove Live API

In this topic, you will learn what the Policy API is for and how to use it. The Live API is a REST-based API that allows you to create and manage live streaming events, with or without server-side ad insertion (SSAI). It also allows you to create video-on-demand (VOD) assets from clips taken from a live stream.

Base URL

The base URL for the Live API is:

https://api.bcovlive.io

Headers

All requests are authenticated using an API key that will be provided to you when your account is set up. The key is passed in an X-API-KEY header. A Content-Type header is also required:

X-API-KEY : YOUR_APIKey
Content-Type: application/json

Endpoints and operations

The main operations for the Live API are creating and managing live streams, and generate VOD clips from live streams. These operations are carried out through requests to following endpoints, which are explained in more detail in the remainder of the document.

  • /v1/jobs: used to create and get detailed information about live streams
  • /v1/jobs: used to create and get detailed information about live streams
  • /v1/jobs/:jobId/cancel: used to stop a live stream
  • /v1/jobs/:jobId/cuepoint: used to manually insert a cuepoint for an ad break into a live stream
  • /v1/vods: used to generate VOD clips from live streams

/v1/jobs/ endpoint

Creating live streams

This endpoint is used to create live streams via a POST request. In addition to specifying properties of the live stream itself, the request can also specify VOD clips to be generated from the live stream (this can also be done later via the endpoint). Details of the fields that can be included in the request body are given in the API Reference, along with a number of sample request bodies for typical use cases.

/v1/jobs/:jobId endpoint

This endpoint is used to get details of a live stream job you have created via a GET request. Details of response fields returned can be found in the API Reference

/v1/jobs/:jobId/cancel endpoint

This endpoint is used to stop a live stream via a PUT request. No request body is needed. Once canceled, a live stream cannot be restarted.

/v1/jobs/:jobId/cuepoint endpoint

This endpoint can be used to insert a cuepoint for an ad break in an existing live stream. See the API Reference for details.

/v1/vods endpoint

This endpoint allows you to create VOD clips from live streams via a POST request. The start and end points for the clips can be defined in terms of offsets from the start of the stream or UNIX timestamps, and you can push the clips to a server, an S3 bucket, or a Video Cloud account. See the API Reference for details and examples of the request body.
