Brightcove OnceUX Implementation Guide

Overview

This guide provides the necessary information to implement the Brightcove OnceUX ad service within your workflow to obtain a Once URL along with metadata to describe the ad load, including timing, beacons, and captions.. OnceUX is an add-on for OnceVOD, and this document assumes that you are already familiar with the contents of Getting Started with OnceVOD.

Dependencies

OnceUX depends on OnceVOD, so you will need to have a OnceVOD account and OnceUX enabled. Contact your Brightcove representative for assistance.

Syntax

The OnceUX is a pre-configurable web method, based on Brightcove OnceVOD technology that allows you to make a content request and receive a single normalized response, including all ad information, in a defined XML Schema; along with your video content and in-stream video ads.

Information Gathering

Several pieces of information are needed to take advantage of the OnceUX URL construct. Resulting Once URL(s) within responses will follow OnceVOD convention for {deliveryType} and {RequestedFileType} based on the OnceUX request syntax decisions you make as listed below. (See Getting Started with OnceVOD for more information)

VMAP Ad Aggregation Method

With this method, OnceUX retrieves a multiple ad playlist from values that are derived by summarizing or aggregating the ad viewing data based on your application (player, iPhone, etc.) from appropriate ad servers. This OnceUX URL template is pre-defined to elicit a VMAP 1.0 response providing control over ad inventory insertion where you don't control the player application.

You will create a OnceUX URL that sends a request to Brightcove servers for content. The data returned is a list of all the ads for that content viewing session normalized into VAST 3.0 and then embedded into a VMAP 1.0 XML object. This VMAP object contains a Brightcove Once XML object, which includes Brightcove Once data and a URL. See the sample response example.

NOTE: The VAST 3.0 object for each creative in the VMAP object excludes the download link as the URL provided in the Once XML object incorporates the ad resources at the correct locations.

The greatest difference between these two methods is that, in addition to the Once URL, the Ad Aggregation method also provides metadata and a VMAP object containing the ad load for the entire asset. This gives you greater client side flexibility with advertising while still using Brightcove Once platform. Using OnceUX gives you the ability to handle ad features normally unsupported by Once like banners, click-throughs, and VPAID.

Brightcove OnceUX utilizes a proprietary URL syntax based on OnceVOD technology. You configure it through a programmatic URL generation method customized for your specific application.

Segment variables

The VMAP method implements the Ad Server Aggregator behavior, in which content is requested and a fully-formed response is given with a OnceUX URL, metadata and a VMAP object containing the ad load for the entire asset.

Disable Tracking Events

UMTP={integer}

UMTP is used to override or disable the Brightcove server side ad video impression and event tracking default behavior.

  • UMTP=0

By default, Brightcove fires linear ad video impression and tracking events to the ad server on behalf of the client/application. If the parameter is explicitly set, UM does not fire the impression or tracking event.

Output

output={integer}

This Key-Value pair controls the object format of the response Brightcove returns.

  • XML=0
  • JSON=1
  • JSONP=2

NOTE: When using output=2, the 'callback' parameter is always required.

Callback

callback={string}

This Key-Value pair is the callback method name to use when writing out the JSONP response.

NOTE: System default is '?output=0&type=0'. If those are the desired format and type it is not necessary to add parameters to the query string.
