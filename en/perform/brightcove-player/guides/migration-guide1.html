Smart Player to Brightcove Player Migration Guide

This guide will help you transition from using the Smart Player to using the Brightcove Player.

The Brightcove player is a high-performance, lightweight, HTML5-first video player that loads quickly and delivers high-quality video across desktop and mobile platforms. It has many advantages, including a rich plugin environment to add features and funtionality. It also integrates with your favorite advertising and analytics systems. To learn more about this player, see the Brightcove Player Overview document.

This guide contains the following:

  • General information and concepts about the Brightcove Player
  • Player configuration - See how your current Smart Player can be created in the Brightcove Player environment

General information

It is recommended that you first review the general information in this section to help you understand the new environment around the Brightcove Player.

Media content

You will have access to all of your video content that you have uploaded using legacy Studio. Simply login to the new Studio platform and browse your content in the Media module. You may need to update the renditions you have to take advantage of the Brightcove Player's playback technology.

Ingest profiles

When videos are uploaded using the legacy studio, the renditions that are created are based upon the transcode settings configured for the account. All uploaded videos use the same transcode settings. In the new studio, videos uploaded using the Upload module use ingest profiles to determine which, and how many, renditions are created. Different videos can use different ingest profiles. Publishers can also create their own ingest profiles.

Supported video formats

Playback technology

The Smart Player used Flash-first technology to play your content. If the browser did not support Flash, then HTML technology was used with a feature-limited player. The Brightcove Player is able to play most of your content using HTML5, including HLS. If it cannot, it will fall back to using Flash.

For details, see the following:

  • Playback technology guide

Player implementations

The Smart Player has only one implementation, the <object> tag you used in your HTML page. Brightcove Player has two implementations you can use, a <video> tag as well as an <iframe> tag. While the <video> tag is most similar to the Smart Player implementation, the <iframe> tag offers some distinct advantages.

Player templates

When creating a new Smart Player, all players are based upon a player template. Player templates are created using BEML. Player templates are provided by Brightcove and publishers also have the ability to create their own custom templates.

The two APIs are markedly different. Concepts what you can do are similar, but implementation can be quite different.

  • Player API

Captions

With Smart Players, captions must be in DFXP format. Captions can be hosted by Brightcove or a remote url can be specified.

The same is true with Brightcove Players with the exception that captions in Brightcove Players must use the WebVTT format.

  • Add Captions to Videos
  • Convert Captions to WebVTT Format
Domain restrictions

Controlling player availability can be done by restricting your player to a specific domain URL. This feature is available for both the Smart Player and the Brightcove Player.

  • Content Protection
Geographic filtering

Geographic filtering allows you to restrict access to your content based on the viewer's geographic location. With the Smart Player you could set geo restriction at the player level as well as the video level.

With the Brightcove Player, you set geo restriction at the video level.

  • Geo-Filtering Messaging
Logo overlay

Smart Players can be configured to display a logo overlay. Logo overlays display a logo in the corner of the video playback window. Viewers can click through to a specified web site. Logo overlays can be configured for a single video or as part of the account settings so they appear on all the videos.

With Brightcove Players, logo overlays can be configured using the overlay plugin. There is currently not a way to have an overlay appear over all of the videos in an account.

  • Display Overlay Plugin
  • Toggling an Overlay

Player styling

In Smart Player the styling and manipulation of the player was done using BEML.

With Brightcove Player basic web development technologies, CSS, HTML and JavaScript, are used to style and manipulate the player.

Playlists

The Smart Player offered several options for displaying a playlist in a player.

For now, the Brightcove Player has the ability to display a vertical playlist to the right of the player. While you can style the playlist yourself, out-of-the-box styles are coming.
