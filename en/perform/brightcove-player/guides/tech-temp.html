Playback Technology Guide

Playback Technology refers to the specific browser or plugin technology used to play the video or audio. (The playback technology will usually be referred to simply as a Tech.)

Browser technologies

Here are some of the playback technologies that browsers use for video or audio:

HLS

HTTP Live Streaming (HLS) is an adaptive streaming protocol created by Apple. HLS can distribute both live and video on demand files. The source is divided into small pieces, usually between 5-10 seconds long.

HTML5

When using HTML5, the playback technology is the video or audio element. For video, the <video> element delivers content without plugins. Since video format support varies between browsers, this element supports fallback by providing multiple sources.

Flash Player

Adobe Flash Player allows the viewing of video content, and runs from a web browser as a plugin.

When using Flash, the playback technology is the specific Flash player used, e.g. Flowplayer, YouTube Player, video-js.swf, etc. (not just "Flash"). This could also include Silverlight, Quicktime, or any other plugin that will play back video in the browser, as long as there is an API wrapper written for it.

Brightcove player techs

With the Brightcove player, the default Tech order is ["hls","html5","flash"]. Let's examine how the player determines which playback technology to use:

HLS tech

The Brightcove Player includes the HLS playback engine tech by default, but HLS playback depends on browser support.

If the video you provide to the player includes HLS renditions, it will play those before it tries MP4 because the default Tech order is ["hls","html5","flash"].

If the browser supports HLS natively, it will use that capability (e.g. Safari on desktop or iOS).

For browsers that do not natively support HLS, the Brightcove player uses a Flash shim to play HLS content.

HTML5 tech

If the video does not have HLS renditions, or the browser does not support HLS playback, then the player will try to use HTML5.

Flash tech

For browsers that don't support the HTML5 <video> tag, the Brightcove player uses Flash to polyfill support, like playback of MP4 videos on IE8.

You can change this behavior by changing the tech order, as explained in the Adding Playback Technology section, but if you want to take advantage of HLS everywhere the default is the right configuration.

With the Flash playback tech, you may notice some undesirable playback behavior in browsers that are subject to using this playback tech, in particular when scrubbing and seeking within a video. This behavior is a result of Flash's progressive video playback.

Additional techs are supported by Video.js, but they are not part of the official Brightcove player product. For details, see the Video.js Playback Technology guide.

Determining playback tech

It may be helpful to determine which playback tech is being used by your player within the browser. Let's take a look at a video playing in the Chrome browser.

techName property

First, let's see which playback tech is currently being used. With your video in the Chrome browser, open Developer Tools to view the console messages. In the console's command prompt, enter the following:

Note: The vjs_video_3 section of the entry represents the default ID of the player in this case. If you are experimenting and have a player with an ID, that ID will replace vjs_video_3.

videojs.players.vjs_video_3.techName

You should see the value of "Hls". Remember that the default tech order is ["hls","html5","flash"].

You might not expect the hls rendition to be playing in this browser, since Chrome does not natively support hls. But, the Brightcove player will play hls within the Flash player, if Flash is installed in the browser.

vjs-tech class

Next, let's verify that Flash is being used to play the hls rendition. In the console's command promt, enter the following to return the first player element which uses the class named vjs-tech:

videojs.players.vjs_video_3.el().querySelector('.vjs-tech')

Here, you can see that Flash player is being used.

HTML5 rendition

Now, let's see which rendition is used when you turn off Flash player in the browser.

In Chrome, open a new tab and enter the following to navigate to the browser plugins:

chrome://plugins/

On this page, disable the Adobe Flash Player.

In the console's command prompt, enter the following:

videojs.players.vjs_video_3.techName

You should see that the HTML5 rendition is being used.

In the console's command promt, enter the following :

videojs.players.vjs_video_3.el().querySelector('.vjs-tech')

You can see that the mp4 video is playing in the HTML5 video element. Return to the browser plugins page and enable Adobe Flash Player.

Adding Playback Technology

When adding additional Tech to a video player, make sure to add the supported tech to the video object.

Tag Method

You can use the techOrder property in the video element of your in-page embed code:

<video data-setup='{"techOrder": ["hls", "html5", "flash", "other supported tech"]}'

Player Management API Method

Alternatively, if not using the in-page embed player implementation or want the value set for all player instances, you can configure the tech order using the Player Management API using an HTTP PATCH method, as shown here:

curl \
  --header "Content-Type: application/json" \
  --user $EMAIL \
  --request PATCH \
  --data '{
      "techOrder": ["html5", "hls", "flash"]
    }' \
  https://players.api.brightcove.com/v1/accounts/$ACCOUNT_ID/players/$PLAYER_ID/configuration

Creating your own tech

To create your own tech, you will need to build a wrapper for it. For guidance on writing a wrapper, the best resource is the Video.js source, where you can see how both the HTML5 and video-js.swf API wrappers were created.
