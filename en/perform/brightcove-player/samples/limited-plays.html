Player Sample: Limited Plays

In this topic, you will learn how to use a Brightcove player that limits how many times a user can view a video in a single day.

This sample adds and updates a cookie in the user's web browser. This may be a potential privacy concern in certain geographic areas.

Development sequence

Here is the recommended development sequence:

  1. Use the embed_in_page player implementation to test the functionality of your player, plugin and CSS (if CSS is needed)
  2. Put the plugin's JavaScript and CSS into separate files for local testing
  3. Deploy the plugin code and CSS to your server once you have worked out any errors
  4. Use Video CloudPerform Studio to add the plugin and CSS to your player
  5. Replace the embed_in_page player implementation if you determine that the iframe implementation is a better fit (detailed in next section)

For details about these steps, review the Quick Start to Plugin Development guide.

iframe or embed_in_page

When developing enhancements for the Brightcove player you will need to decide if the code is a best fit for the iframe or embed_in_page implementation. The best practice recommendation is to build a plugin for use with an iframe implementation. The advantages of using the iframe player are:

  • No collisions with existing JavaScript and/or CSS
  • Automatically responsive
  • The iframe eases use in social media apps (or whenever the video will need to "travel" into other apps)

Although integrating the embed_in_page player can be more complex, there are times when you will plan your code around that implementation. To generalize, this approach is best when the containing page needs to communicate to the player. Specifically, here are some examples:

  • Code in the containing page needs to listen for and act on player events
  • The player uses styles from the containing page
  • The iframe will cause app logic to fail, like a redirect from the containing page

Even if your final implementation does not use the iframe code, you can still use the embed_in_page code with a plugin for your JavaScript and a separate file for your CSS. This encapsulates your logic so that you can easily use it in multiple players.

The player

This example uses a Brightcove player that sets a browser cookie each time the viewer watches the video. After two video playbacks, an error message will display stating that you have reached the view limit for the day.

Start video playback, and then refresh the page. You don't have to watch the entire video. Then repeat. The third time you try to play the video, you should see an error message.

Clear your browser's history/browsing data to remove the cookie and start over.

How it works

The following sections explain the player, JavaScript and CSS code used in the example.

Create the player

To create a player and assign a video to it, follow these steps:

  1. In the Players module, create a new player or use an existing one.
  2. In the Settings section, select the Edit button and set the player's Width and Height based on your video's aspect ratio.
  3. Click Save and then Publish the player.
  4. In the Media module, select a video and publish it with this player.Assign video content to the player and publish it.
  5. Copy the Advanced (In-Page) embed code and paste it into a new HTML file.
  6. In the <video> tag for the embed code, add an id attribute with a unique value (see line 1 below).

For details on how to get this embed code, see the Quick Start to Brightcove Player document.

The embed code should look like this:

<video id="video_1"
  style="width: 640px; height: 360px;"
  data-video-id="4845798282001"
  data-account="1752604059001"
  data-player="default"
  data-embed="default"
  class="video-js"
  controls></video>
<script src="//players.brightcove.net/1752604059001/default_default/index.min.js"></script>

The HTML code

For testing purposes, we'll add an element to display the number of video views have been watched.

  1. Add the following HTML to your page code, just below the player embed code:
  • Line 1: Defines an element that will be used to display the number of video views the user has watched. 
<p>Number of views = <span id="cookieDisplay1"></span></p>

The JavaScript code

Here, you will add code to create and increment a browser cookie each time a video is viewed. When the user view limit is reached, an error message is displayed.

  1. Add the following JavaScript to your page code:
  • Lines 3-6: Define a function that reads the browser cookie.
  • Line 8: Gets a reference to the player when it is ready.
  • Line 11: Calls the function to read the browser cookie named "BC_views".
  • Lines 13-21: Defines a maximum views error message.
  • Line 23: Displays the number of user views.
  • Lines 24-30: Checks if the cookie value exceeds the maximum views limit. If so, then it displays our custom error message.
  • Lines 32-37: Increments the cookie value when the video is played. This cookie expires in 24 hours.
  • Lines 39-44: Hide the ok and close buttons in our custom error message when it is displayed. This keeps the user from closing the error message and playing the video.
<script type="text/javascript">
videojs("video_1").ready(function () {
	var read_cookie = function (key) {
		var result;
		return (result = new RegExp('(^|; )' + encodeURIComponent(key) + '=([^;]*)').exec(document.cookie)) ? result[2] : null;
	}

	var myPlayer = this,
		videoStart = 0,
		currentPosition,
		cookie = read_cookie("BC_views");

	myPlayer.errors({
		"errors": {
			"-3": {
				"headline": "You've reached your maximum views.",
				"type": "USER_MAX_VIEWS",
				"message": "Please come back tomorrow."
			}
		}
	});

	document.getElementById("cookieDisplay1").innerHTML = cookie;
	if (cookie != null && cookie >= 2) {
		myPlayer.on("loadedmetadata", function () {
			myPlayer.error({
				code: '-3'
			});
		})
	}

	myPlayer.one("play", function () {
		var d = new Date();
		d.setTime(d.getTime() + (1 * 24 * 60 * 60 * 1000));
		cookie++;
		document.cookie = "BC_views=" + cookie + ";expires=" + d.toUTCString();
	});

	myPlayer.on("error", function () {
		setTimeout(function () {
			myPlayer.errorDisplay.el().querySelector('.vjs-errors-ok-button').classList.add('vjs-hidden');
			myPlayer.errorDisplay.el().querySelector('.vjs-errors-close-button').classList.add('vjs-hidden');
		}, 50)
	});

});
</script>

Use a plugin

It is a best practice to use a plugin so that you can easily add this functionality to multiple players.

Create the JavaScript file

Move the plugin's JavaScript code to a file located on your machine for easy testing.

  1. Create a new file for your plugin JavaScript code. In this case, name it limited-plays.js.
  2. Cut and paste the JavaScript code from your main HTML file into this new file.

Your JavaScript code should look similar to this:

  • Lines 1,42: Wrap your JavaScript code using the videojs.plugin() function. The first argument, limitedPlays, is the plugin name.
  • Line 2: Gets a reference to your player.
  • The rest is your JavaScript code which was reviewed in a previous section, except we have removed the page element to display the number of views.

Line 42: Has a trailing semicolon at the end of the plugin's JavaScript code which is required.

      }
    });

    if (cookie != null && cookie >= 2) {
      myPlayer.on("loadedmetadata", function () {
        myPlayer.error({
          code: '-3'
        });
      })
    };

  });
  1. In the HTML file, add the following code just below the existing <script> tags. This code includes the JavaScript file and then calls the method defined in that JavaScript.
<script type="text/javascript" src="limited-plays.js"></script>

Implement using Players module

To implement your plugin using the Players module, follow these steps:

  1. Copy the plugin's JavaScript file and place it in an Internet accessible URL.
  2. Open the Players module and locate the player.
  3. Click the link for the player to open the player properties.
  4. Locate the Plugins section and click Edit.
  5. For the JavaScript URL, enter the URL for the file you created in step 13.
  6. For the Name, enter your plugin name. In this example, we are using limitedPlays.
  7. Click Save and then Publish the player.
  8. In the Media module, select a video and publish it with this player.
  9. Copy the Advanced (In-Page) embed code and paste it into a new HTML file. Your page should look similar to this:
<!doctype html>
<html>

<head>
	<meta charset="UTF-8">
	<meta name="viewport" content="width=device-width, initial-scale=1.0">
	<title>Sample: Limited Plays</title>
</head>

<body>

	<video style="width: 640px; height: 360px;"
		data-video-id="4845798282001"
		data-account="1752604059001"
		data-player="HJen7XCbS"
		data-embed="default"
		class="video-js"
		controls></video>
	<script src="//players.brightcove.net/1752604059001/HJen7XCbS_default/index.min.js"></script>

</body>
</html>

  1. Browse the HTML page and you should see the limited plays message functioning properly.

Use iframe

It is a best practice to use the iframe player implementation. If it is possible for this example, replace the Advanced (In_Page) embed code with the iframe implementation.

Here is the complete code:

<!doctype html>
<html>

<head>
	<meta charset="UTF-8">
	<title>Sample: Limited Plays</title>
</head>

<body>

	<iframe style="width: 640px; height: 360px;"
			src="//players.brightcove.net/1752604059001/HJen7XCbS_default/index.html?videoId=4845798282001"
			allowfullscreen
			webkitallowfullscreen
			mozallowfullscreen></iframe>

</body>
</html>

  1. Browse the HTML page and you should see the limited plays message functioning correctly.
