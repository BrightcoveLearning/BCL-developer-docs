Concepts: The Module Pattern

In this topic, you will learn how to use the JavaScript Module Design Pattern to reduce the chance that your code will conflict with other scripts on your web page.

Scope conflicts

By default, JavaScript functions and variables exist within the scope of the page. This means that they are vulnerable to colliding with other scripts on your page.

Test your setup

To run cURL commands, you just type curl (with appropriate flags for the request you want to make) at the Terminal or DOS prompt, and then pressing Return. To make sure cURL is correctly installed, open a Terminal or DOS window, type curl at the prompt, and press Return. You should see a message like this:

curl: try 'curl --help' or 'curl --manual' for more information

Sending Data

If you need to send data with the API request (as in a POST or PATCH request, for example), there are several ways to do it. If you need to submit the data as a form field, use the --form (or -F) flag, and then include the data inline or in a separate file. Here is a Media API example:

$ curl -F json=<afl.json https://api.brightcove.com/services/post
Attach the file as json=<afl.json, not json=@afl.json - the "@" attaches the file as an upload, while the "<" just sends the file contents.
