The HLS Player for Android

This document explains the architecture of the HLS Player and its relation to the SeamlessVideoView and BrightcoveVideoView.

Introduction

The HLS player consists of a number of classes which extend classes from the Brightcove Player SDK for Android. First, there is the SeamlessVideoView, which extends the BrightcoveVideoView by replacing:

  • the BrightcoveSurfaceView with a SeamlessSurfaceView
  • the DefaultSourceSelectionController with a SeamlessSourceSelectionController
  • the VideoDisplayComponent with a SeamlessVideoDisplayComponent

The HLS player also adds seekToLive() to support live streams.

Surface view

SeamlessSurfaceView, which extends BrightcoveSurfaceView, overrides onSizeChanged() to set the fixed size of the surface and overrides invalidate() and requestLayout() to prevent unnecessary measure and layout passes, which can cause flickering. Its SurfaceHolder also overrides setFixedSize() to prevent setting the surface to the same size as the video width and height. This allows the layout constraints to determine the width and height of the surface.

Selection controller

SeamlessSourceSelectionController overrides getMinimumHLSVersion() to allow HLS playback as far back as Gingerbread (Android 2.3.3).

Video display component

The bulk of the HLS implementation is in SeamlessVideoDisplayComponent, which extends VideoDisplayComponent. This class uses a voOSBasePlayer instead of an Android MediaPlayer to render the video. It also contains a second voOSBasePlayer instance, which is used to pre-buffer subsequent videos in a playlist. Just like VideoDisplayComponent, a ScheduledFuture is used to emit PROGRESS events at a fixed interval of 500 milliseconds.

Some methods, like getMediaPlayer() become unsupported, as they are replaced by methods like getSeamlessPlayer().

Other methods, like getClosedCaptionsEnabled(), setClosedCaptionsEnabled(), redraw(), isLive(), setBufferMaxSize(), provide additional functionality.
