OAuth Guide

Brightcove has created an OAuth 2 system that will be used across all of our new APIs for authorization. You don't need to know anything about OAuth in order to use the system, and we'll explain everything you need to do to get started below. Also see the documentation on the Brightcove OAuth API.

Get started

The authorization explained in this document allows you to use curl statements to communicate with the Player Service API. The curl tool is used at the command line to transfer data with URL syntax. More information about curl can be obtained from http://curl.haxx.se.

A few preliminary steps need to be taken before you can start to use the API. These are:

  • Login to Studio. If you have multiple accounts, use the drop-down to select the one in which you'd like to create your new players. For the credential system to work properly, you do need to have admin rights on this account. If you are unsure if your user has admin rights, try going to the Users Settings page in Studio. If you see users listed on this page, you have admin rights.
  • On the home page of Studio, just below the menu, you will see your name and a Publisher ID below that. Copy your Publisher ID.

    • The Publisher ID from Studio is called the Account ID in the API. For this reason, in the rest of this document the ID will be referred to as the Account ID.

  • In Terminal enter the following to assign the account ID value to an environment variable:
    • export ACCOUNT_ID=YourAccountID
  • Authentication will be handled by supplying your Brightcove account email address as part of the curl statement, and then the API will prompt you for a password. Since the email address will be used frequently, also assign that value to an environment variable:
    • export EMAIL=YourEmailAddress

You are now ready to start using the API.

Other API usage

The client ID and client secret obtained in "One time steps" are standard client credentials that can be used with any standard OAuth 2 client. Set this up with https://oauth.brightcove.com/v3/access_token as the access token endpoint for the client credentials flow. You'll find many libraries out there for your language of choice. And while we provide one way of doing things on the command-line in the command-line section above, there's other command-line utilities for OAuth 2 for making things easier.
