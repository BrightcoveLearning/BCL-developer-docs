Brightcove Player Sample: Loading Multiple Videos

This document shows how to manually build a playlist. Since Video Cloud has playlists as a feature, this is content is only pertinent to Brightcove Player customers.

Video Cloud customers see the following documents on playlists and manually controlling playlists.

  • Using Playlists
  • Playlist API Guide

This topic shows you how to create a Brightcove player that loads successive videos from an array. The next video in the array is loaded when the current video finishes playing.

Development sequence

Here is the recommended development sequence:

  1. Use the In-Page embed player implementation to test the functionality of your player, plugin and CSS (if CSS is needed)
  2. Put the plugin's JavaScript and CSS into separate files for local testing
  3. Deploy the plugin code and CSS to your server once you have worked out any errors
  4. Use Studio to add the plugin and CSS to your player
  5. Replace the In-Page embed player implementation if you determine that the iframe implementation is a better fit (detailed in next section)

For details about these steps, review the Quick Start to Plugin Development guide.

iframe or In-Page embed

When developing enhancements for the Brightcove player you will need to decide if the code is a best fit for the iframe or In-Page embed implementation. The best practice recommendation is to build a plugin for use with an iframe implementation. The advantages of using the iframe player are:

  • No collisions with existing JavaScript and/or CSS
  • Automatically responsive
  • The iframe eases use in social media apps (or whenever the video will need to "travel" into other apps)

Although integrating the In-Page embed player can be more complex, there are times when you will plan your code around that implementation. To generalize, this approach is best when the containing page needs to communicate to the player. Specifically, here are some examples:

  • Code in the containing page needs to listen for and act on player events
  • The player uses styles from the containing page
  • The iframe will cause app logic to fail, like a redirect from the containing page

Even if your final implementation does not use the iframe code, you can still use the In-Page embed code with a plugin for your JavaScript and a separate file for your CSS. This encapsulates your logic so that you can easily use it in multiple players.

Select implementation

In this sample, you will test with the In-Page Embed code and then move the CSS and JavaScript into a plugin. Since this example uses JavaScript code that manipulates elements on the page, an iframe implementation will not work.

The player

How it works

The following sections explain the player, JavaScript and CSS code used in the example.

Create the player

To create a player and assign a video to it, follow these steps:

  1. In the Players module, create a new player or use an existing one.
  2. Assign video content to the player and publish it.

  3. Copy the In-Page embed code and paste it into a new HTML file.

    The embed code should look similar to this:

    <video id="video_1"
    data-account="3676484087001"
    data-player="4k1SOaZFe"
    data-embed="default"
    class="video-js" controls=""></video>
<script src="//players.brightcove.net/3676484087001/4k1SOaZFe_default/index.min.js"></script>
    For details on how to get this embed code, see the Quick Start to Brightcove Player document.

The HTML code

This example uses additional HTML code.

  1. Add the following HTML to your page code:

    • Lines 30-31: Define the display area for the video name and calculated time remaining.

    • Line 34: Links to a script which extracts the hours, minutes and seconds from seconds.

    Line 33: Make sure to move the player <script> tag below the page HTML code. The Brightcove player script must be placed after your page code. It is common practice to put your scripts just before the closing </body> tag.
    <div class="player-block">
  <video id="video_1"
    data-account="3676484087001"
    data-player="4k1SOaZFe"
    data-embed="default"
    class="video-js" controls=""></video>
  <p id="videoName"></p>
  <p id="timeRemaining"></p>
</div>
<script src="//players.brightcove.net/3676484087001/4k1SOaZFe_default/index.min.js"></script>
<script src="//solutions.brightcove.com/bcls/scripts/secondsToTime.js"></script>

The JavaScript code

Define the video array, and load each successive video from the array when a video finishes playing.

  1. Add the following JavaScript to your page code:

    • Line 87: Gets a reference to the player when it is ready.
    • Lines 88-90: Listen for the ended event and calls the loadVideo()method.
    • Lines 91-95: Listen for the timeupdate event and calculates the time remaining by using using the duration() and currentTime() methods. Then, it displays the current time on the page.

    • Lines 70-85: Use the src() method to load the next video in the array. It displays the video name on the page and uses the play() method to start playing the video.

    For this sample, the playlist data is hard-coded in an array, but you could easily read this data from an external file.
    <script type="text/JavaScript">
    var myPlayer,
      eVideoName = document.getElementById("videoName"),
      eTimeRemaining = document.getElementById("timeRemaining"),
      timeRemaining,
      totalTime,
      currentVideoIndex = 0,
      newVideo,
      firstVideo = true,
      playlistData = [{
        "name": "Great Blue Heron",
        "thumbnailURL": "//solutions.brightcove.com/bcls/assets/images/Great-Blue-Heron.png",
        "sources": [{
          "type": "application/x-mpegURL",
          "src": "//solutions.brightcove.com/bcls/assets/videos/Great-Blue-Heron.m3u8"
        }, {
          "type": "video/mp4",
          "src": "//solutions.brightcove.com/bcls/assets/videos/Great-Blue-Heron.mp4"
        }]
      }, {
        "name": "Birds of a Feather",
        "thumbnailURL": "//solutions.brightcove.com/bcls/assets/images/BirdsOfAFeather.png",
        "sources": [{
          "type": "video/mp4",
          "src": "//solutions.brightcove.com/bcls/assets/videos/BirdsOfAFeather.mp4"
        }]
      }, {
        "name": "Sea Marvels",
        "thumbnailURL": "//solutions.brightcove.com/bcls/assets/images/Sea Marvels.png",
        "sources": [{
          "type": "video/mp4",
          "src": "//solutions.brightcove.com/bcls/assets/videos/Sea-Marvels.mp4"
        }]
      }];
    function loadVideo() {
      if (currentVideoIndex < playlistData.length) {
        // load the new video
        myPlayer.src(playlistData[currentVideoIndex].sources);
        // update the video title display
        eVideoName.innerHTML = "Now playing: <strong>" + playlistData[currentVideoIndex].name + "</strong>";
        // increment the current video index
        currentVideoIndex++;
        // play the video
        if (firstVideo) {
          firstVideo = false;
        } else {
          myPlayer.play();
        }
      }
    };
    videojs("video_1").ready(function () {
      myPlayer = this;
      myPlayer.on("ended", function () {
        loadVideo();
      });
      myPlayer.on("timeupdate", function (evt) {
        timeRemaining = Math.round(myPlayer.duration() - myPlayer.currentTime());
        totalTime = BCLSsecondsToTime(timeRemaining);
        eTimeRemaining.innerHTML = "Time remaining = " + totalTime.m + ':' + totalTime.s;
      });
      // load the first video
      loadVideo();
    });
  </script>

The stylesheet

This example uses CSS styles for the display fields and the Brightcove player.

  1. Add the following CSS to your page code:

    • Lines 9-13: Define the layout and look of the video name and time remaining fields.
    • Lines 14-19: Set the dimensions and position of the player.
    <style id="pageStyles">
  #videoName,
  #timeRemaining {

    color: #990000;
  }
  #video_1 {
    width: 640px;
    height: 360px;
    margin-left: 20px;
    margin-top: 10px;
  }
</style>

    Browse your HTML page in the browser. You should see multiple videos play, one after the other.

Use a plugin

It is a best practice to use a plugin so that you can easily add this functionality to multiple players.

Create the CSS file

If you are using CSS, move styles into a file and place it in an Internet accessible URL.

  1. Create a new file for your CSS styles. In this case, name it multiple-video.css.
  2. In the <head> section of your HTML file, add the following link to include your newly created CSS file. 
    <link href="multiple-video.css" rel="stylesheet">
  3. Browse the HTML page. You should see that the multiple videos still display, except now your styles have been removed from the HTML page.

Deploy plugin and CSS

On your player's properties page, edit the Plugins section.

  1. Begin by specifying a JavaScript URL for the multiple video plugin and the seconds to time JavaScript file.
  2. Next, apply a CSS URL to define styles to the video player.
  3. Save and publish your changes.
  4. Copy the In-Page embed code and place it within your HTML page. Your page should look similar to this: 
    <!doctype html>
<html>

<head>
    <meta charset="UTF-8">
    <title>Multi Video Player with Plugin</title>
    <link href="multiple-video.css" rel="stylesheet">
</head>

<body>

    <div class="player-block">
        <video id="video_1"
            data-account="1752604059001"
            data-player="9e4a8265-12c0-4065-896f-bb872506518a"
            data-embed="default"
            class="video-js" controls></video>

        <p id="videoName"></p>
        <p id="timeRemaining"></p>
    </div>

    <script src="//players.brightcove.net/1752604059001/9e4a8265-12c0-4065-896f-bb872506518a_default/index.min.js"></script>

    <script src="//solutions.brightcove.com/bcls/scripts/secondsToTime.js"></script>

    <script type="text/javascript" src="mulitiple-video.js"></script>
    <script>videojs('video_1').multipleVideo();</script>

</body>
</html>
  5. Browse the HTML page and you should see the player with the plugin functioning correctly.

Since this example uses JavaScript code that manipulates elements on the page, an iframe implementation will not work.
