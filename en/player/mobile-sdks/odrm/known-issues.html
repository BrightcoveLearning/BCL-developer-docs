Offline Playback with DRM for the Native Player SDKs: Known Issues

The following areas have known issues which are currently under investigation by the Brightcove Native Player SDK teams.

Offline playback with DRM for the Native SDKs is currently a limited availability release.

Android

The following known issues are related to the Native SDK for Android.

Offline DRM Supported Versions

For the Limited Availability release, Offline DRM is not supported on Android version 4.x.

Download Performance

For the Limited Availability release, it is recommended to test with shorter downloads, which will contain fewer segments. Our internal testing has shown overall download performance can be slow because of the number of segments, Android download manager bugs etc.

Long-form Video Downloads

Testing has shown that long-form video downloads (i.e. downloads of videos greater than 20 minutes in length) overly stress the Android DownloadManager, and can cause the android.process.media process to crash. While this issue has been observed in all versions of Android tested, it is more prevalent in earlier versions of the operating system.

Multiple Concurrent Downloads

Related to the long-form download issue above, testing has shown that downloading multiple videos, regardless of the length of any individual video, also stresses the DownloadManager, and can cause Controller errors and download failures. This issue has been observed in all Android versions tested, but is more prevalent in older Android versions. Allow users to download a video only after the previous one has been successfully completed.

Intermittent Audio Problems During Playback or Seeking

Testing has uncovered intermittent audio dropouts during playback, seeking and orientation changes of the device while playing downloaded content. This issue does not persist for more than a few seconds as the player recovers. When this happens, the following will be written to the logcat: 

04-18 19:03:46.934 28764-28764/com.brightcove.player.test E/ExoPlayerVideoDisplayComponent: onAudioTrackUnderrun: bufferSize = 48000, bufferSizeMs = 250, elapsedSinceLastFeedMs = 1766

Selecting another downloaded video did not cause these errors. Further, the errors did not reproduce when going back to the downloaded video that originally experienced them.

Download File Management: Download Cancellation and Deletion

Testing has shown that canceling downloaded videos, and deleting downloaded videos, can sometimes fail to entirely clean up a download directory.

It is recommended that if a deleted video download is not entirely removed from the device, that it is possible to delete the remainder of the download through the Storage section of the Settings on the device.

For the Limited Availability release, overriding the download directory is not supported.
