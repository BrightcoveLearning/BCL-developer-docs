DRAFT - Brightcove Beacon Playlists

In this topic, you will learn how to create a playlist using the Playlists tab for Brightcove Beacon.

Introduction

A playlist is a collection of videos imported into Brightcove Beacon from your Video Cloud library. Playlists are displayed on your content page for users to select and watch.

Start creating a playlist

To create a playlist, follow these steps:

  1. Login to Brightcove Beacon.
  2. In the top navigation, select Playlists.
  3. Select the Add new playlist button.
  4. From the Basic Data tab, you will define the type of playlist that you want and add content.
  5. In the Name field, add an internal system name for your playlist. You will add the viewable title in a later step.
  6. If you want a dynamic playlist, check Dynamic playlist. When selected, video content that matches the tags you select will be automatically added to your playlist.
  7. Select a layout from View All Playlist Layout. Options include:
    • Carousel Posters 5 Columns - A carousel of poster images which is 5 columns wide and can be scrolled left or right
    • Carousel Landscape In - A carousel with 16 x 9 poster images
    • Grid 4 Columns Landscape Labels In - A 4 column wide carousel with 16 x 9 poster images. Labels will display over the poster image.
  8. Select a View all menu type. If the playlist is long, and only a few videos are displayed at a time, you can give users the ability to view all. Options include:
    • None - No option to view all videos
    • Tab - Tabbed list of all videos
    • Combobox - A dropdown menu where you can view the titles of all the videos
  9. Select a Playlist Reference.
    • No Reference - default

Use Builder to add content

The Builder tab and functionality allows you to add content to your playlist based on a set of rules that you define.

  1. By selecting Special playlist you will have to option to choose a special type of playlist.
  2. Select a Special Type of playlist.
    • Most Watched - Most watched videos playlist
    • Recommendations - Recommended videos playlist
    • Builder - Use the Builder tab below to create your playlist content
  3. If you choose Builder as special type, then you can use the Builder functionality to add content to your playlist.
  4. Select the builder options and select the Add button. Then, select the Update List button to see content added to your playlist on the left.

Manually add content

When the Special playlist option is not selected, you can use the other tabs to add content to your playlist.

  1. Make sure Special playlist is not selected.
  2. The Assets tab allows you to add content from your library of movies, series, seasons, episodes, channels and advertisements.
  3. In the Actions column, use the add icon to add an item to your playlist.
  4. In the Actions column, use the remove icon to remove an item from your playlist.
  5. The Seo-Tags tab allows you to choose content that has tag information associated with it.
  6. The Cast tab allows you to choose content based on actors, directors, music composers, singers song writers and writers.
  7. The Categories tab allows you to choose content based on categories that you have defined.
  8. The Genres tab allows you to choose content based on genres and sub-genres.
  9. Select Newest first to order your playlist by asset creation date.

Add Textual Data

The Textual Data section allows you to add displayable text for your playlist.

  1. Select the Textual Data tab.

  2. Add values for the following fields:

    Field Required Notes
    Title Yes The displayed title for your playlist.
    Description Yes The displayed Description for your playlist.
    Subtitle No A short synopsis about your playlist. This has a maximum of 188 characters.

Add Non-Textual Data

The Non-Textual Data section allows you to upload icon images for your playlist.

  1. Select the Non-Textual Data tab.

  2. Choose images from your computer to be used as playlist icon images.

    The maximum allowed upload size is 500 KB.

Add SEO Data

The SEO Data section is where you add information related to search engine optimization (SEO).

  1. Select the SEO Data tab.

  2. Add values for the following fields:

    Field Required Notes
    Friendly URL Yes This is generated automatically from the title you entered in the Textual Data section.
    SEO Title No The title used by search engines.
    SEO Keywords No Enter one or more keywords to make your content more easily found by search engines.
    SEO Description No The description used by search engines.

    The following table displays custom Video Cloud fields that you must create. These fields must exist for Brightcove Beacon to properly import the video with which the metadata is associated.

    Video Cloud Field Notes Required Type
    beacon.genre Genre for the asset - will show up in details page and will be used to organize assets in different screens. If this value is not set, then “No Genre” will be shown in the user interface. Not setting this value will have an impact on Related Content functionality. No Custom field
    beacon.productionYear Year of production of the asset - will show up in details pages No Custom field
    beacon.cast.director Comma separated list of names of directors No Custom field
    beacon.cast.singer Comma separated list of names of singers No Custom field
    beacon.cast.actor Comma separated list of names of actors No Custom field
    beacon.cast.composer Comma separated list of names of composers No Custom field
    beacon.cast.songWriter Comma separated list of names of song writers No Custom field
    beacon.cast.writer Comma separated list of names of writers No Custom field
    beacon.ageRating Provides the age rating for the asset. The value in this field should be in the form of key=value pairs separated by commas, where the key is the country code (like us) and the value is the age rating appropriate for that country code supported by Beacon. If this value is not set, the most permissive rating for the default country age rating system set in Beacon will be used. No Custom field
    beacon.shortDescription Allows to provide a short description tailored for Beacon - if this value is not set it uses the text set in the video short description field No Custom field
    beacon.longDescription Allows to provide a long description tailored for Beacon - if this value is not set it uses the text set in the video long description field No Custom field
    beacon.viewerScore Allow to set an initial score for the viewer ratings (either coming from some other system or to set the initial value).     beacon.trailer.id Points to another video asset in video cloud that will be used as a trailer for the asset where this field is set. The asset being pointed to will be discarded as a Movie or Episode No Custom field
    beacon.ingest If this field is not present, then it will default to ingest. If you do not want some specific asset to not be synched, then this should be set to No No Custom field

    Rights Management fields

    To support the monetization and access restrictions for assets, a set of custom fields should be used. As you can have multiples instances of rights management configurations, the suggested approach is to create multiple sets of customer fields. In each instance's name a counter is used to differentiate between them. The first instance should use the counter to 0.

    The following table displays custom Rights Management fields that you must create. These fields must exist for Brightcove Beacon to properly import the video with which the metadata is associated.

    Video Cloud Field Notes Required Default Value Type
    beacon.rights.<counter>.type Identifies if the type of monetization for the asset - possible values are: FREE, AVOD, SVOD and INHERIT; INHERIT is only applicable to Episode assets No FREE Custom field
    beacon.rights.<counter>.startDate Date when the asset should be exposed under the access restrictions and monetization options in this particular set No If not set, defaults to video availability start date Custom field
    beacon.rights.<counter>.endDate Date when the asset should be no longer exposed under the access restrictions and monetization options in this particular set No If not set defaults to video Availability end date Custom field
    beacon.rights.<counter>.devices Comma separated list of devices supported in Beacon to which the asset should be exposed under the access restrictions and monetization options in this particular set.Possible values are: iOS, Android, web, Roku, STV, Fire TV, LGTV, Samsung, Apple TV, Android TV and Panasonic No If not set defaults to all devices supported in Beacon Custom field
    beacon.rights.<counter>.locationsPermit Comma separated list of locations defined in Beacon to which the asset should be exposed under the access restrictions and monetization options in this particular set No If not set defaults to all locations supported in Beacon Custom field
    beacon.rights.<counter>.locationsDeny No Custom field
    beacon.rights.<counter>.packageName Yes if the type is set to SVOD Custom field
    beacon.rights.<counter>.adConfiguration Identifier of an ad configuration in Beacon Yes if the type is set as AVOD Custom field
