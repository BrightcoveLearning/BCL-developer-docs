Introduction

You can ingest HEVC videos to Video Cloud and have HEVC renditions output by the transcoding process.

To do this, you will need to contact Brightcove Support and have them create special HEVC ingest profiles for your account.

The output will be two HEVC renditions for DASH playback. These may be packaged for Widevine and/or Playready DRM if these are enabled for your account. The detailed characteristics of the renditions are shown in a later section.

Notes

  • The source file can be in AVC/H264 of HEVC/H265 format.
  • See Video Source File Specifications and Recommendations for other source file recommendations, but note that you may use H.265 as well as H.264 source.
  • The output will be H.265/HVEC only.
  • Only ingestion and transcoding are officially supported. Brightcove has tested playback only with Roku devices, not with the Brightcove or Ooyala players.

How to ingest videos

You can ingest videos in three ways:

  • Using the Upload Module in Studio
  • Using the Dynamic Ingest API
  • Using OIS

Either method is fine, but you must be sure to use the special ingest profile for HEVC discussed in the previous section.
