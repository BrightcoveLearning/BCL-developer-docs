Introduction

The Social Syndication API is a publicly accessible API that allows syndications to be created, managed, and used to generate dynamic feeds (such as MRSS feeds) from a VideoCloud account's video catalog.

OpenAPI Specification

The formal API specification is defined by a pair of OpenAPI specification files:

  • The Configuration API for creating and managing Syndications
  • The Feed API for accessing the feeds defined by Syndications

Authentication

Access to the Configuration API requires specification of a bearer token from the Brightcove OAuth service in the request's Authorization header. The various API methods also require one of the following operations (depending upon the method accessed) to be specified for the credentials in question:

  • video-cloud/social/mrss/read
  • video-cloud/social/mrss/write

These operations can be configured via the API Authentication section of the Studio Admin Module.

If you prefer, you can also create credentials via the OAuth API:

  • Create Client Credentials Using cURL
  • Create Client Credentials Using Postman

Access to the Feed API does not require authentication, as feeds are intended to be publicly accessible.

Syndication Resource

The syndication resource (also the response for creating a new syndication, and the request body, excluding read-only fields) looks like the following:

{
  // Read-only, generated when syndication is created
  "id": "7f594cd3-4853-4174-aff3-203c3e99e9c2",
  "name": "an internal name for this syndication",
  // If true, all catalog videos are included in this syndication
  "include_all_content": false,
  // Must be specified if include_all_content is false.  Value is a CMS search string using the
  // most recent CMS query syntax (see below)
  "include_filter": "tags:mytag",
  // Valid values are [advanced, google, iphone, ipad, mp4, itunes, roku, source, universal]  These dictate the type
  // of feed that will be generated.  The universal type allows custom feeds to be generated by an uploaded feed template.
  "type": "mp4"
  // The title and description of this feed. These are included inside of the  tag for applicable feed types.
  "title": "Latest racing videos",
  "description": "Amateur Tokyo drift!",
  // Read only.  The URL of this syndication's MRSS feed.
  "syndication_url": "https://social.feeds.brightcove.com/v1/accounts/9999999999999/mrss/syndications/7f594cd3-4853-4174-aff3-203c3e99e9c2/feed"
  // The URL to be included inside of the  tag for applicable feed types.
  "destination_url": "http://mywebsite.com",
  
  // Fields below are only used for itunes and (potentially) universal feeds

  "keywords": "80s, rock",
  "author": "Rick Astley",
  "subtitle": "Best Of Astley",
  "category": "Music",
  "album_art_url": "http://my_album_art.jpg"
}
Syndication Resource
Field Type Read-Only Description
id String Yes generated when syndication is created
name String No an internal name for this syndication
include_all_content Boolean No If true, all catalog videos are included in this syndication
include_filter String No Must be specified if include_all_content is false. Value is a CMS search string using the CMS query syntax
type String No The type of feed that will be generated. The universal type allows custom feeds to be generated by an uploaded feed template. Valid values: advanced, google, iphone, ipad, mp4, itunes, roku, source, universal.
title String No The title of this feed. This is included inside of the <channel> tag for applicable feed types
description String No The description of this feed. This is included inside of the <channel> tag for applicable feed types
syndication_url String Yes The URL of this syndication's MRSS feed
destination_url String No The URL to be included inside of the <channel> tag for applicable feed types
keywords String No comma-delimited list, only used for itunes and (potentially) universal feeds
author String No only used for itunes and (potentially) universal feeds
category String No only used for itunes and (potentially) universal feeds
album_art_url String No URL for image, only used for itunes and (potentially) universal feeds

See Search Syntax for the Syndication Resource for details on the include_filter property.
