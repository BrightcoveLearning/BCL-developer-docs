Implementation overview

To use playlists, complete the follow tasks:

  1. Create a playlist either in Studio or manually using JavaScript
  2. Enable your player to use playlists
  3. Use playlists with the iframe player implementation
  4. Use playlists with the in-page embed implementation
  5. Use playlists with the Playlist API, where you programmatically control playlist behavior

Enable player

You can use Video CloudPerform Studio to enable your player to use playlists. In the Settings section, click the Edit button then enable the display of playlists. Note the default playlist version is version 2.

iframe player

Once you have a playlist to use, and have enabled your player to use a playlist, you are ready to actually utilize the playlist with the player. As usual you have the iframe and in-page embed player implementations available. In this section, instructions for using the playlist with the iframe implementation are shown. When using the iframe, the playlist will appear on the right side of the player, as shown in this screenshot (NOT a functional player and playlist):

Note: If you need to position the playlist in a different location other than on the right side of the player, you will need to use the in-page embed implementation, detailed below.

To use a playlist with the iframe implementation, you append one of the following query strings to the src attribute of the iframe. If using an ID:

      <iframe src='//players.brightcove.net/.../index.html?playlistId=1754200320001'

or if you are using a reference ID:

      <iframe src='//players.brightcove.net/.../index.html?playlistId=ref:myrefid001'

If you added one of the query strings, then browsed the page with your iframe player implementation, chances are you would not see the playlist. The reason being is that you need to style the player to allow the playlist to show. For instance, the styles shown here could be placed in the head of your HTML page:

      <style>
        iframe {
          width: 88%;
          min-height: 360px;
        }
      </style>

Of course the actual styles and associated values you use will vary depending on the size of the videos being displayed and your HTML page layout.

An example HTML page using the iframe implementation is:

      <html>

      <head>
        <meta charset="UTF-8">
        <title>Untitled Document</title>
        <style>
          iframe {
            width: 88%;
            min-height: 360px;
          }
        </style>
      </head>

      <body>

        <iframe src='//players.brightcove.net/1752604059001/f50a2d3c-af51-4d8c-84e3-0c7cdec0edbe_default/
          index.html?playlistId=1754200320001' allowfullscreen webkitallowfullscreen mozallowfullscreen></iframe>

      </body>

      </html>

In-page embed player

Once you have a playlist to use, and have enabled your player to use a playlist, you are ready to actually utilize the playlist with the player. As usual you have the iframe and in-page embed player implementations available. In this section, instructions for using the playlist with the in-page embed implementation are shown. Just below is a screenshot of an in-page embed playlist. Note that in the in-page V2 implementation the playlist you do NOT get the functionality to hide and show the playlist, that is only with iframe implementations. This is because in the in-page V2 implementation the playlist does not overlay the player, and as a result does not have the show and hide button.

When using the in-page embed implementation of Brightcove Player the playlist element needs to be sized and positioned using CSS. This provides you with the flexibility to position the playlist separately from the player, in a location and direction of your choice. With these options you can present the playlist in many ways, as shown later in this document in the Moving the playlist section. When using a playlist with the iframe implementation of the player you simply don't have these options.

To use the playlist in the in-page embed implementation you need to do the following:

  • Add one of the following attributes to the <video> tag (the first is using a playlist ID, the second is using a reference ID): 
              data-playlist-id="5126480959001"
              data-playlist-id="ref:myrefid001"

    Here is an example <video> tag:

              <video data-playlist-id="5126480959001"
            data-account="1752604059001"
            data-player="H1EyAYN3"
            data-embed="default"
            data-application-id
            class="video-js"
            controls></video>
  • In your HTML page, in the location you wish the playlist to appear, insert an ordered list (<ol></ol>) with a class of vjs-playlist, as shown here: 
              <ol class="vjs-playlist"></ol>

The playlist will be injected into your HTML page at the location of the ordered list.

Chances are you will wish to style the player and playlist to appear the way you wish. For instance, the styles used for the player and playlist at the top of the document are:

            <style type="text/css">
              .video-js {
                height: 350px;
                width: 640px;
                float: left;
              }
              .vjs-playlist {
                width: 280px;
                height: 365px;
              }
          </style>

An example HTML page using the in-page embed implementation is:

          <!doctype html>
          <html>

          <head>
            <meta charset="UTF-8">
            <title>Untitled Document</title>
            <style type="text/css">
              .video-js {
                height: 350px;
                width: 640px;
                float: left;
              }
              .vjs-playlist {
                width: 280px;
                height: 333px;
              }
            </style>
          </head>

          <body>

            <video data-playlist-id="5126480959001"
            data-account="1752604059001"
            data-player="f50a2d3c-af51-4d8c-84e3-0c7cdec0edbe"
            data-embed="default"
            data-application-id
            class="video-js"
            controls></video>
            <script src="//players.brightcove.net/1752604059001/f50a2d3c-af51-4d8c-84e3-0c7cdec0edbe_default/index.min.js"></script>

            <ol class="vjs-playlist"></ol>

Set the initial video

By default, the first video in the playlist is initially loaded into the player. If you wish to load another video in the playlist first, there are three possibilities:

  1. You are using an in-page embed player on a page. You can add a URL parameter 
    ?playlistVideoId=5510487311001
    to the URL page request and the player will automatically play the desired video from the playlist. This is built in Brightcove Player functionality.
  2. You are using an iframe player on a page. You can add a URL parameter 
    <iframe src=".../index.html?playlistId=5531423971001&playlistVideoId=5510487311001" ...
    to the iframe's src and the player will automatically play the desired video from the playlist. This is built in Brightcove Player functionality.
  3. You are using an iframe player and wish to pass the desired video's ID on the URL page request. This differs from #2 as you do not want to have to edit the iframe's code on the actual page, but use the URL from the page request. This is NOT built in Brightcove Player functionality, but can be done with a custom code and custom plugin. The solution is detailed later in this section.

Changing iframe code on the page sample

      <div style="position: relative; display: block; max-width: 640px;"><div style="padding-top: 56.25%;"><iframe src="//players.brightcove.net/1752604059001/HkYoUgMwW_default/index.html?playlistId=5531423971001&playlistVideoId=5510487311001"
          allowfullscreen
          webkitallowfullscreen
          mozallowfullscreen
          style="position: absolute; top: 0px; right: 0px; bottom: 0px; left: 0px; width: 100%; height: 100%;"></iframe></div></div>

Customize appearance

You can think of a playlist divided into four main parts to which you can apply styles. They are:

  • The container of the playlist items. In HTML terms it is an <ol> element. The playlist container is surrounded in red below.
  • The playlist item representing the video currently in the player, called the selected item. In HTML terms it is an <li> element. The selected item is surrounded in orange below.
  • The descriptions in the items. In HTML terms they are <p> elements. An example description is surrounded in yellow below.
    Descriptions in the playlist only appear in version 1 of playlists.
  • All the playlist items that are not currently in the player. In HTML terms they are <li> elements. An example playlist item is surrounded in blue below.

The four parts are shown here with their selectors.

Each playlist item is composed of the following:

  • Containing list item element
  • Picture element
  • Image element, which is a child of the picture element
  • Time element
  • Citation element
  • Paragraph element for the description

CSS playlist examples

The playlist below is shown with default styling.

The playlist below uses the following CSS to change the background of the playlist as well the color of the fonts in the playlist. Note that the descriptions are NOT affected by the color style.

      .vjs-mouse.vjs-playlist {
        color: #466675; /* darker blue */
        background-color: #CCCCCC; /* gray */
      }

The playlist below uses the following CSS to style the playlist as in the previous example, but now a new style is added to also change the font color of the descriptions.

      .vjs-mouse.vjs-playlist {
        color: #466675; /* darker blue */
        background-color: #CCCCCC; /* gray */
      }
      .vjs-mouse.vjs-playlist .vjs-playlist-description {
        color: #466675; /* darker blue */
      }

The playlist below uses the following CSS to change the background color of the playlist itself, then uses another color to change the background of the list items.

      .vjs-mouse.vjs-playlist {
        background-color: #466675; /* darker blue */
      }
      li.vjs-playlist-item {
        background-color: #6A8D9D; /* lighter blue */
      }

The playlist below uses the following CSS to style the playlist as in the previous example, but now a new style is added to change the background color of the selected item.

      .vjs-mouse.vjs-playlist {
        background-color: #466675; /* darker blue */
      }
      li.vjs-playlist-item {
        background-color: #6A8D9D; /* lighter blue */
      }
      li.vjs-playlist-item.vjs-selected {
        background-color: #FAC8BF; /* pink */
      }
It has been reported that at times the selector shown above will not change the style of the selected video. You can also try this selector:

.vjs-playlist .vjs-selected, .vjs-playlist.vjs-mouse .vjs-selected

The playlist below uses the following CSS to change the color of the font for the video time durations.

      time.vjs-playlist-duration {
        color: #94FFFC; /* turquoise */
      }

The playlist below uses the following CSS to alter the margins around the playlist and list items. Note that in the original styles the bottom margin is not set for the playlist, but in the list item. The new styles reflect the same approach.

      .vjs-mouse.vjs-playlist {
        background-color: #466675; /* darker blue */
        padding: 5px 5px 0 5px;
      }
      .vjs-mouse.vjs-playlist .vjs-playlist-item {
        background-color: #6A8D9D; /* lighter blue */
        margin-bottom: 5px;
      }

Moving the playlist

By default the playlist displays to the right of the video, displaying the videos vertically. This can be changed using CSS, and is described in this section.

Note: This section of the document applies ONLY to in-page embed implementations of the player and playlist.

Vertical playlist below player

The following screenshot shows the playlist positioned below the player, and the videos displayed in their normal way, vertically.

  • Lines: 574-585: A <div> that surrounds both the player and playlist. A myPlayer class is added to the element.
  • Lines: 581-583: A <div> that surrounds both the playlist. A playlist-wrapper class is added to the element.

Playlist API

The Playlist API gives you programmatic control over the usage of playlists, for instance next(), previous() and autoadvance() methods. The Guide: Playlist APIdocument provides full details.
