Example

The playlist example below shows the enhancements from the Playlist UI plugin versus the legacy playlist UI, which include:

  • Overall look and feel improvements
  • Larger thumbnails
  • Support for display of longer video titles
  • Better responsive behavior
  • When using the Standard (iframe) player implementation, ability to hide and show the playlist (mouse over the player below to see that option)
When using the Standard (iframe) player implementation, the playlist has a show/hide button (as just mentioned) and is automatically sized and positioned. When using the Advanced (in-page embed) implementation, you insert the playlist as an HTML <ol> element and write your own CSS to position and size the playlist. This gives you full control over the playlist's display. In the latter case, the playlist does not even need to be adjacent to the player if you don't want it to be. The In-page embed player section of the Implementing Playlists Programmatically document has substantial content on formatting and styling the playlist.

Implement using Players module

When creating a player in the Players module, edit the SETTINGS and enable playlists. For the Playlist version the default, V2 (Default) enables the playlist UI plugin. The highlighted version in the screenshot shows the playlist options.

Note: If you select the Responsize sizing option, the Standard (iframe) code works well and both the player and playlist are responsive. If you plan to use the Standard (in-page embed) code, the player will be responsive, but the playlist will not.

Using Standard code

For best results, when you publish a playlist with a Playlist UI plugin enabled player, and you are going to use the Standard (iframe) player implementation, choose to have a responsive player and enter the player size. These selections will provide an player/playlist that is responsive.

The resulting HTML will appear similar to the following:

  <div style="display: block; position: relative; max-width: 750px;">
    <div style="padding-top: 56.25%;">
      <iframe src="//players.brightcove.net/1507807800001/BkAAGGDn_default/index.html?playlistId=4400450992001"
        allowfullscreen
        webkitallowfullscreen
        mozallowfullscreen
        style="width: 100%; height: 100%; position: absolute; top: 0px; bottom: 0px; right: 0px; left: 0px;"></iframe>
    </div>
  </div>

Using Advanced code

If you wish to use the Advanced (in-page) implementation of the player, you create the player the same way as described above. After you copy the code, you will need to add some CSS and HTML around it, as detailed here:

  • Lines 8-12: Style the player
  • Lines 13-16: Style the playlist (for best results make the height of the playlist 15px greater than the height of the player)
  • Line 22-28: Code copied for the Advanced player implementation
  • Line 30: The <div> tag where the playlist will be inserted
  • Line 32: The player's <script>, part of the copied code
<!doctype html>
<html>

<head>
  <meta charset="UTF-8">
  <title>Using the Playlist UI Plugin</title>
  <style type="text/css">
    .video-js {
      height: 350px;
      width: 640px;
      float: left;
    }
    .vjs-playlist {
      width: 280px;
      height: 365px;
    }
  </style>
</head>

<body>

  <video data-playlist-id="4400450992001"
    data-account="1507807800001"
    data-player="BkAAGGDn"
    data-embed="default"
    data-application-id
    class="video-js"
    controls></video>

  <div class="vjs-playlist"></div>

  <script src="//players.brightcove.net/1507807800001/BkAAGGDn_default/index.min.js"></script>

</body>

</html>

Separation between player and playlist

The CSS shown in the above code has the player and playlist with no separation between them, as show here:

If you wish to have separation between the player and playlist, as shown here, you can use a margin.

To create separation, simply use a margin-right style on the player:

.video-js {
  height: 350px;
  width: 640px;
  float: left;
  margin-right: 5px;
}

Implement using code

To implement a plugin, the player needs to know the location of the plugin code, a stylesheet if needed, the plugin name and plugin configuration options. The location of the plugin code and stylesheet are as follows:

//players.brightcove.net/videojs-bc-playlist-ui/2/videojs-bc-playlist-ui.min.js
//players.brightcove.net/videojs-bc-playlist-ui/2/videojs-bc-playlist-ui.css

The name of the plugin is bcPlaylistUi, and the only property is playOnSelect (detailed later in the document).

The following shows using the Advanced (in-page) implementation of the player to associate the Playlist UI plugin with a single instance of a player.

  • Line 8: Link to the plugin's stylesheet
  • Lines 11-15: Style the player
  • Lines 16-19: Style the playlist (for best results make the height of the playlist 15px greater than the height of the player)
  • Lines 25-32: Code copied for the Advanced player implementation
  • Line 34: The <div> tag where the playlist will be inserted
  • Line 36: The player's <script>, part of the copied code
  • Line 37: The plugin's <script>
  • Lines 40,45: Standard code to wait for the player to be ready for interaction
  • Line 41: Assign the player instance to a variable
  • Lines 42-44: Call the plugin and set the value for the property
  <!doctype html>
  <html>

  <head>
    <meta charset="UTF-8">
    <title>Using the Playlist UI Plugin - Code Implementation</title>

    <link rel="stylesheet" type="text/css" href="//players.brightcove.net/videojs-bc-playlist-ui/2/videojs-bc-playlist-ui.css">

    <style type="text/css">
      .video-js {
        height: 350px;
        width: 640px;
        float: left;
      }
      .vjs-playlist {
        width: 280px;
        height: 365px;
      }
    </style>
  </head>

  <body>

    <video id="myPlayerID"
      data-playlist-id="4400450992001"
      data-account="1507807800001"
      data-player="HkXKkOSO3"
      data-embed="default"
      data-application-id
      class="video-js"
      controls></video>

    <div class="vjs-playlist"></div>

    <script src="//players.brightcove.net/1507807800001/HkXKkOSO3_default/index.min.js"></script>
    <script src="//players.brightcove.net/videojs-bc-playlist-ui/2/videojs-bc-playlist-ui.min.js"></script>

    <script type="text/javascript">
      videojs('myPlayerID').ready(function(){
        var myPlayer = this;
        myPlayer.bcPlaylistUi({
          "playOnSelect": true
        });
      });
    </script>

  </body>

  </html>

Options

You may pass in an options object to the plugin upon initialization. This object may contain any of the following options:

The next section details how to use the options using both code and Studio.

autoadvance

  • autoadvance:
    • Type: number
    • Default: undefined
    • Determines if, and for now long, a pause between videos in a playlist occurs. For complete details see the autoadvance section in the Playlist API Guide.

hideOnStart

The hideOnStart property Determines whether the playlist is initially hidden from view.

  • hideOnStart:
    • Type: boolean
    • Default: false
    • This is only functional when using the iframe player implementation, which is logical since the show/hide playlist functionality is only available in the iframe. This option does NOT work with a horizontal playlist.

horizontal

  • horizontal:
    • Type: boolean
    • Default: false
    • Displays the playlist horizontally below the player instead of vertically.

nextButton

  • optionname:
    • Type: boolean
    • Default: false
    • .

nextEndscreen

  • optionname:
    • Type: boolean
    • Default: false
    • .

nextOverlay

  • optionname:
    • Type: boolean
    • Default: false
    • .

playlist

  • optionname:
    • Type: boolean
    • Default: false
    • .

playOnSelect

The playOnSelect property is used to control if the video should begin playing when the user clicks a video in the playlist.

  • playOnSelect:
    • Type: boolean
    • Default: false
    • When playOnSelect is set to true, selecting a new video from the playlist will start playing that video, even if the player was previously paused. By default, clicking a new video from the playlist will load the new video but keep the player paused if it was paused beforehand.

repeat

  • optionname:
    • Type: boolean
    • Default: false
