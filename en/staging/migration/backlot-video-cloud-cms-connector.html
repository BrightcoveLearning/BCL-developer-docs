Backlot / Video Cloud CMS Connector

The Backlot CMS API Adapter provides support for a select set of the Backlot API operations in order to help you keep integrations and apps working while you transition to the Video Cloud APIs. To learn more about the Video Cloud platform APIs, see Getting Started with the Brightcove APIs.

See other Ooyala to Brightcove Migration Guides.

Asset Endpoints

Note: this topic applies only to Ooyala customers migrated to Video Cloud.

The table below shows what Backlot API asset operations are supported.

Asset Endpoints
Backlot API Endpoint Method Supported? Notes
/v2/assets GET
  • ad type videos will be regular videos in Video Cloud with the tag ooyala_my_ads
  • youtube type videos will not be migrated to Video Cloud
  • external_id is not being migrated to Video Cloud
  • Recurring time restrictions not being migrated to Video Cloud
POST
  • Since ad and youtube type videos are not supported in Video Cloud, attempting to create videos of these types will return an error
  • youtube type videos will not be migrated to Video Cloud
/v2/assets/{asset_id} GET
  • ad type videos will be regular videos in Video Cloud with the tag ooyala_my_ads
  • youtube type videos will not be migrated to Video Cloud
POST
  • Only updating properties via PATCH
PATCH
  • Only properties that are migrated to Video Cloud can be updated see Backlot and Video Cloud: Video and Playlist Fields
DELETE
/v2/assets/{asset_id}/ad_set GET
  • Ad sets will be migrated to Video Cloud as cuepoints
/v2/assets/{asset_id}/closed_captions GET
  • This request will return the default WebVTT content
PUT
  • Replace the existing WebVTT text tracks; note that you can upload WebVTT or DFXP captions - DFXP will be converted to WebVTT
DELETE
/v2/assets/{asset_id}/drm_attributes/{drm_type} GET
  • DRM in Video Cloud is applied to all videos in a DRM-enabled account unless they are flagged not to be packaged with a drm_disabled field
PATCH
  • DRM in Video Cloud is applied to all videos in a DRM-enabled account unless they are flagged not to be packaged with a drm_disabled field
/v2/assets/{asset_id}/generated_preview_images GET
  • Video Cloud does not store multiple thumbnail images
/v2/assets/{asset_id}/labels GET
  • All labels will be migrated to tags, so this request will return the video tags
POST
  • All labels will be migrated to tags, so this request will add a video tag
/v2/assets/{asset_id}/labels/{label_id} PUT
  • Video Cloud does not have the concept of a label id
DELETE
  • Video Cloud does not have the concept of a label id
/v2/assets/{asset_id}/metadata GET
  • metadata items will be migrated to tags in the form meta:;:{key}:;:{value}
PUT
  • metadata items will be migrated to tags in the form meta:;:{key}:;:{value} - this request will add or replace tags in that format
PATCH
  • metadata items will be migrated to tags in the form meta:;:{key}:;:{value} - this request will modify tags in that format
DELETE
  • metadata items will be migrated to tags in the form meta:;:{key}:;:{value} - this request will delete all tags in that format
/v2/assets/{asset_id}/movie_urls GET
  • movie_urls are sources in Video Cloud
POST
  • Works the same way in Video Cloud - this request will add a remote rendition (and therefore source) to the video
/v2/assets/{asset_id}/movie_urls/{type} DELETE
  • This request delete only remote renditions of the specified type
/v2/assets/{asset_id}/player GET
  • This request will return the default Ooyala player construct, because Video Cloud does not associate players with videos; instead, any video can be associated with any player instance by its video id. You can get Video Cloud players via the Player Management API
/v2/assets/{asset_id}/player/{player_id} GET
  • Video Cloud does not associate players with videos; instead, any video can be associated with any player instance by its video id. You can get Video Cloud players via the Player Management API
/v2/assets/{asset_id}/preview_image_urls GET
  • Video Cloud does not store multiple thumbnail images, so this request will return the poster image
POST
  • Video Cloud does not store multiple thumbnail images, so this request will set the poster image using the first image in the list
PUT
  • Video Cloud does not store multiple thumbnail images, so this request will set the poster image using the first image in the list
DELETE
  • Video Cloud stores only a single poster image, and you can replace, but not delete it
/v2/assets/{asset_id}/preview_image_files/{image_id} DELETE
  • Video Cloud stores only a single poster image, and you can replace, but not delete it
/v2/assets/{asset_id}/preview_image_files GET
  • Video Cloud does not store multiple thumbnail images, so this request will return the poster image URL
POST
  • Video Cloud does not store multiple thumbnail images, so this request will set the poster image using the first image URL in the list as a remote asset
PUT
  • Video Cloud does not store multiple thumbnail images, so this request will set the poster image using the first image in the list URL in the list as a remote asset
DELETE
  • Video Cloud stores only a single poster image, and you can replace, but not delete it
/v2/assets/{asset_id}/primary_preview_image GET
  • Video Cloud stores only a single poster image, so this request will return that
/v2/assets/{asset_id}/publishing_rule GET /td>
  • Only geo-restrictions and scheduling will be returned, because Video Cloud sets other playback restrictions on players rather than videos
/v2/assets/{asset_id}/publishing_rule/{rule_id} PUT
  • Video Cloud does not have the concept of a publishing rule. Only geo-restrictions and scheduling properties for the video can be updated
/v2/assets/{asset_id}/source_file_info GET
/v2/assets/{asset_id}/streams GET
  • Will return Video Cloud sources
/v2/assets/{asset_id}/uploading_urls GET
  • Supported through the Ingestion Adapter
/v2/assets/{asset_id}/upload_status PUT
  • Supported through the Ingestion Adapter
/v2/assets/{asset_id}/youtube GET
  • Not supported in Video Cloud - see Brightcove Social for Video Cloud alternatives
PUT
  • Not supported in Video Cloud - see Brightcove Social for Video Cloud alternatives
PATCH
  • Not supported in Video Cloud - see Brightcove Social for Video Cloud alternatives
/v2/assets/{asset_id}/lineup GET
PUT
/v2/assets/lineup/{asset_id} PUT
DELETE
/v2/assets/{asset_id}/closed_captions GET
  • Will return the default text_track for the video
PUT
  • Will replace the default text_track for the video - ingested captions can be in WebVTT or DFXP format; DFXP will be converted to WebVTT
DELETE
  • Will remove the default text_track for the video
