Page level parameters

The page level parameters are used to configure the Ooyala player. They are part of the embed code, assigned to a configuration object named playeParam, then that configuration object is passed to the create() method. An example is shown here:

<div id="container" style="width:640px; height:360px;"></div>
<script>
    var playerParam = {
        "pcode": "YOUR_PCODE",
        "playerBrandingId": "YOUR_PLAYER_ID",
        "adManagerLoadTimeout": 5,
        "autoplay": true,
        "initialPlaybackSpeed": .5,
        "skin": {
            // Config contains the configuration setting for player skin. Change to your local config when necessary.
        }
    };
    OO.ready(function() {
        window.pp = OO.Player.create("container", "YOUR_ASSET_ID", playerParam);
    });
</script>
Ooyala Parameter Brightcove Equivalent Notes (if applicable)
pcode - The pcode is the account identifier data-account Brightcove Player attribute
playerBrandingId - The player branding ID is a reference to the player; if you do not include your player branding ID, the player will not load data-player Brightcove Player attribute
adManagerLoadTimeout - Specify the timeout (in seconds) for the loading of the Ad Manager module. Timeouts can be set for both the FreeWheel and IMA3 advertising plugins
autoplay autoplay Set as a <video-js ...> attribute or player configuration property
autoPlayUpNextVideosOnly - Enables (default) or disables the automatic playing of Up Next or Discovery videos after the main video has played Although no direct equivalent, setting a playlist for continuous play is similar
dynamicFilters - filters are server-side rules that are most commonly used to specify a subset of audio and video renditions to provide the best viewing experience on a given device; you can set up a filter to request that when an asset is transcoded, only HD and SD renditions are created Part of Video Cloud encoding
encodeingPriority - define the video encoding priority in a series of encodings separated by commas Configured by Brightcove Player plugins used (HLS, DRM, etc.) The HLS plugin is loaded by default
initialBitrate defaultResolution as part of the Quality Selection Plugin See Quality Selection Plugin document for details
initialPlaybackSpeed playback_rates Brightcove Player configuration property See Player Configuration Guide document for details
initialTime currentTime() method from the Brightcove Player API
initialVolume volume() method from the Brightcove Player API
iosPlayMode - Use this parameter to specify the initial playback mode for Safari Mobile on iOS devices (iOS 10 and later) playsinline See iOS and Brightcove Player document for details
location - Specify individual, custom paths for Bitmovin HTML5, CSS, and Flash files if any of those resources are used N/A
locationBaseUrl - Specify a custom path for the resources (Bitmovin HTML5, CSS, and Flash files) associated with the bit_wrapper.min.js plugin N/A N/A
loop loop Set as a <video-js ...> attribute or player configuration property
muteFirstPlay - Use this parameter to control audio when a video starts muted() method from the Brightcove Player API
onCreate - Use this parameter to listen to an event message and perform an action; this parameter enables you to subscribe to event messages from the message bus before the player is fully created videojs.getPlayer() method from the Brightcove Player API
platform - This parameter applies only to the bit_wrapper.min.js plugin for DASH and HLS N/A N/A
playerControlsOverAds - Use this parameter to enable (true) the Ooyala Player control bar during ads N/A The normal Brightcove Player controls used during ads except on iOS when the native controls are used
playlistsPlugin - Use this parameter to hide the Playlist plugin but still have it drive the autoplay a sequence of videos This behavior can be replicated by using a playlist player and selecting the Hidden playlist type in PLAYERS -> Styling -> Playlist Type
preload - Use this parameter to specify whether to start loading the video after the player is loaded and before the user starts playback This behavior can be replicated by selecting the desired option from PLAYERS -> Playback -> Preload Video Before Playback
preloadAds This behavior can be replicated by selecting the desired option from PLAYERS -> Advertising -> Request Ads
