Creating a Live Stream using Client SDK for Live SSAI (SLS)

This is a temporary STAGING document for review only. For the official Brightcove documentation site, go to support.brightcove.com
In this topic, you will learn how to create a live stream that includes client-side interactivity and Server-Side Ad Insertion.

Introduction

Brightcove's Client SDK with Live SSAI (SLS) allows client-side interactivity with live streams, including ad countdown timers and click through ads.

Your ad information is specified using an ad configuration which retrieves VAST responses to identify what ads to display. The VAST file also contains information about companion ads and beacon URLs.

If you are not familiar with SLS, see the Overview: Client SDK with Live SSAI (SLS) document.

Workflow

To use Brightcove's Client SDK with Live SSAI (SLS), follow these steps:

  1. Ask your account manager to enable your account for Live, SSAI and SLS
  2. Create an ad configuration
  3. Optional: Create slate assets to fill unused ad time
  4. Create a live job
  5. Create a remote video
  6. Configure the player
  7. Begin streaming
  8. Trigger ads using cue points (this can be done before or during the live event)
  9. Cleanup

Create an ad configuration

An ad configuration defines the information needed to include server-side ads with a live stream. This includes ad provider URL, expected ad response type and whether beacons should be sent.

For SLS, Ad configurations MUST have an ad_configuration_expected_response_type value of Vast.

To create a new ad configuration, send a POST request as follows:

Method POST
URL https://api.bcovlive.io/v1/ssai/applications
Header X-API-KEY: your API KEY

All requests are authenticated using an API key that is provided to you when your account is set up. The key is passed in the X-API-KEY header.

For details about the request body fields, see the Live API Reference document.

Sample request body: 

{
  "application_ad_configuration": {
	"ad_configuration_description": "$YOUR_DESCRIPTION",
	"ad_configuration_expected_response_type": "Vast",
	"ad_configuration_headers_for_impressions": false,
	"ad_configuration_strategy": "SingleAdResponse",
	"ad_configuration_transforms": [],
	"ad_configuration_url_format": "$YOUR_AD_SERVER"
  },
  "application_description": "$YOUR_DESCRIPTION",
}

Player management API

When configuring players with the Player Management API, you can set the delivery_config_id configuration property as follows:

Standard embed code

With the standard embed code (iframe), use the deliveryConfigId query parameter:

<iframe src=”https://players.brightcove.net/.../index.html?deliveryConfigId=abcdef”>

This will override any Delivery Rules set in the player configuration.

Advanced embed code

With the advanced embed code (in-page embed), use the data-delivery-config-id attribute:

<video-js data-delivery-config-id=”abcdef”>

This will override any Delivery Rules set in the player configuration.
