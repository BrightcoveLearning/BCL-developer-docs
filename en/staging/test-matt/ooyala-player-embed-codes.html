Ooyala to Video Cloud Migration: Player Embed Codes

Introduction

This document provides a basis to work from when you are working from an Ooyala Player embed code and creating an equivalent Brightcove Player embed code. Of course, with the configurations/customizations of both players be so very large this document cannot be exhaustive, but provides an excellent starting point.

Embed code architectures

The Ooyala Player and Brightcove Player take different approaches in how the player embed code functions. The following highlight the approaches:

Ooyala Player

  • Common embed code: 
    <script src="//player.ooyala.com/core/player_branding_id"></script>
<script>
var player;
var playerParams = {};

OO.ready(function() {
  player = OO.Player.create('oo-container', 'embed_code', playerParams);
});
</script>
  • The player code pieces are gathered on page load (essentially at "runtime") and then the player is built.
  • A script is included that provides basic player functionality, and also provides a way to include plugins. Details are provided in the Ooyala player plugins section below.
  • The playerParams variable is created to hold page level parameters that provide player behavior customization. Details are provided in the Page level parameters section below.
  • A JSON file provides the means for Ooyala Player visual customization. Details are provided in the Player customization section below.
  • You can also use an iframe implementation for the player, as shown here: 
    <iframe width="480" height="320" src="url_where_hosted/iframe.html?
  ec=YOUR_ASSET_ID
  &pbid=YOUR_PLAYER_ID
  &pcode=YOUR_PCODE
  &optional_parameter_1=value
  &optional_parameter_2=value
  &optional_parameter_N=value"
  frameborder="0" allowfullscreen>
</iframe>

    The page level parameters can be used in the iframe implementation also. For details see the Embedding Player V4 in an HTML iframe document.

Brightcove Player

  • Common embed code (also called Advanced or in-page): 
    <video data-video-id="5796745147001"
  data-account="1507807800001"
  data-player="default"
  data-embed="default"
  data-application-id
  class="video-js"
  controls
  width="640"
  height="360"></video>
<script src="//players.brightcove.net/1507807800001/default_default/index.min.js"></script>
  • The player code pieces are gathered on player creation and configuration (essentially at "compile time") and then the player is built on page load.
  • Some player configuration is available on player build using player attributes, as shown here: Available Options for the Advanced (in-page) Embed Code.
  • A script is included that provides complete player functionality. The script includes the player essentials, as well as any configuration done, including plugins and customization.
  • You can also use an iframe (also called Standard) implementation for the player, as shown here: 
    <iframe src="//players.brightcove.net/1507807800001/default_default/index.html?videoId=5796745147001"
  allowfullscreen
  webkitallowfullscreen
  mozallowfullscreen
  allow="encrypted-media"
  width="640"
  height="360"></iframe>

    Player attributes are available for the iframe implementation. For details see the Available Options for the Standard (iframe) Embed Code document.

Page level parameters

The page level parameters are used to configure the Ooyala player. They are part of the embed code, assigned to a configuration object named playeParam, then that configuration object is passed to the create() method. An example is shown here:

<div id="container" style="width:640px; height:360px;"></div>
<script>
    var playerParam = {
        "pcode": "YOUR_PCODE",
        "playerBrandingId": "YOUR_PLAYER_ID",
        "adManagerLoadTimeout": 5,
        "autoplay": true,
        "initialPlaybackSpeed": .5,
        "skin": {
            // Config contains the configuration setting for player skin. Change to your local config when necessary.
        }
    };
    OO.ready(function() {
        window.pp = OO.Player.create("container", "YOUR_ASSET_ID", playerParam);
    });
</script>

Player customization

Brightcove Player and Ooyala Player take very different approaches to player customization. Ooyala Player uses a large (2668 lines) JSON file that contains basically all the customizations you could want to do to the player. Brightcove Player's basic approach to player customization is more CSS focused. That being said, some of the Ooyala Player's JSON goes beyond what Brightcove Player does in CSS. The following table shows a few examples of how the Ooyala JSON envelopes many customizations done if different parts of the Brightcove Player ecosystem.

Customization Ooyala Location Brightcove Player Approach
Responsive Design JSON Studio settings / Player configuration
End Screens JSON Endscreen plugin configuration
Player Controlbar/Buttons JSON CSS
Captions JSON Studio settings / Player configuration / APIs
