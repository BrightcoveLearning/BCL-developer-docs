Product Use Cases: Good and Bad

Audience

Proper Use Case(s)
  • Track video engagement data for videos that are viewed inside of an Audience-enabled Brightcove Player. Viewing data is stored in Video Cloud and can optionally be synchronized to popular marketing automation platforms such as Eloqua, Marketo, HubSpot and Salesforce.
  • Build a custom lead form which allows you to accomplish advanced marketing functions such as lead routing and progressive profiling directly in the player.
Poor Use Case(s)
  • Track video engagement for such a high number of viewers that your allowed API calls to your marketing automation platform exceeds your daily limit.
Restrictions
  • Marketing automation platforms limit the number of API calls. For instance, Marketo quotes these limits in the Marketo Integration Best Practices document:
    • Daily Quota: Most subscriptions are allocated 50,000 API calls per day (which resets daily at 12:00AM CST). You can increase your daily quota through your account manager.
    • Rate Limit: API access per instance limited to 100 calls per 20 seconds.
    • Concurrency Limit: Maximum of 10 concurrent API calls.
  • Brightcove Audience will pass through the calls received from the players, but it cannot effect the limits of the platform's API calls.
You can set Brightcove Player to report video usage directly from the client to the marketing automation platform. This will allow limited engagement information in super high-volume traffic. Marketo and HubSpot support this implementation.

Playback API

Proper Use Case(s)
  • Fetch video and playlist data from Video Cloud for the client in a low-latency environment. It is typically used to get video data to a player on a web page or in a mobile app.
  • Search videos and page through results, sorting the results if desired.
  • Retrieve videos or playlists to display information about them, without having a player on the screen, as the landing page for a video portal.
Poor Use Case(s)
  • Retrieve information about the video currently in the player. (A best practice is to use the mediainfo object which contains information for the video currently in the web player.)
  • Retrieve information expecting it to be real-time.
Restrictions Playback API search is capped at 1000 videos.

CMS API

Proper Use Case(s)
  • Retrieve information for time-sensitive publishing workflows to make changes to videos and playlists.
  • Fully manage your library of videos and assets using write access.
Poor Use Case(s)
Restrictions To ensure the performance of the Video Cloud system, no more than 20 concurrent calls to the CMS API are allowed per account. Access frequency should be less than 10 requests per second.
