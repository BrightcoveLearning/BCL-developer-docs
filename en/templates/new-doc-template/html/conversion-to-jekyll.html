Doc Conversion Guide - Moving to Jekyll

Guide to converting docs for use in Jekyll.

Following the steps, there is a style reference showing examples of the new styles.

Steps

File naming

  1. File names should be from the Drupal URL.

Wrap the main content in an <article class="bcls-article"> tag

  1. The main contents of the page should all be included in <article> tag.

Convert section divs to section tags

  1. For each <div> tag with a section class, change the tag from <div> to <section> (remember to change the end tag as well as the start tag!).
  2. Also change the class name section to bcls-section. (You can use find/replace-all for that.) The template look is: 
    <section class="bcls-section">
  <h2 id="section-name">section-title</h2>
  <p></p>
</section>
  3. For each section you should have an h2 tag in the format: <h2 id="section-name">section-title</h2> The title is not title case, only the first letter is capitalized, for example How are the device apps updated?. The section-name is the title with underscores for spaces, for example How_are_the_device_apps_updated (any punctuation removed).
  4. If the section has the additional class video-cloud-only or player-only, leave that class as is.

Fix the asides

  1. Do a search on class="BCL-aside" to find all the asides.
  2. If the element is a <p> or <div>, change it to an <aside> tag and change the class to one of the 3 new aside types. Note that the new asides include an icon, so beginning asides with "Note:" is redundant now.
  3. If the BCL-aside class is applied to a <li> or <td> tag, remove the class from the tag and wrap the contents inside one of the 3 new aside types.

Fix code blocks

  1. See the Dash snippets for code blocks above - they'll be helpful in the following steps.
  2. <pre><code> tags should be changed to: <pre class="line-numbers"><code class="language-XXXX">, where XXXX is the language code (for HTML that includes CSS and/or JavaScript code, you can just use language-html). See codes for all supported languages here: http://prismjs.com/#languages-list. The ones you will use most often are:
    • html
    • css
    • javascript
    • http
    • json
    • swift
    • objectivec
    • java
    • bash
    • ruby
    • python

    There are Dash snippets for each of those.

    For a code block without line numbers, just omit the class="line-numbers" attribute. For no syntax highlighting, just omit the class="language-XXXX" attribute. Note the difference between this code with a language class and this code with no language class and decide which is better for the context.
  3. You can also use the class="language-XXXX" attribute on inline <code>

Fix images

All images and other external assets should be moved to the doc-assets folder on S3 via the BCL-Doc-Assets repo!
  1. Move images to local file structure for your site.
  2. Find all <img> tags in the doc.
  3. For inline images, do nothing.
  4. All block images need to be wrapped in a <figure class="bcls-figure"> tag.
    If you are using the group Dash snippets, type bcimg to get a figure block - you will be promted to enter a description that will be used for the alt value as well as a numbered image caption. (Or type bcimgnoborder to get the same thing without the image border.)
  5. While you are there, if the image is a screenshot, check to make sure it's up to date.

Convert Task Steps

  1. If there are task steps in the document, give the <ol> tags for the steps lists the class bcls-tasklist and remove any old start attributes.
  2. If you want to restart numbering for the list use: <ol class="bcls-tasklist-restart"> otherwise omit the -restart.

Identify non-standard classes

  1. Search the doc for class=.
  2. The only classes that should not start with bcls- are:
    • video-cloud-only
    • player-only
    • Prism classes: line-numbers and language=xxxx
    • classes for a sample in the page (requires a stylesheet for page -- but please let Robert know where you think this is necessary, so that Robert can look and decide if the styles should be added to the site stylesheet)
  3. If you see any other classes that don't begin with bcls-, send Robert the branch name so I can take a look.

Fix expander sections

(applies only to some pages, mostly player samples)

  1. Find the <h2 id="expanderHead"> tag(s) and change id="expanderHead" to class="bcls-expander-head"
  2. In the following <div id="expanderContent" style="display:none"> tag, change id="expanderContent" to class="bcls-expander-content" and remove the style="display:none" attribute.
