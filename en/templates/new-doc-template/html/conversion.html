Doc Conversion Guide

Guide to converting docs to the new template. The general aim is to have the docs work on the current docs.brightcove.com site, but have all the content that needs to migrate to the new site wrapped in an <article> tag.

Following the steps, there is a style reference showing examples of the new styles.

If you are using the team Dash snippets, see helpful new snippets under the template tag. Here is a reference for the available Dash snippets showing the code they produce,

Steps

Preparation

  1. Sync BCL-developer-docs to make sure it's up to date.
  2. Create a new branch for the doc you are going to convert so that you can fully test it locally before adding it to the Master branch (Bob, I know you don't work in Git that much, so get Matt or Jeff to show you what to do this if needed.)
  3. Open the doc you're going to convert in your editor.

Wrap the main content in an <article class="bcls-article"> tag

  1. The main contents of the page should all be included in <div> tags that have the class section (there may be additional classes on some sections).
  2. Select all the section blocks except the related topics section (that one will be included automatically in the new site, so we don't want to import it). Most of the related topics blocks will have the id BCLReferences, but not all - it should always be the last section div in the page, however.
  3. Wrap all the content sections together in one <article class="bcls-article"></article> tag.

Make the first section a <header>

  1. The first <div> tag inside the article should have the attribute id="top".
  2. Change this <div> tag to a <header> tag (remember to change the end tag as well as the start tag!).
  3. Change the id from top to bcls_header
  4. Add the following attribute to the <header> tag: class="bcls-header"
The class name is hyphenated (bcls-header); the id is camelcase (bcls_header).

Convert section divs to section tags

  1. For each <div> tag with a section class, change the tag from <div> to <section> (remember to change the end tag as well as the start tag!).
  2. Also change the class name section to bcls-section. (You can use find/replace-all for that.)
  3. If the section has the additional class video-cloud-only or perform-only, leave that class as is.
  4. If there are additional classes besides video-cloud-only or perform-only, leave those class names as is, but send me the URL for the page so I can see if the additional class is needed and add it to the stylesheet.

Fix the asides

  1. Do a search on class="BCL-aside" to find all the asides.
  2. If the element is a <p> or <div>, change it to an <aside> tag and change the class to one of the 3 new aside types. Note that the new asides include an icon, so beginning asides with "Note:" is redundant now.
  3. If the BCL-aside class is applied to a <li> or <td> tag, remove the class from the tag and wrap the contents inside one of the 3 new aside types.

Fix code blocks

  1. See the Dash snippets for code blocks above - they'll be helpful in the following steps.
  2. <pre><code> tags should be changed to: <pre class="line-numbers"><code class="language-XXXX">, where XXXX is the language code (for HTML that includes CSS and/or JavaScript code, you can just use language-html). See codes for all supported languages here: http://prismjs.com/#languages-list. The ones you will use most often are:
    • html
    • css
    • javascript
    • http
    • json
    • swift
    • objectivec
    • java
    • bash
    • ruby
    • python

    There are Dash snippets for each of those.

    For a code block without line numbers, just omit the class="line-numbers" attribute. For no syntax highlighting, just omit the class="language-XXXX" attribute.
  3. You can also use the class="language-XXXX" attribute on inline <code>

Fix expander sections

(applies only to some pages, mostly player samples)

  1. Find the <h2 id="expanderHead"> tag(s) and change id="expanderHead" to class="bcls-expander-head"
  2. In the following <div id="expanderContent" style="display:none"> tag, change id="expanderContent" to class="bcls-expander-content" and remove the style="display:none" attribute.

Fix images

  1. Find all <img> tags in the doc.
  2. For inline images, do nothing.
  3. All block images need to be wrapped in a <figure class="bcls-figure"> tag.
    If you are using the group Dash snippets, type bcimg to get a figure block - you will be promted to enter a description that will be used for the alt value as well as a numbered image caption. (Or type bcimgnoborder to get the same thing without the image border.)
  4. While you are there, if the image is a screenshot, check to make sure it's up to date.

Convert Task Steps

  1. If there are task steps in the document, give the <ol> tags for the steps lists the class bcls-tasklist and remove any start attributes.

Identify non-standard classes

  1. Search the doc for class=.
  2. The only classes that should not start with bcls- are:
    • video-cloud-only
    • perform-only
    • Prism classes: line-numbers and language=xxxx
    • classes for a sample in the page (requires a stylesheet for page -- but please let me know where you think this is necessary, so that I can look and decide if the styles should be added to the site stylesheet)
  3. If you see any other classes that don't begin with bcls-, send me the branch name so I can take a look.
