Timezone API

API to query an account's timezone setting for a point in time as well as all current account timezone settings.

API endpoints

GET /accounts/all/timezone

Return a map of all accounts and their latest timezone setting. An optional day parameter can be provided to return account timezones for a particular day.

  • The day must be in yyyyMMdd format.

Response

{

  687140549: "America/New_York",

  687883034: "UTC",

  688539885: "America/Denver",

  711881421: "UTC",

  713285227: "America/Chicago",

  ...

}

GET /accounts/$account/timezone

Lookup an account's current timezone setting. An optional day parameter can be provided to return an account's timezone on that day.

  • The day must be in yyyyMMdd format.

Response

{

"account": "1836195192001",

"timezone": "America/Los_Angeles"

}

PUT /accounts/$account/timezone

Update an account's timezone setting.

  • Request body must be a valid JSON document

{timezone: "valid/timezone"}

Response

Status Code

Result

204 No Content

Update successful

404 Not Found

Account ID not found

400 Bad Request

Invalid JSON body

Invalid timezone provided
