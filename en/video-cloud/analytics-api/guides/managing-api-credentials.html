Analytics API: Managing API Credentials

In this topic, you will learn how to create and manage API authentication credentials for the Analytics API.

Introduction

When you use the Analytics API, your method calls will require access tokens.

In this topic, you will learn about:

  • Creating client credentials
  • Managing client credentials
  • Revoking client credentials

For a general understanding of Brightcove's OAuth Service, see the OAuth API Overview. For an example of how to use your client credentials to get an access token, see the OAuth Quick Start, which shows you how to build a proxy for your API calls.

In addition to using the UI described in this topic, you can all register a client app and get your client id and secret directly from the OAuth API. See:

  • Get Client Credentials Using CURL
  • Get Client Credentials Using Postman

Creating client credentials

The API Authentication page is used to create new client credentials. When creating credentials, you can select the service and the roles that will be enabled with the credentials.

To create new client credentials for your account:

  1. Login to Video Cloud Studio.
  2. Select API Authentication from the Admin menu:
  3. The API Authentication page will open displaying your current client credentials.
    Only the client credentials for the account you are logged into will display.
  4. Click Register New Application to display the dialog where you can register a new app:
    1. Enter a Name (optional but recommended)
    2. Enter a Short Description (optional but recommended)
    3. Select one or more accounts that the credentials will apply to
    4. Click >> to add all accounts, or > to add selected accounts
    5. Select the operations authorized by these credentials - for the Analytics API, you must include at least Analytics>Read and CMS>Video Read
  5. Click Save.
  6. A Client ID and Client secret will be returned.
    Copy the client secret and store it in a secure location as the client secret will not be displayed again.

Managing client credentials

You can return to the API Authentication page to view and manage client credentials. To view the registration details for any registered app, click the on its name to open an editing dialog. From here, you can:

  • Modify the name
  • Copy the client ID
  • Add and remove accounts for authorization
  • Select additional APIs to enable
  • Revoke the credential

Click Save to save your changes.

Revoking client credentials

To revoke a client registration, click the Revoke link next to the registration. Revoking a registration permanently disables it and cannot be undone.
