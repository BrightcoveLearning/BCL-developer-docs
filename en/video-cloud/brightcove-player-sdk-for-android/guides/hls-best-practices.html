Using the HLS Player

When wishing to play HLS video with the Brightcove Player SDK for Android there are four options when using the Brightcove HLS Player. The options are listed here in order of recommendation:

Gradle and .aar

The easiest solution is to use the .aar file with Gradle and Android Studio. This is the direction that Google is pushing the Android community, so Brightcove is following that recommendation.

The .aar file has many benefits, one of which is that it includes the compiled Java code, the dependent Java libraries and the dependent native libraries all in one file.

Brightcove supplies two versions of this file. The first is watermarked and hosted in Brightcove's public Maven repo at http://repo.brightcove.com/releases (URL not meant to be accessed via a browser). The second is non-watermarked and Brightcove recommends to install it into your local Maven repo. Brightcove customers can get the non-watermarked version by contacting support.

Legacy .jar & .apklib

The second best solution is to take the legacy .jar file and legacy .apklib files and install them into your in local Maven repo. This solution is great for doing automated builds with Maven, but it requires a separate solution for building with Eclipse. The difficulty with Eclipse is likely what pushed Google in the direction of Android Studio.

Eclipse & Android

The next best solution is create an Eclipse Android library project in your workspace. Then put the legacy .jar file into the library project's libs folder and unzip the .apklib file onto the library project. You then add the library project as a project dependency in your application project(s). This is slightly better than adding the legacy .jar file to the libs folder of your application and unzipping the .apklib file on top of your application. This approach is less messy when you want to apply an update. To apply an update you just delete the library project and recreate it with the new legacy .jar file and new legacy .apklib files.

.jar in libs folder

Last in the list of possible solutions is to put the legacy .jar file into your libs folder and unzip the legacy .apklib file on top of your Android application. This is a simple approach to perform once, but it becomes tricky when applying updates as you have to manually delete all the individual files from the previous version.
