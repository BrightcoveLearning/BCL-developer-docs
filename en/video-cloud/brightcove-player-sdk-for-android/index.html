Brightcove Player SDK for Android

Welcome to the Brightcove Player SDK for Android developer documentation. Here you will find information you need to get started developing with the SDK. The SDK provides tools to help you easily include Video Cloud videos in your Android applications.

Support

Support for the SDK is available via Brightcove Support and also a Google Group for the Player SDKs.

Supported versions

Support is defined in terms of the [major version].[minor version].[minor revision] of an OS (for example: Android 4.1.1). There are two levels of support

  • Active: Brightcove will test and fix bugs on these OS versions.
  • Passive: Brightcove will test on these OS versions, but bug fixes will be handled on a case by case basis.

Generally, Brightcove provides active support for the versions of the OS that are used by 80% of the user base, and passive support for other versions used by at least 5% of the user base. Only the latest minor revision of a version is supported.

Brightcove provides active support for the latest Android SDK on the following Android versions:

  • 4.4
  • 4.3
  • 4.2
  • 4.1
  • 4.0
  • 2.3

Core SDK Installation

Installation of the core SDK is very straightforward. You simply download the SDK and unzip it. As mentioned above, you will find the SDK archive file contains six other zip files. Unzip the android-sdk-4.x.x.zip file. From this archive add the android-sdk-4.x.x.jar file to your library.

Architectural overview

The Brightcove Player SDK for Android is built on top of existing Android tools to simplify the use of video on Android and ease the use of Video Cloud assets, like videos and playlists. The SDK is designed around an event driven architecture where components emit events to allow other components to listen and respond to state changes.

Architectural components

Below is a list of important architectural components to help you understand the SDK.

Brightcove Player

The class BrightcovePlayer is a basic Activity, which wraps a BrightcoveVideoView and sets up an EventLogger and default MediaController. It implements basic lifecycle management and full screen logic. It is meant to be subclassed. At a minimum the onCreate() should be overridden with logic to wire up the BrightcoveVideoView variable to the layout.xml. For example:

brightcoveVideoView = (BrightcoveVideoView) findViewById(R.id.bc_video_view);

Brightcove Video View

The class BrightcoveVideoView is an Android widget and a lower level alternative to the BrightcovePlayer. It is designed to be drop in replacement for Android's native VideoView. It provides public methods to mirror that of VideoView and Java collection-like API for managing the internal playlist. It is designed to use the Event workflow and will emit events related to video playback.

The BrightcoveVideoView is designed to work with Android's MediaController class to provide a controller for video playback that matches the device and OS version.

Events

Events in the Brightcove Player SDK for Android represent a state or action that can be passed down through a chain of listeners. All events have a type, and a map of properties associated with them. Events can also be stopped from propagating down the chain of listeners, or can be instructed to prevent their default listeners in the video playback classes. Events can represent state or an action occurring. For example, the event type DID_SEEK_TO represents the state of the action SEEK_TO.

EventEmitter

The EventEmitter is the public facing API for classes that wish to emit and listen to events. The event emitter can be obtained from the BrightcoveVideoView. It is also possible to create your own event emitter and instruct the BrightcoveVideoView to use it. This would be useful if you want control of the chain of listeners.

.

Components

Components in the Brightcove Player SDK for Android are objects that can use events and the event emitter system. Using a subclass of AbstractComponent will allow you to quickly create a component that listens for and emits events. As a component, you must describe in annotations which events the component will listen for and emit.

Catalog

The Catalog provides access to Video Cloud videos and playlists. This component's methods allow retrieval of videos and playlists by ID and/or ReferenceID. The catalog will make the URL requests and parse the returned data.

Analytics

If you use the Brightcove player and Catalog included in the SDK to retrieve and play videos, analytics data will be automatically collected and will appear in your Video Cloud Analytics. If you do not use the Brightcove player and catalog, you can still send analytics data to your Video Cloud Analytics using the Data Collection API.

Known issues

GEO restriction feature

The GEO restriction feature does not work with the Brightcove Player SDK for Android.

loadVideoBy* mixed results when in full screen playback

If you call videoPlayer.loadVideoBy*() while you are in full screen playback the current video stops but you stay in full screen. If you exit full screen you see the new video is cued. This can happen if these methods are called asynchronously.
