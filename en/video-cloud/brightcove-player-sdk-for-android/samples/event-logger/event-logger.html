Using the Event Logger

This topic describes how to use the event logger functionality to observe events on Brightcove components.

This document is based on working code. Code snippets appear in the document. You can view the complete MainActivity.java here. You can also download the entire ADT project for experimentation.

Create event loggers

One of the strengths of the Brightcove Player SDK for Android is the sophisticated event model. Being able to observe event information on Brightcove components makes application development more understandable and debugging easier. You can use the EventLogger class to observe event information.

The first step in making the event information observable is using code to create event loggers. For example, here is code creating an event logger on the BrightcoveVideoView and Catalog. Notice that you create an event emitter on both components before you create the event logger.

The class definition for EventLogger class can be found here.

Observe events

Once the event loggers are created you will want to leverage them to observe event information. To do this you use the LogCat view, which is included in ADT. To open LogCat select Window > Show View > Other and in the Android folder select LogCat and click OK. At this point, when you run your application from within ADT, you can look through the Tag column for the tags you used when creating your event logger objects. This can be a daunting task, as there may be many, many lines to look through. A better option is to create a filter as described next.

To make viewing of the events reported by the event logger objects you created easier, it is beneficial to create a filter. In the LogCat tab click the green plus sign (+). A dialog will pop up and fill it in appropriately. The only value you must match from your code is the by Log Tag option. This must match the tag entered (the third, String argument) when creating the objects.

When you run your application click on the value you supplied for the Filter Name.
