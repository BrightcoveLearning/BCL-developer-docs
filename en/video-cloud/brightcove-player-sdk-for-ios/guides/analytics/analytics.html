Using Brightcove Analytics

This topic describes how to set properties in Brightcove Analytics from your mobile app using the Brightcove SDK for iOS.

Analytics integration

If you use the Brightcove player and Catalog included in the SDK to retrieve and play videos, analytics data will be automatically collected and will appear in your Video Cloud Analytics. If you do not use the Brightcove player and catalog, you can still send analytics data to your Video Cloud Analytics using the Data Collection API.

Customize properties

From your iOS app, you can customize values sent to Brightcove Analytics for your account id, source URI and destination URI:


id<BCOVPlaybackController> controller = 
    [[BCOVPlayerSDKManager sharedManager] createPlaybackController];
		
controller.analytics.account = <insert-valid-accountId>;
controller.analytics.destination = <insert-example-of-valid-uri>;
controller.analytics.source = <insert-example-of-valid-uri>;

account

Use the account property to set the Account ID value in Brightcove Analytics. When data is sent to analytics, the value of this property will be used instead of the video's accountId value. The video's accountId value remains unchanged from when they were retrieved from the server.


controller.analytics.account = <insert-valid-accountId>;

destination

Use the destination property to set the URI that originates the event. The value of this field must be a URI with valid structure:


<scheme name> : <hierarchical part> [ ? <query> ] [ # <fragment> ]

For more details about the URI structure, see the URI Scheme Wikipedia page.

The value of the destination property:

  • The default value is bcsdk:/ followed by the app's bundle identifier as listed in the app's info.plist file.
  • A URI without a hierarchical part (e.g. just a scheme) is invalid.
  • A URI without a scheme name is invalid.

Example:


controller.analytics.destination = <insert-example-of-valid-uri>;

source

Use the source property to set the URI that sent the end-user to the destination URI. The value of this field must be a URI with valid structure:


<scheme name> : <hierarchical part> [ ? <query> ] [ # <fragment> ]

For more details about the URI structure, see the URI Scheme Wikipedia page.

The value of the source property:

  • The default value is nil. When the value is nil, the source property is not sent to the Analytics server.
  • A URI without a hierarchical part (e.g. just a scheme) is invalid.
  • A URI without a scheme name is invalid.

Example:


controller.analytics.source = <insert-example-of-valid-uri>;
