Simple Video Playback App

This tutorial will familiarize you with the basics of working with the Brightcove Player SDK for iOS by walking you through the development of a simple video playback app.

This document is based on a working sample with code snippets throughout. You can view the entire code for the view controller header and the view controller implementation. You can also download the entire Xcode project for experimentation.

Create the project

First get the SDK, set up a project in Xcode, and add the SDK to the project.

Get the SDK

  1. Download the latest version of the SDK and extract it to your local system.

Set up the Xcode project

Now start a new Xcode project for the app.

  1. Start the creation of a new iOS project in Xcode. Click Create a new Xcode project .

  2. Choose Single View Application for the template and click Next.

  3. Set the project information as follows:

    • Product Name: Simple Video Playback
    • Organization Name: YourCompanyName
    • Company Identifier: com.example-company
    • Class Prefix: SVP

  4. Choose the location where you want to save your project and create it.

  5. In the Build Settings section, remove the arm64 architecture from the projectâ€™s Architectures and Valid Architectures settings. For the Architectures setting, you may have to select the Other option and create your own architecture without arm64.

  6. Now close the project (yes, close it — this is important!)

Add the SDK and its dependencies to your project

  1. The easiest way to add the SDK and its dependencies to your project is to use CocoaPods, by including ReactiveCocoa. If you do not have CocoaPods installed, see the installation instructions on the CocoaPods site.

    NOTE: Using CocoaPods (the pod tool) on an existing Xcode project will modify the project file. You may want to make a backup before doing this.

  2. Create a plain text file called Podfile (no file extension) in your project folder.

  3. Using a text editor, add the following lines of code to the Podfile and save it. In this file, we do the following:

    • Define the iOS platform version.
    • Install the current version of the Brightcove player SDK.
    • Include post_install steps to remove the arm64 architecture.

      • NOTE: You only need the post_install steps for iOS apps which use the Widevine component, as this do not support the 64-bit architecture. By default, CocoaPods includes the arm64 architecture in its generated targets' build settings. This code in the Podfile reverts to the standard build architectures.

    platform :ios, '7.1' # (or whatever version you want to target)

pod 'Brightcove-Player-SDK', '4.1.7'

# Remove 64-bit build architecture from Pods targets
post_install do |installer|
  installer.project.targets.each do |target|
    target.build_configurations.each do |configuration|
      target.build_settings(configuration.name)['ARCHS'] = '$(ARCHS_STANDARD_32_BIT)'
    end
  end
end

    NOTE: ReactiveCocoa will be added to your project as a part of the Brightcove SDK.

  4. Open a Terminal session and navigate to your Simple Video Playback Xcode project folder.

  5. Type the command

    pod install

    and press Return to run the command. You should see a series of messages in the terminal indicating that the ReactiveCocoa library has been integrated.

    Note the last line, which is important — from this point on, you must open Simple Video Playback.xcworkspace in Xcode, not Simple Video Playback.xcodeproj
