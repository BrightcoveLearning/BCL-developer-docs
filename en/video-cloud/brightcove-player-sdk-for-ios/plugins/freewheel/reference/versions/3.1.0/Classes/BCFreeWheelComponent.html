Overview

The BCFreeWheelComponent manages communication between FreeWheel’s iOS SDK and the Brightcove iOS SDK. Like all components, it inherits its initialization and setup behavior from BCComponent.

Allowed to Emit

This component may emit the following events:

  • BCEventCuePoint – to know when a cue point has been triggered
  • BCEventDidSetVideo – to know when the video has changed
  • BCEventVideoDidEnd – to know when the video has ended
  • BCEventPlay – to know when playback has been requested

Allowed Listeners

This component may listen for the following events:

  • FW_AD_CONTEXT_CREATED – to report that an AdContext is available
  • FW_AD_REQUEST_COMPLETE – to report that the FreeWheel ad request has finished
  • FW_AD_REQUEST_FAILURE – to report that the FreeWheel ad request failed
  • BCEventShowControls – to request UI controls to be made visible
  • BCEventHideControls – to request UI controls to be hidden
  • BCEventPause – to pause the video so an ad can be shown
  • BCEventSetCuePoint – to specify cue points for ads
  • BCEventPlay – to request playback to be resumed

onCuePoint:

Callback for BCEventCuePoint. Determine type of cue point and play appropriate ad type. Although a cue point may be associated with a specific time, it will actually be triggered at the nearest possible moment on or after that time. So a cue point event lists both the start and end times inside of which the cue points were triggered.

- (void)onCuePoint:(BCEvent *)event

Parameters

event

The BCEvent to be handled, should have type BCEventCuePoint. A details dictionary is expected to be present on the event object with the following properties:

  • cuePoints – (required) NSArray of BCCuePoint objects.
  • startTime – (required) NSNumber for the beginning of the time range spanning the cue point.
  • endTime – (required) NSNumber for the end of the time range spanning the cue point.
  • method – (required) Either “play” or “seek”.
