Detailed Description

The delegate protocol used to communicate movie events to application code.

Implement this protocol in application code to receive notice when any of the below events occur in the player. These methods are all optional, implement as many or as few make sense for your application.

Member Function Documentation

- (void) playerComplete   [optional]

The player experience has completed.

This message occurs when the player ui has been dismissed. If the movie ends naturally and the more videos view option is enabled, this method will be invoked when the user exits the more videos view by pressing the close button. If the more videos view option is disabled this method will be invoked when the user presses the done button during playback or the video ends.

- (void) selectedRelatedVideo: (BCVideo *)  relatedVideo   [optional]

The user has selected a related or popular video.

This message is sent when the user has tapped a related or popular video in the video browsing view, which can occur during video playback and/or at the end of video playback, depending upon the options your app decalres on the player instance.
