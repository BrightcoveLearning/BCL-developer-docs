Quick Start: Brightcove Player Debugger

In this topic, you will learn how to use the Brightcove player debugger to capture and log information about your player, media and advertising.

Once you add the debugger plugin in your player, you can track information in real-time as playback occurs in the web browser.

This is an Open Source project and not officially supported by Brightcove.

Review the documentation

Learn about the Brightcove Player Debugger:

  1. Review the README information for the Brightcove player debugger. Here, you will find a code sample along with debugger options and display data.

    The Brightcove player debugger attaches itself to a Brightcove player by using the <video> element's id attribute.

  2. The features of the debugger include:

    • A toggle button lets you minimize the debugger to reduce its footprint when testing players in the wild.
    • It listens to player events using the player.on() method, and displays them in the Log section.
    • It captures the browser's debug console messages and displays them in the Log section.
    • For the current video loaded in the player, the debugger records data from the mediainfo property.
    • When the IMA plugin is used, the debugger records IMA3 ad settings.
    • When the FreeWheel plugin is used, the debugger records the FreeWheel ad settings.
    • When an ad is playing, the debugger records the current ad information. Pause the player to make sure you can view this information in the Ad Settings tab.
    • It captures and displays the current player state using CSS classes that are attached to the player.
    • By selecting/de-selecting a class name in the Player Classes section, you can add/remove classes from the player.
    • By toggling the Log, Classes, Player Settings and Ad Settings tabs, you can show/hide the corresponding display sections.

Create the player

First, we'll create a player in Studio and assign a video to it.

  1. In Studio, navigate to the Players module. Here, create a new player or use an existing one. You can even use the default player in your account.
  2. In the Settings section, select the Edit button and do the following:
    • Set the Width option to 640.
    • Set the Height option to 360.
    You can set the size of the player to whatever you want.
  3. Click Save and then Publish the player.
  4. In the Media module, select a video and publish it with this player.Assign video content to the player and publish it.

  5. Copy the Advanced (In-Page) embed code and paste it into a new HTML file.

    For details on how to get this embed code, see the Quick Start to Brightcove Player document.

  6. In the opening <video> tag, add an id attribute with a value of myPlayer.

    The embed code should look similar to this:

    <video id="myPlayer"
        data-video-id="4825296720001"
    	data-account="1752604059001"
    	data-player="SJJokFGWx"
    	data-embed="default"
    	data-application-id
    	class="video-js"
    	controls
    	width="640"
    	height="360"></video>
    <script src="//players.brightcove.net/1752604059001/SJJokFGWx_default/index.min.js"></script>
    <video id="myPlayer"

Add the debugger files

In this section, we will add the styles and scripts and needed for the Brightcove player debugger.

  1. Below the player code, add a <link> tag which points to the debugger CSS file. 

    <link href="https://solutions.brightcove.com/marguin/debugger/css/brightcove-player-debugger.css" rel="stylesheet">

  2. Below the previous code, add a <script> tag which points to the debugger JavaScript file. 

    <script src="https://solutions.brightcove.com/marguin/debugger/js/brightcove-player-debugger.min.js"></script>

Test your plugin

The plugin generator makes it easy to develop and test your plugin using a local test environment. Follow these steps to test your plugin:

  1. In the Terminal, enter the following command to start the development server: 

    $ npm start

  2. In a browser, enter the following to open the development server: 

    http://localhost:9999/

    You should see the player with a test video. The generator gives you a demo player that runs in the page. Open the Elements tab in the browser developer tools to see the HTML for the player.

  3. Inspect the Elements of this web page. You should see that the vjs-demo class has been added to the player. Remember that we added this class to the player in the plugin code.

  4. Now, let's try adding code to the plugin to automatically start playing the video when the player loads. Return to the src > plugin.js file in your editor.

  5. In the onPlayerReady() method, add code to start playback of the video. 

    const onPlayerReady = (player, options) => {
  player.addClass('vjs-demo');
  player.play();
};

  6. Save the plugin.js file in your editor. You should notice that the video starts playing in the test environment.

    As you develop your plugin in the plugin.js file and save changes, the tool automatically rebuilds and reloads the player in the browser. This makes it easy to develop and test your plugin.

  7. Remove the line of code to start playing the video. 

    player.play();
    To stop the development server, press CTRL-C.

Edit the JavaScript file

In this section, you will add code to the JavaScript source file to add a paragraph element with custom text to the player.

  1. In your editor, return to the src > plugin.js file.

  2. In the onPlayerReady() function, add code to add a <p> element with custom text to the player. 

    const onPlayerReady = (player, options) => {
  player.addClass('vjs-demo');
  var textDisplay = document.createElement('p');
  textDisplay.className = 'vjs-text';
  textDisplay.innerHTML = "Becoming a plugin developer";
  player.el().appendChild(textDisplay);
};
  3. Save the file. Remember that your changes will be automatically updated in the testing browser.

  4. Return to the testing browser. You will see nothing has changed in the player. The problem is that the text is there, but it is not visible. We'll fix that next.

    To verify that the text element has been added to the player, use the development tools in the browser. In the Elements section, expand the player's <div> element. You should see the newly added paragraph tag.

    We'll make the text visible in the next section by using CSS.

Edit the CSS file

In this section, you will add code to the CSS source file to display text over the player.

  1. In your editor, return to the src > plugin.scss file.

  2. Add the .vjs-text selector with styles to display the custom text in the player. 

    // Sass for videojs-demo

.video-js {

  // This class is added to the video.js element by the plugin by default.
  &.vjs-demo {
    display: block;
  }
  .vjs-text {
    background-color: #333333;
    color: white;
    position: absolute;
    font-size: 2em;
    text-align: center;
    width: 100%;
    margin-top: 10%;
  }
}
  3. Save the file. Remember that your changes will be automatically updated in the testing browser.

  4. Return to the testing browser. You should now see the custom text displayed over the player.

    Now, you are ready to prepare your plugin for distribution. We'll do that next.

Build your plugin

In this section, you will build your plugin. This takes your source code and creates distributable CSS and JavaScript files.

  1. In the Terminal, run the following command: 

    $ npm run build

    The build takes your ES6 source code and converts it into ES5 JavaScript code. It also takes your SASS styles and converts it into CSS.

  2. In your project folder, expand the dist folder. Here, you should find the following files (assuming you selected Yes for styling):

    • videojs-demo.css
    • videojs-demo.js
    • videojs-demo.min.js

    The dist folder is where you will find a distributable version of your plugin.

  3. In the dist folder, open the videojs-demo.js file.

    A few things to note about this distribution file:

    • A license banner has been added to the top of the file.
    • Your plugin has been wrapped in a browserify namespace. This means that there are no global variables that might conflict with your page code.
    • There are no global variables that might conflict with your page code.
    • The plugin internally registers itself with videojs. This means that you can create multiple plugins, each working independently, for a player.

Pass data to your plugin

This section is optional. You can skip this section unless you want to learn how to pass data to your plugin.

  1. We'll use the options property to pass data from our HTML page into the plugin. For details on using this property, see the Pass Data to the Plugin document.

  2. In your editor, return to the src > plugin.js file.

  3. In the onPlayerReady() function, add code to use the text value in the options property if it exists, otherwise use a default text value.

    • Line 6: checks if the displayText object exists in the options property.
    • Line 7: uses the displayText value to populate the text display.
    • Line 9: uses the default text display value. 
    const onPlayerReady = (player, options) => {
  player.addClass('vjs-demo');
  var textDisplay = document.createElement('p');
  textDisplay.className = 'vjs-text';

  if ('displayText' in options) {
    textDisplay.innerHTML = options.displayText;
  } else {
    textDisplay.innerHTML = "Becoming a plugin developer";
  }

  player.el().appendChild(textDisplay);
};
  4. Save the file and return to the testing browser. You should not see any change to the text display.
  5. Build your plugin. We'll use the files in the dist folder in a test web page.
  6. In your local testing server, like MAMP, create a folder named plugin-generator.
  7. In this folder, copy the videojs-demo.css and videojs-demo.js files from the dist folder and paste them into your testing server folder, plugin-generator.

  8. Next, in this folder, create an HTML file with a player that calls our demo plugin and passes in a value for the display text. We'll name this file testing.html.

    • Line 8: includes our plugin styles.
    • Lines 13-22: add a Brightcove player to our web page.
    • Line 24: includes our plugin JavaScript file.
    • Lines 26-32: execute a custom page script.
    • Line 27: waits for the player to be ready.
    • Line 28: gets a reference to the player.
    • Line 29: defines the options object.
    • Line 30: calls our demo plugin and passes in the options object. Notice that the plugin name is demo, while the plugin files names are videojs-demo. 
    <!doctype html>
<html>

<head>
  <meta charset="UTF-8">
  <title>Passing data to the plugin</title>

  <link href="videojs-demo.css" rel="stylesheet">
</head>

<body>

  <video id="myPlayerID"
    data-video-id="4784463159001"
    data-account="1507807800001"
    data-player="ByAlTf9xl"
    data-embed="default"
    data-application-id
    class="video-js"
    width="640px" height="360px"
    controls></video>
  <script src="//players.brightcove.net/1507807800001/ByAlTf9xl_default/index.min.js"></script>

<script type="text/javascript" src="videojs-demo.js"></script>

<script type="text/javascript">
  videojs('myPlayerID').ready(function(){
    var myPlayer = this,
      options = {"displayText": "This data supplied at initialization"};
      myPlayer.demo(options);
  });
</script>

</body>
</html>

  9. In a browser, run the testing.html file. You should see the display text value is coming from the page code instead of the plugin.

Recommendations

It is recommended to use this generator for all of your plugins, even the simple ones. This way, you will always have the same setup for all of your plugins. It also frees you from the task of having to create scripts for perform certain functions, like linting or minifying.

With this generator, you can focus on developing and testing your plugin without having to worry about the underlying tools.

You can keep your plugin local or put it on your private GitHub repo. Creating a videojs plugin means that it will work with the Brightcove player.
