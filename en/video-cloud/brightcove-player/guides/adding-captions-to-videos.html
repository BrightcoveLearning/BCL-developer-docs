Add Captions to Videos

This topic provides an overview of how to add captions to videos using the new Brightcove player.

Closed captions can be added to your videos to provide additional or interpretive information to individuals who wish to access it. Captions are necessary so the deaf and hard of hearing can access the audio portion of video presentations. To display captions for a video in the Brightcove player, you need to complete the following tasks:

  • Create a captions file for your video
  • Modify the player publishing code to include a <track> tag
  • Style the captions using CSS

Create caption files

To add captions to a video, you first need to create a captions file. The Brightcove player requires that captions be in Web Video Text Tracks Format (WEBVTT) format. The WEBVTT file format is in plain text and basically contains time text data followed by the desired caption or subtitle text. A portion of a WEBVTT file is shown below.

WEBVTT

1
00:00:00.000 --> 00:00:03.530 align:middle line:84%

2
00:00:03.530 --> 00:00:06.510 align:middle line:90%
In this video we'll learn how
to log into and navigate around

3
00:00:06.510 --> 00:00:09.550 align:middle line:84%
inside of Video Cloud Studio.

4
00:00:09.550 --> 00:00:12.020 align:middle line:90%
Video Cloud Studio is
a web-based application

For more information on the WEBVTT format, see this link. You should have a separate WEBVTT file for each caption language you need to support. While there are many third-party tools to help create captions files, Brightcove has several partners that provide transcription, captioning and translation services. For a complete list of Brightcove partners, visit the Partners section of our website.

Include a <track> tag

To associate WEBVTT caption file(s) to a Brightcove player, you need to modify the In-Page embed code to include <track> tag(s) for each captions file you have. For example:

<track kind="captions" src="<location of caption file>" srclang="en" label="English" default>

The <track> tag is used to specify subtitles, caption files or other files containing text, that should be visible when the media is playing. The steps to do this are outlined below.

  1. Generate the In-Page embed code for your player. This can be done using the Media module or the Players module. A sample is shown below.
      <video
    data-account="20318290001"
    data-player="d508c6e1-efa7-4aad-ba91-1a8db9761b8c"
    data-embed="default"
    data-video-id="2070993148001"
    class="video-js" controls>
  </video>
  <script src="//players.brightcove.net/20318290001/d508c6e1-efa7-4aad-ba91-1a8db9761b8c_default/index.min.js"></script>
  2. Add <track> tag(s) inside of the <video> tag that point to location of all caption files. Note that publishers are responsible for hosting caption files. Brightcove does not provide a way for publishers to upload caption files. In the example below, English, Japanese and French caption files have been specified.
      <video
    data-account="20318290001"
    data-player="d508c6e1-efa7-4aad-ba91-1a8db9761b8c"
    data-embed="default"
    data-video-id="2070993148001"
    class="video-js" controls>
    <track kind="captions" src="http://solutions.brightcove.com/intro.vtt" srclang="en" label="English" default>
    <track kind="captions" src="http://solutions.brightcove.com/intro-jp.vtt" srclang="ja" label="Japanese">
    <track kind="captions" src="http://solutions.brightcove.com/intro-fr.vtt" srclang="fr" label="French">
  </video>
  <script src="//players.brightcove.net/20318290001/d508c6e1-efa7-4aad-ba91-1a8db9761b8c_default/index.min.js"></script>
  3. Publish the page with the player. When a player with associated captions is published, a CC button will appear on the player chrome.
  4. Click the CC button to select a caption language and display the captions.

Style captions using CSS

Typically you will need to style the captions to make them easier to see or read. The recommended method to style captions is to use your browser's developer tools to inspect the page elements to determine class names and then add CSS code to your page with the Brightcove player. A sample is shown below.

  1. Open the page URL containing the player.
  2. Use the browser developer tools to determine class names of the page elements. In the example below, the class name for the text track is being determined.
  3. Use CSS to style the captions. In this example, the background color is being changed to black, the font size is increased and the font is bolded. Having the CSS as part of the page source ensures that all of the captions will be styled the same way (vs. adding styling options inside of the WEBVTT file). The final publishing code with the CSS styling is shown below.
    <style>
	.vjs-text-track {
		background-color: black;
		font-size:medium;
		font-weight: bold;
    }
</style>  
   <video
    data-account="20318290001"
    data-player="d508c6e1-efa7-4aad-ba91-1a8db9761b8c"
    data-embed="default"
    data-video-id="2070993148001"
    class="video-js" controls>
    <track kind="captions" src="http://solutions.brightcove.com/intro.vtt" srclang="en" label="English" default>
    <track kind="captions" src="http://solutions.brightcove.com/intro-jp.vtt" srclang="ja" label="Japanese">
    <track kind="captions" src="http://solutions.brightcove.com/intro-fr.vtt" srclang="fr" label="French">
  </video>
  <script src="//players.brightcove.net/20318290001/d508c6e1-efa7-4aad-ba91-1a8db9761b8c_default/index.min.js"></script>
  4. After republishing the page, the style changes to the captions can be seen.
