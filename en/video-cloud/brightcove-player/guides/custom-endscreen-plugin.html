Custom Endscreen Plugin

This topic covers the use of the Custom Endscreen plugin and how it can be implemented using Video Cloud Studio and through custom coding. The plugin allows display of customer supplied HTML at the conclusion of video playback.

The sample video below demonstrates the use of the Custom Endscreen plugin. At the end of this short video you will see text displayed from the following HTML:

<strong>Content</strong> for <em>custom</em> end screen

Note: The replay icon and the social sharing icon always appear on the endscreen. At this time, the use of custom endscreens requires social sharing to be enabled.

Implement using Players module

Custom endscreens can be implemented by editing the endscreen properties or by using the custom endscreen plugin. To implement the custom endscreen plugin using the Players module, follow these steps:

  1. Open the Players module and locate the player.
  2. Click the link for the player to open the player properties.
  3. Locate the Plugins section and click Edit.
  4. The JavaScript for the plugin is provided and will not change for different plugin implementations. For the JavaScript URL, enter:

    5.  https://players.brightcove.net/videojs-custom-endscreen/dist/videojs-custom-endscreen.min.js

  5. The CSS for the plugin is provided and will not change for different plugin implementations. For the CSS URL, enter:

     https://players.brightcove.net/videojs-custom-endscreen/dist/videojs-custom-endscreen.css

  6. For the Name, enter customEndscreen.
  7. Enter the plugin options. The example below will display the rendered HTML on the endscreen above the icons.
    {
  "content": "<strong>Content</strong> for <em>custom</em> end screen"
}
  8. Click Save and then Publish the player.

Implement using code

To implement a plugin the player needs to know the location of the plugin code, a stylesheet if needed, the plugin name and plugin configuration options. The location of the plugin code and stylesheet are as follows:

Property

The following plugin property is used to control the Custom Endscreen plugin:

  • content:
    • Type: string
    • Any string literal or HTML that is rendered on the custom endscreen.
