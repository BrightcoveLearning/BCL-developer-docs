Quick Start to Player Customization

To customize the appearance of Brightcove Player you need to use CSS. This document provides hands-on steps you can follow to become familiar with the process. The task used as the focus of this document is changing the appearance and location of the play button that initially overlays the video.

This document will have several sections as using CSS with the in-page embed and the iframe implementations of the player are quite different.

The content in this document uses Chrome's development tools, but all modern browsers have equivalent functionality. For a brief introduction to using development tools in different browsers see the Debugging Basics document.

Determine player selector

The first thing you must do when trying to alter the look of an element on an HTML page is determine the selector to use to address to the element. The following steps show how to find the selector for the player itself. In these steps you will find the correct selector and change the size of the player.

  1. Create an HTML page that has a uses the Brightcove Player to play some video.
  2. Browse the newly created page.
  3. Open your browser's development tools and drill down into the HTML to find the <div> tag that surrounds the player. You will see the tag has class of video-js, which will always be the case. You see the class highlighted in red in the screenshot. Highlighted in green is the video tag, which is Brightcove Player.

  4. You now know the correct class to use for a CSS selector to change styles for the player itself. Use that class selector to change the size of the player. The <style> block below should be placed just above the end of the head section of your HTML page. In CSS terminology, this is an example of using a class selector.

  5. Save and browse your HTML page and you will see the player has a new size.

Play button selector

Another approach is going to be used to find the selector for the play button. The following steps show you how to find the correct selector for the play button.

  1. Browse your HTML page with the developer tools open.
  2. Right click on the play button that overlays the player and select Inspect Element.
  3. Note that the <div> tag has a class of vjs-bit-play-button.
  4. Alter your styles to include CSS that changes the color of the button and its opacity (lines 17-18). as well as centers the button (lines 19-23).

  5. Browse your page and you will see not all the styles have been used. The button is not red.
  6. Look to the right of your developer tools and you will see a summary of your styles. You can look down and see that changing the background to red is overridden and crossed out.

    What has happened is that numerous changes are made to the background color in the style you set has be overridden in the "cascading" part of CSS. You can change this by utilizing CSS specificity. This technique makes your style more important than others, so it will be used. There is a brief introduction to CSS specificity in the Customize Player Appearance doc, and numerous references in any search.

  7. Note that the <div> tag surrounds the player has numerous classes associated with it.
  8. In this case, by using the combination of video-js and vjs-mouse will "boost" your styles to take precedence. Alter your styles to use the new combination selector.

  9. Browse your page and see that all the styles are now applied.
  10. Again look to the right of your developer tools and you will see a summary of your styles. Now you see that your background color change is not overridden.

In fact, you could have and it would have worked. The problem with that is this approach will not work when using the iframe player implementation, so only existing class names were used. By not using an id attribute the styles will be directly applicable to iframe player implementations.

iframe

The iframe implementation, not surprisingly, places Brightcove Player in an iframe that you then place in an HTML page. You can style the actual iframe itself, change its size for instance, but styles on the parent page of the iframe DO NOT cascade into the iframe itself. The iframe literally pulls in a separate HTML page, so desired styles for the iframe contents must associated with the iframe's contents.

When using the iframe implementation of Brightcove Player here are the two steps to apply CSS:

  • Place styles in an Internet accessible file. Do not include the opening and closing <style> tags.
  • Link the stylesheet to the player. This can be done in Studio or using the Player Management API.
