iframe Player Implementation

In this topic, you will learn about the use and configuration of the iframe code. This version of the player implementation is also referred to as the Standard player code.

Introduction

The iframe player implementation code provides the ease of use of the player as a single, contained unit in social media applications and sharing.

Usage

Note: If you are a Perform customer, you will not use the videoId query parameter to assign a video to your player. You would either assign the video in the Players module or using curl.

To use the iframe code follow these steps:

  1. In Studio's PLAYERS module create a player.
  2. Go to the MEDIA module and publish a video using the newly created player.
  3. Copy the Standard code. The HTML will be similar to the following: 
    <iframe src="//players.brightcove.net/921483702001/default_default/index.html?videoId=4570583963001"
  allowfullscreen
  webkitallowfullscreen
  mozallowfullscreen></iframe>
  4. Copy the HTML from the browser and paste into the body of a full HTML page.
  5. Browse the HTML page to see the player functioning.

Query parameters

Numerous query parameters are available for the iframe tag to provide additional information about how a player should behave. The table below details available attributes.

When using tag attributes that can only be true or false (data type Boolean), you simply include the attribute (no equals sign) to turn it on, or exclude it to turn it off.

RIGHT: controls

WRONG: controls="true"

Attribute Description Data Type
account_id The Video Cloud account ID associated with this player. String
application_id By default, this is interpreted from the applicationId query parameter for iframe players or the data-application-id attribute for in-page players. The application ID is used to differentiate multiple uses of a single player in metrics. String
autoadvance This will set the autoadvance behavior of the playlist. Only applicable if a playlist is being used. See the Playlist API document for more information. Number
autoplay Causes the video to start playing as soon as page is loaded (without any interaction from the user). Supported on APPLE iOS devices only if muted used concurrently. Boolean
class Standard HTML attribute that will be assigned the video-js value by default. String
controls Shows player controls. Boolean
compatibility If explicitly true, the player will include videojs-4to5, which is a layer for backward compatibility to Brightcove Player 1.x. Boolean
controls Determine if the controls should be visible in the player. Boolean
crossorigin If true, the underlying <video> element will have crossorigin="anonymous". This means that any videos or text tracks pulled into the player must have CORS headers.
Note: This feature should only be used if you know that all your content (video and text tracks) have CORS headers applied to them.
 Boolean
data-account The account ID, sometimes also called the Publisher ID. String
data-application-id Allows re-use of a single player, but differentiates analytics on a per-site or per-application basis. See the Player Application ID for Analytics document for full details. String

poster

Note: To disable the poster image, set this attribute as follows: poster="".
preload Informs the browser whether or not the video data should begin downloading as soon as the video tag is loaded. Boolean
