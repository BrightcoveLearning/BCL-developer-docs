Playlists Guide

Vocabulary Review:

  • When a method retrieves a value it is called a getter.
  • When a method assigns a value it is called a setter.
  • When defining a method the placeholder for the value that is passed to the method when called is a parameter, e.g. function myMethod(parameter).
  • When calling a method the value in the parentheses is called an argument, e.g. myMethod(argument).

NOTE: It is not unusual for one method to act both as a getter and setter. When the method uses an argument the method acts as a setter, with no argument the method acts as a getter.

currentItem( ) method

The player.playlist.currentItem(index) method either retrieves the current video from the playlist playing in the player, or assigns a new video to play in the player, based on the index. If called without an argument the method performs the retrieval and acts as a getter, with an argument is assigns the value and acts as a setter.

Parameter (setter):
  • index: A number that represents the array index from the playlist of the video to be loaded into the player.
Returned value (getter):

The method returns n value of type number. The returned value represents the array position of the video currently playing in the player. If the player is currently playing a non-playlist video, currentItem() will return -1.

autoadvance( ) method

The player.playlist.autoadvance( ) method sets playlist auto advancement. Once enabled it will wait a timeout amount of milliseconds at the end of a video before proceeding automatically to the next video. Any value which is not a positive, finite, integer, will be treated as a request to cancel and reset the auto advancing. If you change autoadvance during a timeout period, the auto advance will be canceled and it will not advance the next video but it will use the new timeout value for the following videos.

Parameter:
  • timeout: An integer number in milliseconds to delay before loading the next video in the playlist
Returned value:

The method does not return a value.

Example uses of the method follow:

player.playlist.autoadvance(0); Will not wait before loading the next item

player.playlist.autoadvance(5000); Waits 5 seconds before loading the next item

player.playlist.autoadvance(null); Resets and cancels the auto advance
