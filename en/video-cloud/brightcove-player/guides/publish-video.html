Publishing Video Options

When you create a player you see three options for publishing the player, and of course playing the video the player contains. These implementations are:

  • URL
  • In-page embed
  • iframe

For each of these implementations can take two approaches. The first approach is where a video is placed in the player with the intention of not changing the video in the player. In other words the video is statically bound to the player. This can be done in Studio or using a curl statement. Note that this does not mean it cannot be dynamically changed, it is more the intention is not to change it. A use case of this approach is a product page is built with a player that contains a video detailing the product.

A second approach is to create the player with the intention that it will play may different videos. The technical implementation of dynamically changing the video is detailed in this document. A use case for this approach is a template page that displays product information for many products, so the text and video on the page would change.

Note: Although it is possible to change the video statically bound to a player, it is not a good practice. If you do this, it is possible you will see the poster for the statically bound video could briefly appear in the player before the video dynamically added is displayed. For this reason is it a best practice to create a player that does not have a video initially associated with it for use in a situation where the video will be dynamically changed.

URL

The URL implementation can be copied from Studio or the returned results from a curl statement. You can simply paste the URL into a browser and you will see the video full screen. In this case, a video source has been has been statically bound to the player.
