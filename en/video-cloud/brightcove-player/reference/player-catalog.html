Player Catalog Reference

Brightcove Player's catalog is a helper library of methods and properties for making requests to the Video Cloud catalog. The catalog makes it easy to get information on Video Cloud media and load them into a player.

The basic syntax for using the catalog is as follows:

  • Methods: playerName.catalog.methodName()
  • Properties: playerName.catalog.property

Loading options are also documented in this reference.

Methods

getVideo(videoID,callback)

Makes a catalog request for the video with the specified id and invokes a callback when the request completes.

Parameters:
  • videoID: A string containing the video ID or a reference ID prefixed with ref:.
  • callback: A function that will be called when the request completes, whether it's successful or not. It is invoked with two arguments.
    • The first argument will contain an error object, if an an error occurs. This object contains details of the error. If no error occurs, this object is null.
    • The second argument is the requested video object, if no error occurs. If an error occurs, this value contains the empty string.

load(video)

Load a video object into the player. This method will update the player's mediainfo property, and update the video source and poster. In most cases, you would call this method with the result of a call to getVideo(). If you'd like to be notified when mediainfo has been updated, listen for the loadstart event.

Parameters:
  • video: A video object to load. This object should have the same format as the response object from a call to getVideo() method.

Properties

  • catalog.data: The last response object returned by the catalog API.
  • catalog.error: The last error that occurred in using the catalog. This property is undefined if no request has caused an error.
  • catalog.loading: The number of currently in-flight catalog requests. You can use this property to determine whether, and how many, responses are forthcoming.

Loading options

You can pass options to the catalog when it's being initialized. Unless you have specifically disabled the catalog, this information is automatically filled out by the Player Management service.

  • accountId: The account Id
  • baseUrl: [optional] the base url to use. Defaults to https://edge.api.brightcove.com/v1/
  • policyKey: the policy key to send to the catalog
