Brightcove Player Sample: Download Video Plugin

This topic shows you how to create a link to allow downloading the best quality MP4 of a video playing in Brightcove Player. The code will eventually be built as a plugin for easy use with different players.

The link will be implemented in three different ways:

NOTE: There is a working sample is below in the The player section.

  • Icon in the control bar
  • Overlay placed on the video
  • An HTML link you can place anywhere on your HTML page

NOTE: Video downloads are not supported on iOS devices.

+Development sequence

Here is the recommended development sequence:

  1. Use the embed_in_page player implementation to test the functionality of your player, plugin and CSS (if CSS is needed)
  2. Put the plugin's JavaScript and CSS into separate files for local testing
  3. Deploy the plugin code and CSS to your server once you have worked out any errors
  4. Use the new Video Cloud Studio to add the plugin and CSS to your player
  5. Replace the embed_in_page player implementation if you determine that the iframe implementation is a better fit (detailed in next section)

For details about these steps, review the Quick Start to Plugin Development guide.

iframe or embed_in_page

When developing enhancements for the Brightcove player you will need to decide if the code is a best fit for the iframe or embed_in_page implementation. The best practice recommendation is to build a plugin for use with an iframe implementation. The advantages of using the iframe player are:

  • No collisions with existing JavaScript and/or CSS
  • Automatically responsive
  • The iframe eases use in social media apps (or whenever the video will need to "travel" into other apps)

Although integrating the embed_in_page player can be more complex, there are times when you will plan your code around that implementation. To generalize, this approach is best when the containing page needs to communicate to the player. Specifically, here are some examples:

  • Code in the containing page needs to listen for and act on player events
  • The player uses styles from the containing page
  • The iframe will cause application logic to fail, like a redirect from the containing page

Even if your final implementation does not use the iframe player, you can still use the embed_in_page player with a plugin that contains your JavaScript and CSS. This encapsulates your logic so that you can easily use it in multiple players.

Select implementation

In this sample, the functionality is contained in the code and CSS with no dependencies on the parent page, so a plugin is a good choice for this functionality.

The player

Click the download icon located on the left side of the control bar to download the highest quality MP4 rendition of the video loaded in the player.

How it works

The following sections explain the player, JavaScript and CSS code used in the example.

Create the player

Login to your Video Cloud Studio account and create a new video player. Open the player's properties page and edit the Video Content section. Here you can specify the Video ID for the video you wish to publish. Video IDs can be found in the Media module.

The player code

Once your player has been published, you can click the Embed Code & URL button and choose the Published Player link. You will find the player code in the In-Page Embed Code section.

The JavaScript code

Although not terribly complex, the code is a bit long. The following provides an overview of the functionality of the code:

  1. Retrieve the video ID of video in the player. Notice that the data retrieved by the options() method includes data-video-id.
  2. Use the video's ID to get the video object and load the video. This may seem unnecessary as the video is already playing, but you need to load the video to populate the mediainfo property. Notice that the data retrieved in the madiainfo property contains a sources array that contains renditions.
  3. Retrieve the sources array, from the mediainfo property, which includes as elements all renditions of the currently loaded video.
  4. Create another array which contains the filtered and sorted renditions so only the MP4 renditions are in a new array, sorted from largest to smallest.
  5. Assign the zeroth element of the new array to a variable. The zeroth element is the largest, hence the best quality.
  6. Create a link for download.

The following code is the actual implementation of the general steps above.

  • Lines 28-35: Create variables used later in the code.
  • Line 36: Assign a variable the video ID, retrieved from data by calling the options() method.
  • Line 37: Use the getVideo() method, passing the video ID as a parameter, to get the currently playing video object.
  • Line 38: Use the load() method.
  • Line 39: Assign the name of the video to a variable for later use in the link tag.
  • Line 40: Assign the sources array to a variable.
  • Line: 41: Assign the length of the array to variable. You could have also just assigned the length property in sources array.
  • Lines 42-46: Loop over the array and assign to a second array renditions where the container is MP4, AND rendition has a src property. The MP4 renditions are listed twice, once for with a src and once with a stream_name. The first is for progressive download and the second for streaming.
  • Lines 47-49: Sort the array of MP4s.
  • Line 50: Assign the largest rendition to a variable.
  • Lines 51-54: Create and style a div tag that contains the link.
  • Lines 55-56: Retrieve the vjs-control-bar element and append the newly create div to it. It may seem odd that with an append the icon appears on the left side of the control bar, but this is because all the other elements or positioned are the control bar using CSS.

The stylesheet

No stylesheet is used in this implementation. The two styles for the icon were assigned via JavaScript.

Use a plugin

It is a best practice to use a plugin so that you can easily add the JavaScript code's functionality to multiple players.

Create the JavaScript file

Move the plugin's JavaScript code into a file and place it in an Internet accessible URL.

  • Lines 146,177: Use the standard syntax to begin and end a new player plugin
  • Line 146: The first argument of the function call is the name of the function defined in the plugin code, in this case downloadVideoPlugin
  • Line 177: The trailing semicolon is required

Create the CSS file

There is not a CSS file needed for this plugin.

Deploy plugin and CSS

To deploy the NAME Plugin using the Players module, follow these steps:

  1. In the new Video Cloud Studio, open the Players module and locate the player.
  2. Click the link for the player to open the player properties.
  3. Locate the Plugins section and click Edit.
  4. For the JavaScript URL, enter the URL which specifies the location of where you saved your plugin's JavaScript.
  5. There is no CSS file to enter.
  6. For the Name, enter downloadVideoPlugin.
  7. There are no plugin options to enter.
  8. Click Save and then Publish the player.

Use iframe

It is a best practice to use the iframe player implementation. If it is possible for this example, replace the embed_in_page implementation with the iframe implementation.
