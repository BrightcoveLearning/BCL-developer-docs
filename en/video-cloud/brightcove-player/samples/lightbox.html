Brightcove Player Sample: Lightbox with Scroller

Note: the Brightcove player is currently in beta and requires some developer skills to implement. If you are interested in participating in the beta, please contact your account manager.

This topic shows you how to create a lightbox and simple scroller with the Brightcove player.

Development sequence

Here is the recommended development sequence:

  1. Use the embed_in_page player implementation to test the functionality of your player, plugin and CSS
  2. Put the CSS and the plugin's JavaScript into separate files to test locally
  3. Once you have worked out any errors, deploy the plugin code and styles to your server
  4. Use the HTTP PATCH method to add the styles and plugin to your player
  5. Replace the embed_in_page player implementation if you determine that the iframe implementation is a better fit (detailed in next section)

For details about these steps, review the Quick Start to Plugin Development guide.

iframe or embed_in_page

When developing enhancements for the Brightcove player you will need to decide if the code is a best fit for the iframe or embed_in_page implementation. The best practice recommendation is to build a plugin for use with an iframe implementation. The advantages of using the iframe player are:

  • No collisions with existing JavaScript and/or CSS
  • Automatically responsive
  • The iframe eases use in social media apps (or whenever the video will need to "travel" into other apps)

Although integrating the embed_in_page player can be more complex, there are times when you will plan your code around that implementation. To generalize, this approach is best when the containing page needs to communicate to the player. Specifically, here are some examples:

  • Code in the containing page needs to listen for and act on player events
  • The player uses styles from the containing page
  • The iframe will cause app logic to fail, like a redirect from the containing page

Even if your final implementation does not use the iframe code, you can still use the embed_in_page code with a plugin for your JS and a separate file for your CSS. This encapsulates your logic so that you can easily use it in multiple players.

Select implementation for this sample

In this sample, you will test with the embed_in_page code and then move the CSS and JS into a plugin. Since the lightbox interacts with the page elements, the iframe implementation will not be used.

API resources used

This sample uses both the Brightcove player API and the Player management API.

Brightcove player API methods

  • src()
  • play()
  • pause()

Player management API

  • HTTP POST method with the endpoint https://players.api.brightcove.com/v1/accounts/$ACCOUNT_ID/players to create the player and its proper configuration

The HTML code

Add <div> tags to your page code for the following:

  • A scroller that will hold the video thumbnails. The style for the scroller will be defined in CSS and the video content will be loaded in JavaScript.
  • The lightbox container with the video player. Initially, the lightbox will be hidden.
  • A Close button in the lightbox.

Note: Make sure to move the player <script> tag below the page HTML code. The Brightcove player script must be placed after your page code. It is common practice to put your scripts just before the closing </body> tag.

The JavaScript code

Load the scroller of video thumbnails and manage the lightbox interaction in the JavaScript code. This is done by:

  1. When the player is ready, get a reference to it.
  2. Call the buildPlaylistData() method to load the scroller with video thumbnail images and short descriptions.
  3. Define the scroller item onclick event as a function that loads the selected video, reveals the lightbox and attempts to start playing the video.
  4. Define the Close button onclick event as a function that stops playing the currently loaded video and hides the lightbox.

Note: For this sample, the playlist data is hard-coded in an array, but you could easily read this data from an external file.

The stylesheet

The stylesheet defines the following:

  • The layout and look of the lightbox. You will call classes here to show and hide the lightbox.
  • The layout and look of the scroller and its video items.
  • The dimensions and position of the player.
