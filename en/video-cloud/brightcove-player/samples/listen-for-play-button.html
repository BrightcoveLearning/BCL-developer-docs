Brightcove Player Sample: Play Video from iframe Parent

Note: the Brightcove player is currently in beta and requires some developer skills to implement. If you are interested in participating in the beta, please contact your account manager.

This topic shows you how to use a button on the parent page of an iframe player to play the video in the iframe player.

iframe or embed_in_page

When developing enhancements for the new player you will need to decide if the code is a best fit for the iframe or embed_in_page implementation. The best practice recommendation is to build a plugin for use with an iframe implementation. The advantages of using the iframe player are:

  • No collisions with existing JavaScript and/or CSS
  • Automatically responsive
  • The iframe eases use in social media apps (or whenever the video will need to "travel" into other apps)

Although integrating the embed_in_page player can be more complex, there are times when you will plan your code around that implementation. To generalize, this approach is best when the containing page needs to communicate to the player. Specifically, here are some examples:

  • Code in the containing page needs to listen for and act on player events
  • The player uses styles from the containing page
  • The iframe will cause app logic to fail, like a redirect from the containing page

In this case the point of the example is to interact with the iframe player, so an iframe player will be used.

How it works

The player code

The player code is simply the iframe implementation displayed when the player is published. Also added to this code is an HTML button.

This player was created using the following curl statement. The following configuration options are used:

  • The video to play is added in sources.
  • The location of the plugin's JavaScript code is added in scripts.
  • The location of the plugin's CSS is added in stylesheets.
  • The name of the plugin, defined in the plugin's Javascript, is added in plugins.

The JavaScript code

The iframe player's parent page uses JavaScript's postMessage() method to send a message to the iframe. This is done by:

  1. Use the document.querySelector method to retrieve the iframe element.
  2. Retrieve the Window object generated by an iframe using the iframe's contentWindow property.
  3. On button click, postMessage is used to send a message to the iframe's Window object. The first argument is the data passed as part of the event. (In this case it is simply a string, but can be an object.) The second argument specifies the origin of the iframe. You can use the * wildcard to have the event sent to any content in the iframe, no matter where it came from, but this is considered a poor security practice. Using http://players.brightcove.net as the argument means the event will only be sent to the iframe if the content originated from that URL.

The plugin uses JavaScript to listen for the event, and when handled, play the video. This is done by:

  1. Listen for the event using addEventListener(). The postMessage() method dispatches an event of type message (the first argument). The defined event handler is playVideo (the second argument).
  2. In the playVideo() event handler, an if statement is used to check the value of the data passed by postMessage(), this case the string playVideo. This is not required in this simple example as no other events are sent to the iframe, but is considered a good practice.
  3. The player's play() method is called.

The stylesheet

The stylesheet used by the player does one simple task, prevents the default play button from appearing over the video. Since a play button is created as part of the parent page, the default play button on the video is not needed, although it would not have changed any behaviors.
