Playlists resource

Path

$BASE_URL/accounts/1234/playlists

$BASE_URL/accounts/1234/playlists/favorites

Usage

GET

Returns the list of playlists in the account. Default limit (page size) is 20.

Fields

Field Type Description
id String
account_id String
created_at Number, milliseconds from epoch
description String
favorite Boolean
limit Number
name String
reference_id String
search String, smart playlists only
type String value of established enum
updated_at Number, milliseconds from epoch
video_ids Array of Strings, EXPLICIT (manual) type only

Options

Options Default
limit 20
offset 0
sort updated_at desc

POST

To add a new playlist to the account.

Field Type Required
description String no
favorite Boolean not yet supported
limit Number not yet supported
name String yes
reference_id String no
search String, smart playlists only no
type String value of established enum yes
video_ids Array of Strings, EXPLICIT (manual) type only no

PATCH

not supported

Type

The 'type' value can only be one of: 'EXPLICIT', 'ACTIVATEDOLDESTTONEWEST', 'ACTIVATEDNEWESTTOOLDEST', 'ALPHABETICAL', 'PLAYSTOTAL', 'PLAYSTRAILINGWEEK', 'STARTDATEOLDESTTONEWEST', 'STARTDATENEWESTTO_OLDEST'

'EXPLICIT' means that it is a manual playlist; all the other types are smart playlists with names corresponding to their ordering scheme.

Search

Regarding the 'search' field, the supported values for the search string is very limited. You can only search by tags. Here are some example valid searches:

tags:foo +tags:bar tags:foo,bar +tags:"foo bar","boo yah"

For specific scenarios: contains all: "+tags:foo,bar" ** NOTE: we have a bug about this, ACME-317 contains any: "tags:foo,bar" contains none: "-tags:foo,bar"

Favorites

GET $BASE_URL/accounts/1234/playlists/favorites

  • No custom sorting; always sorted by name ASC
  • No search
  • No paging
  • Limit of 100; will return the first 100 alphabetically
