Quick Start to CMS

The CMS API is currently in beta and is not supported by Brightcove Support. If you would like access to the beta, please contact your Brightcove Account Manager.

In this quick start you will perform the following tasks:

  • Obtain credentials to use the Content Management System (CMS) API.
  • Retrieve all videos from your account.
  • Create a video and view it in Video Cloud Studio.
  • Retrieve information for a specified video.
  • Retrieve rendition information for a specified video.
  • Retrieve image information for a specified video.
  • Search videos in your account by keywords (tags) and by creation date.

For details on the API used in this document you can view the Content Management System API Overview document and the Content Management System API reference.

OAuth proxy

The Content Management System (CMS) API's security is implemented using OAuth2. To permit you to get started using the CMS API as quickly as possible, the API calls in this quick start will use a security proxy running at http://solutions.brightcove.com:8006. If you browse to that URL, you will be redirected to an informational page on the proxy. Be aware that the proxy will only accept requests originating from localhost and *.brightcove.com domains.

Retrieve all videos

One of the basic tasks you will want to perform is retrieve all videos in an account. The fundamental call information is:

  • HTTP method: GET
  • Endpoint: https://cms.api.brightcove.com/v1beta1/accounts/YOUR_ACCOUNT_ID/videos

Note: Before completing this task you must perform steps 1 and 2 above to retrieve and assign account and credential information.

Create video

The fundamental call information for creating a video is:

  • HTTP method: POST
  • Endpoint: https://cms.api.brightcove.com/v1beta1/accounts/YOUR_ACCOUNT_ID/videos

The video object created will eventually contain the master copy of the video, metadata, possible renditions and other information. In the call body you supply a simple JSON object specifying the name of the video to be created. To see the fields for which you can provide values during the POST, browse to the CMS API Overview document's Setting values section.

Note: Before completing this task you must perform steps 1 and 2 above to retrieve and assign account and credential information.

  1. You will now use the proxy in conjunction with the following form to create a video. Be sure to alter the body so the name property reflects your choice for the video name.

Using reference IDs

In the CMS API calls in the rest of this quick start, the video ID is used in the endpoint URL. You can also use the video's reference ID if you wish. If you use the reference ID you must prepend that identifier with ref:. For example, the first URL uses the video ID, whereas the second uses the reference ID.

Retrieve rendition info

  • Endpoint: https://cms.api.brightcove.com/v1beta1/accounts/YOUR_ACCOUNT_ID/videos/VIDEO_ID/sources.

Note: Before completing this task you must perform steps 1 and 2 above to retrieve and assign account and credential information.

  1. You will now retrieve rendition information for a specific video. You will again use the HTTP GET method, with a video ID.

Search by keywords

The CMS API has a rich searching implementation which you can leverage. Not only does the implementation provide many ways to search, but you can also manipulate the display of results by sorting, limiting result set size, etc. A few examples will be shown in this quick start. For complete documentation see the CMS API Overview document's Search section.

You will now search your videos for a certain keyword. These keywords can be set via API, or in Video Cloud Studio where they are referred to as tags. The fundamental call information for searching video data is:

  • HTTP method: GET
  • Endpoint: https://cms.api.brightcove.com/v1beta1/accounts/YOUR_ACCOUNT_ID/videos.

Note: Before completing this task you must perform steps 1 and 2 above to retrieve and assign account and credential information.

  1. You will now search for videos with a keyword of wildlife. To do this, append a question mark to the endpoint to create the query string, then a q:, and finally the syntax to look for the specific keyword, in this case keywords:wildlife.
  2. Check to be sure your URL is in the form:

    https://cms.api.brightcove.com/.../videos?q=keywords:wildlife

Search by creation date

You will now search for videos created after a certain date. To do this add a query string to the endpoint, then a q:, and finally the syntax for a starting date, but no ending date. The fundamental call information for searching video data is:

  • HTTP method: GET
  • Endpoint: https://cms.api.brightcove.com/v1beta1/accounts/YOUR_ACCOUNT_ID/videos.

Note: Before completing this task you must perform steps 1 and 2 above to retrieve and assign account and credential information.

  1. You will now search for videos created after 1 Jan 2014. To do this, append a question mark to the endpoint to create the query string, then a q:, and finally the syntax to specify the date, in this case q=created_at:2014-01-01T00:00:00Z...
  2. To sort the results by creation date, append to the existing URL an ampersand, then the sort directive, which is sort=created_at.
  3. Check to be sure your URL is in the form:

    https://cms.api.brightcove.com/.../videos?q=created_at:2014-01-01T00:00:00Z..&sort=created_at
