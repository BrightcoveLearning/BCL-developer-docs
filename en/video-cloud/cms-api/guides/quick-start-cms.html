Quick Start to CMS

In this quick start you will perform the following tasks:

  • Create a Node.js application to act as your OAuth security proxy. This proxy will use your access token to make calls to the needed APIs.
  • Obtain credentials to use the Content Management System (CMS) API.
  • Search videos in your account in various ways.
  • Create a video using the CMS API and view it in Video Cloud Studio.

For details on the API used in this document you can view the Content Management System API Overview document and the Content Management System API reference.

Note: The steps in the document involve curl and other command-line tools that are native to Mac and Linux systems, but not part of the Windows operating system. If you are on Windows, you can get these tools by installing cygwin. During the install you will be prompted for packages to install. Open the Net package, then check the curl option and then finish the installation. Once you have done this you will be able to follow the Quick Start to the Content Management System successfully as written using the cygwin command line. Tools are available to run curl at the normal Windows command line, but not detailed in this document.

Create proxy

Security on the CMS API uses OAuth2. This requires some work to access the API.

  1. In this quick start it is recommended you set up a Node.js application to act as a proxy. The OAuth Quick Start guides you through the steps to create a basic proxy in Node.js that gets an access token for a Brightcove API, makes the API call, and then returns the response data.
  2. You will also need a local web server running to serve up a testing page, tester.html, which works with the proxy. Note that simply browsing the page as a file will NOT work as the page utilizes an iframe, which requires an HTTP server.

Obtain credentials

To use the CMS API you will need proper credentials. Later in this quick start, you will use the credentials in a form when submitting requests to APIs.

  1. To obtain proper credentials use your choice of these two options (Be sure to read the paragraph following the bullets to ensure you ask for the proper operations):
  • OAuth: Get Client Credentials Using CURL
  • OAuth: Get Client Credentials Using Postman

Whichever option you choose, you will need to ask for the correct operation permissions. In this quick start you will need both video and ingest-profile operations. The following can be used with curl or Postman to get the proper permissions:

Retrieve all videos

One of the basic tasks you will want to perform is retrieve all videos in an account. This is done using an HTTP GET method.

  1. You will now use the proxy in conjunction with the tester.html page to retrieve information on all the videos in your account, returned as a JSON object. Browse to tester.html and complete the form as follows:
Form Field Value
Client id Returned in your client credentials
Client secret Returned in your client credentials
Call type GET
Call body
Full request URL for the API call https://cms.api.brightcove.com/v1beta1/accounts/**YOUR_ACCOUNT_ID**/videos

After you click the Submit button, the text area following the button will be populated with a JSON object with information on all videos in your account, as shown in the following screenshot.

Using reference IDs

In the samples that follow the video ID is used in the endpoint URL. You can also use the video's reference ID if you wish. If you use the reference ID you must prepend that identifier with ref:. For example, the first URL uses the video ID, whereas the second uses the reference ID.

Retrieve one video

You will retrieve a specific videos from your account. This is done using an HTTP GET method and append a video ID to the end of the videos endpoint.

  1. You will now use the proxy in conjunction with the tester.html page to retrieve information on all the videos in your account, returned as a JSON object. Browse to tester.html and complete the form as follows:
Form Field Value
Client id Returned in your client credentials

Call type GET

Retrieve rendition info

  1. You will now retrieve rendition information for a specific video. You will again use the HTTP GET method, with a video ID and the sources endpoint added.
Form Field Value
Call type GET
Call body
Full request URL for the API call https://cms.api.brightcove.com/v1beta1/accounts/**YOUR_ACCOUNT_ID**/videos/**VIDEO_ID**/sources

The results from the call follow. If you have universal delivery service (UDS) enabled, you will see each rendition is represented by a pair of JSON objects. The first one for streaming and the second for download. You need to look carefully at JSON object's properties, for instance encoding_rate, to see that different renditions are being reported.

Create video

  1. Create a video using the HTTP POST method. This video object will eventually contain the master copy of the video, metadata, possible renditions and other information. For the first time you will supply information information for the call body. In this case, it will be a simple JSON object specifying the name of the video to be created. To see the values that can be set during the POST, see the CMS API Overview Setting values section.

Note: When filling out the form:

  • Be sure the body contains no spaces or carriage returns
  • Realize the name you supply will be the video name you see in Video Cloud Studio
Form Field Value
Call type POST
Call body {"name":"**YOUR_VIDEO_NAME**"}
Full request URL for the API call https://cms.api.brightcove.com/v1beta1/accounts/**YOUR_ACCOUNT_ID**/videos

The results from the call follow.

You can confirm the video was created by browsing to Video Cloud Studio, open the Media Module, and a video with the name supplied will be present.

Search by keywords

Note: The CMS API has a rich searching implementation which you can leverage. Not only does the implementation provide many ways to search, but you can also manipulate the display of results by sorting, limiting result set size, etc. A few examples will be shown in this quick start. For complete documentation see the CMS API Overview Search section.

  1. You will now search your videos for a certain keyword. These keywords can be set via API, or in Video Cloud Studio where they are referred to as tags. In this case, videos with a keyword of wildlife will retrieved. To do this add a query string to the endpoint, then a q:, and finally the syntax to look for the specific keyword.
Form Field Value
Call type GET
Call body
Full request URL for the API call https://cms.api.brightcove.com/v1beta1/accounts/**YOUR_ACCOUNT_ID**/videos/**VIDEO_ID**?q=keywords:wildlife
