CMS API Video Fields Reference

This is a reference to video fields returned by the CMS API.

Primary fields

See the following links to see which fields can be set when you are creating or updating a video:

  • Fields that can be set when creating a video
  • Fields that can be set when updating a video
Table 1: Primary Fields
Field Type Read-only Default Description
id string yes none the video id
account_id string yes none the id of the account
complete boolean yes false whether there is at least one playable rendition of the video
created_at ISO 8601 date-time string yes none date-time video was added to the account; example: "2014-12-09T06:07:11.877Z"
published_at ISO 8601 date-time string yes none date-time video was added to the account; example: "2014-12-09T06:07:11.877Z"
cue_points array of cue_point objects no empty array markers for midroll ad requests or some other action to be created via the player API
custom_fields object no empty object map of custom field name:value pairs; only fields that have values are included
description string no null the short description of the video - 250 single-byte characters maximum
duration number yes null length of the video in milliseconds
economics string no AD_SUPPORTED indicates whether ad requests are permitted for the video
geo geo object no see geo object if geo-restriction is enabled for the account[1-3], this object will contain geo-restriction properties for the video
images map of image objects[1-4] no empty object
link link object no null a related link
long_description string no null 5000 single-byte characters maximum
name string no none video title - required field (256 single-byte characters maximum)
reference_id string no null any value that is unique within the account (150 single-byte characters maximum)
schedule schedule object no see schedule object When video becomes available/unavailable[1-1]
sharing sharing object no see sharing object information about the account the video was shared from or to
state string no[1-5] none current status of the video: ACTIVE | INACTIVE | PENDING | DELETED
tags array no empty array array of tags (strings) added to the video (128 single-byte characters per tag maximum)
text_tracks array of text_track objects no empty array data for WebVTT captions associated with the video
updated_at ISO 8601 date-time string yes none date-time video was last modified;[1-2] example: "2015-01-13T17:45:21.977Z"

Notes

  • [1-1] Note that that a scheduled video is not inactive outside the period of scheduled availability
  • [1-2] The modification may have been made by a user or the Video Cloud system
  • [1-3] Geo-restriction is a paid add-on to Video Cloud; if you want to purchase geo-restriction capability, contact your Account Manager
  • [1-4]

    Currently the only supported image objects are "poster" and "thumbnail".

    images will have two properties: images.thumbnail and images.poster - each of these is an image object.

    Note that you can not add remote images by updating the src attribute. Remote image files must be ingested via the Dynamic Ingest API. The functionality to add images does not yet exist, but will be added to the Dynamic Ingest API.

  • [1-5] state can be set to ACTIVE or INACTIVE, but not to PENDING or DELETED.

    Table 2: Cue Point Object Fields
    Field Type Read-only Default Description
    id string yes none the cue point id
    name string no null optional name for the cue point (128 single-byte characters maximum)
    type string no AD AD or CODE
    time number no none time of the cue point in seconds; example: 10.527
    metadata string no null optional metadata string (128 single-byte characters maximum)
    force-stop boolean no false whether video is force-stopped at the cue point

    Field Type Read-only Default Description
    countries array of strings no null ISO 3166 list of 2-letter codes for countries that will be whitelisted or blacklisted for viewing the video
    exclude_countries boolean no false if true, country array is treated as a list of countries excluded from viewing
    restricted boolean no false whether geo-restriction is enabled for thie video

    Table 7: Schedule Object Fields
    Field Type Read-only Default Description
    ends_at ISO 8601 date-time string[7-1] no null date-time when video will become unavailable

    • sharing object

    Table 8: Sharing Object Fields
    Field Type Read-only Default Description
    by_external_acct boolean yes false whether the video was shared from another account
    by_id string yes null id of the account that shared the video - note that this field is populated only for the shared copy, not for the original video
    source_id string yes null id of the video in its original account - note that this field is populated only for the shared copy, not for the original video
    to_external_acct boolean yes false whether the video is shared to another account
    by_reference boolean yes false whether the video is shared by reference[8-1]

    Notes

  • [8-1] When a video is shared by reference, replacing or retranscoding the video in the original account will automatically update the renditions in the shared-to account

    • text_track object

    Table 9: Text Track Fields
    Field Type Read-only Default Description
    src string (URL) no null URL where the .vtt file resides
    srclang ISO-639-1 language code no null language for the captions
    label string no null The label to be used in the player menu - it should be in the same language as the captions
    kind string no null "captions" is the only value currently supported
    mime-type strings no null The mime-type of the captions[9-1]
    default boolean no false Setting this to true makes this the default captions file in the player menu

    Notes

  • [9-1] Setting the mime-type is optional, but the captions should be UTF-8 encoded and have a mime-type of "text/vtt"
