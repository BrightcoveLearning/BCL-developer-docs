Code Sample: Add Cue Point

In this sample, you will learn how to add cue point to a video programmatically using the CMS API.

Introduction

Cue points are specified places on the video timeline. When the video is played back, an event is automatically broadcast every time one of the cue point is reached - using the Player API, you can listen for these events and take whatever action you wish. In addition, you can read all the cuepoint information from the video metadata stored in the player mediainfo object, which allows you to mark the cue point on the video timeline in some way if you want to.

There are two kinds of cue point:

  • Ad cue point: these are used to mark places where an ad request should be made (if the video and player are enabled for advertising).
  • Code cue point: these are used to take some action using the Player API - some typical uses are displaying some kind of call-to-action (CTA) or displaying some image or text relating to video content, perhaps as a video overlay or in the page around the player.

Implementation overview

You can add cue points in the Studio Media module. You can also add them using the CMS API, which the method we will use in this sample.

HTTP method

Cue points cannot be added when you create the video. Instead, they must be added by making a video update request, which means you will use the PATCH method.

Endpoint

The endpoint for the update video request is:

https://cms.api.brightcove.com/v1/accounts/account_id/videos/video_id

Request body

The request body is a JSON object that can contain many items of video metadata. Here we will just look at the cue_points field, which is set to an array of cue point objects, as in the following sample:

{
"cue_points": [
    {
      "name": "Ad Cue Point",
      "type": "AD",
      "time": 3.0,
      "metadata": null,
      "force_stop": false
    },
    {
      "name": "Code Cue Point",
      "type": "CODE",
      "time": 5.0,
      "metadata": "The metadata might be text or the url for an image to display",
      "force_stop": false
    }
  ]
}
