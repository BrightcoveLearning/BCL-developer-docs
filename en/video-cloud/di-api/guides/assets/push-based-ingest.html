Push-Based Ingestion

In this topic, you will learn how to add videos to your Video Cloud account using the Dynamic Ingest API with push-based ingest. Push-based ingest is provided for those who cannot make their videos available on an HTTP(S) URL or an S3 URI for Dynamic Ingest to pull.

Introduction

For push-based ingestion, Brightcove provides an S3 bucket that you can upload your videos to, and Dynamic Ingest then pulls the video from S3 bucket in same way it would from your own S3 bucket or URL. The diagram below shows the difference between pull-based and push-based logic.

Authentication

Authentication for Brightcove API requests for push-based ingestion are the same as for pull-based ingestion. See Dynamic Ingest API Overview: Operations for details.

API requests

There are four API requests involved in push-based ingestion:

  1. CMS API request to create the video object in Video Cloud (same as for pull-based ingestion)
  2. Dynamic Ingest request to get the Brightcove S3 bucket URLs
  3. PUT request to upload the video to the Brightcove S3 bucket
  4. Dynamic Ingest request to ingest the video (same as for pull-based ingestion)

Since the first and last requests are no different from any other Dynamic Ingest operation, only the second and third requests are detailed in the sections that follow.
