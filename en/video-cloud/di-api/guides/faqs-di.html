Dynamic Ingest API FAQs

FAQs

What are the limits?

  • API Request: 20 per minute
  • Number of jobs processing simultaneously: 100
  • File size: 100GB
  • If you are ingesting files in batches, limit batches to 100 and wait for some of the previous jobs to complete processing before continuing.

How fast is Dynamic Ingest?

The speed depends on several factors, the most important of which are how quickly Dynamic Ingest can download the file from your storage location and how fast CDNs can handle Brightcove's output for storage. However, Dynamic Ingest is a horizontally scalable, cloud transcoding solution that processes all of your content in parallel. Because of this, we generally see transcoding times significantly faster than those for older ingest methods like the Media API and Batch Provisioning.

What region is Dynamic Ingest located in?

Currently, Dynamic Ingest workers are running in the AWS US East region. Any content hosted in S3 buckets in that region can be downloaded with no egress charge. Content hosted on other cloud providers or in other regions may incur an egress cost with your cloud provider. In the future, we plan to enable this technology to run in any of the current AWS regions.

Is DRM supported?

Yes. The following DRM types are available via DI:

  • Adobe Access (FAXS) for MP4 streaming
  • HDS-FAXS (Adobe Access for HD)
  • Widevine (for mobile devices)
  • DASH with Common Encryption (for HBBTV)
  • Smooth with Playready

Why pull files instead of uploading?

Legacy Video Cloud Ingest methods all supported some form of push - Media API, Batch FTP, and Aspera. Uploading works well for certain use cases, but pull has significant advantages. Since a job request consists only of a URL to the file, any file corruption or transfer issues can be solved by automatically retrying the job. Downloading files allows Brightcove to horizontally scale the platform - instead of funneling all traffic through an FTP or HTTP post server, downloads can occur on the same worker that performs the transcoding. Pull provides more flexibility, reliability, and speed.

Is Dynamic Ingest secure?

Security is a paramount concern for Brightcove. Dynamic Ingest supports secure transport methods - S3, and HTTPs. Once in the cloud, if the decision is made not to store the master, the master file is hosted on transcoding workers and deleted immediately after the transcode is completed. File names are hashed and never revealed. The same security policies maintained by Zencoder are used for the transcode process, and in over 4 years of operations, no file has ever been compromised on the Zencoder platform.

Can I integrate Dynamic Ingest with my existing workflow?

Dynamic Ingest is designed to work alongside the other existing ingest methods. Media API, Batch FTP, and Aspera ingest will all work as expected alongside Dynamic Ingest. You can start implementing Dynamic Ingest without disrupting your current workflow until the new integration is fully tested.

How long does it take to integrate with Dynamic Ingest?

Implementation time will vary according to complexity of your system, but Dynamic Ingest is a standard REST api returning JSON and using OAuth2 for authentication. If you are familiar with these widely used technologies, integration should be straightforward.

What if our data center doesn't allow external access?

There are a number of ways to deal with this. Dynamic Ingest supports secure transfer methods, so it’s possible to open a single, secure entry point into the data center. It’s also possible to build a workflow that pushes content to a cloud storage provider, triggers the ingest, then deletes the content.

How much does Dynamic Ingest cost?

Dynamic Ingest billing is the same as existing methods.
