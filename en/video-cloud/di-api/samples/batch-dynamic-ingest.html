Code Sample: Ingesting Multiple Videos Using Dynamic Ingest

In this sample, you will learn how to ingest multiple videos using the Dynamic Ingest API. To get access tokens and make the API calls, it uses a proxy built by Brightcove Learning services.

Dynamic Ingest does not have a mechanism for processing multiple videos - this sample simply executes a loop to make a Dynamic Ingest request for each video.

Getting Credentials

To get a client_id and client_secret, you will need to go to the OAuth UI and register this app:

  • https://studio.brightcove.com/products/videocloud/admin/oauthsettings

You can also get your credentials via CURL or Postman - see:

  • Get Client Credentials Using CURL
  • Get Client Credentials Using Postman

Rate limiting

No more than 100 concurrent Dynamic Ingest jobs are allowed. The best way to avoid violating this rule is to create an app that receives notifications from the Dynamic Ingest system as well as submitting ingest requests, so that you can keep track of how many jobs are running and how many have completed - see Fallback / Retry Strategy. This sample implements rudimentary rate limiting by pausing for 30 minutes after submitting 100 requests.
