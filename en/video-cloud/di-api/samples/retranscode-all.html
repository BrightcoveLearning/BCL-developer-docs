Code Sample: Re-transcoding All Videos Using Dynamic Ingest

In this sample, you will learn how to retranscode all videos in an account using the Dynamic Ingest API. All videos will remain playable with existing renditions until retranscoding is complete.

Dynamic Ingest does not have a mechanism for processing multiple videos - this sample simply executes a loop to make a Dynamic Ingest request for each video.

Authentication

This section will show you how to get the client credentials you need for this app if you are going to use a standard profile. If you want the app to retrieve custom ingest profiles from your account, see the advanced authentication section.

To get a client_id and client_secret, you will need to go to the OAuth UI and register this app:

https://studio.brightcove.com/products/videocloud/admin/oauthsettings

Advanced authentication

If you want this app to retrieve ingest profiles from your account so that you can retranscode to a custom profile, you need client credentials with expanded permissions that are not currently available use the Studio admin.

To get expanded credentials, you will need to get them directly from the Brightcove OAuth API using cURL or an HTTP client such as Postman. Whichever method you use, you will need to obtain the BC_TOKEN cookie from your browser while you are logged into Studio.

.

Get your BC_TOKEN

Steps

  1. Login to Studio, and open your browser's developer console. For most browsers, you can open the console by pressing OPTION-COMMAND-i (Mac) or ALT-CTRL-i (Windows)
  2. In the console, paste the following JavaScript and press return (Mac) or Enter (Windows): 
    var cookiesArray = document.cookie.split(";"), cookiesObj = {}, i, tmpArray = [];
for (i = 0; i < cookiesArray.length; i++) {
    tmpArray = cookiesArray[i].split("=");
    if (tmpArray[0].indexOf('BC_TOKEN') > -1) {
        cookiesObj.BC_TOKEN = tmpArray[1];
    }
}
window.prompt("BC_TOKEN:", cookiesObj.BC_TOKEN);
  3. Save the cookie value for the next step
    If for some reason the script above does not work, simply locate the BC_TOKEN cookie in your cookies, and copy and save the value.

Once you have the BC_TOKEN, you can get your credentials using either cURL or Postman.

Get credentials using cURL

If you have a Mac (or other Linux/Unix system), cURL is built into the operating system, and you can access it by opening the Terminal app. If you are on Windows and have not installed cURL, see Using cURL for details on how to obtain and install it.

Steps

  1. Copy the code below into a text editor 
    curl \
  --include \
  --header "Authorization: BC_TOKEN YOUR_BC_TOKEN" \
  --data 'name=DIcredentials \
    name&maximum_scope=[
    {
      "identity": {
        "type": "video-cloud-account",
        "account-id": YOUR_ACCOUNT_ID
      },
      "operations": [
        "video-cloud/upload-urls/read",
        "video-cloud/video/create",
        "video-cloud/video/read",
        "video-cloud/video/update",
        "video-cloud/ingest-profiles/profile/write",
        "video-cloud/ingest-profiles/account/write",
        "video-cloud/ingest-profiles/profile/read",
        "video-cloud/ingest-profiles/account/read"
      ]
    }
  ]' \
https://oauth.brightcove.com/v4/client_credentials
  2. Replace the placeholders YOUR_BC_TOKEN and YOUR_ACCOUNT_ID with values for your account
    Be sure that you keep the closing " after the BC_TOKEN!
  3. Copy the edited code, paste it in Terminal (Mac) or Command Prompt (Windows) and press return/enter.
  4. Locate the client_id and client_secret in the JSON response and save them (there is no way to retrieve the client_secret from the API again, so if you lose it, you will need to generate a new set of credentials)

Get credentials using Postman

Note that the steps below assume you are generally familiar with using Postman. If you are new to it, you may want to read Use Postman for HTTP Requests before proceeding.

Steps

  1. Open Postman, and set up a new POST request to: 
    https://oauth.brightcove.com/v4/client_credentials
  2. Under Headers, add two headers:
    • Authorization: BC_TOKEN YOUR_BC_TOKEN
    • Content-type: application/json
  3. In the Body, select raw and enter the following code (with your account id inserted where indicated): 
    {
  "type": "credential",
  "maximum_scope": [
    {
      "identity": {
        "type": "video-cloud-account",
        "account-id": YOUR_ACCOUNT_ID
      },
      "operations": [
        "video-cloud/upload-urls/read",
        "video-cloud/video/create",
        "video-cloud/video/read",
        "video-cloud/video/update",
        "video-cloud/ingest-profiles/profile/write",
        "video-cloud/ingest-profiles/account/write",
        "video-cloud/ingest-profiles/profile/read",
        "video-cloud/ingest-profiles/account/read"
      ]
    }
  ],
  "name": "Source File Upload Credentials"
}
  4. Click Send
  5. Locate the client_id and client_secret in the JSON response and save them (there is no way to retrieve the client_secret from the API again, so if you lose it, you will need to generate a new set of credentials)

Rate limiting

No more than 100 concurrent Dynamic Ingest jobs are allowed. This sample implements rudimentary rate limiting by pausing for 30 minutes after submitting 100 requests.
