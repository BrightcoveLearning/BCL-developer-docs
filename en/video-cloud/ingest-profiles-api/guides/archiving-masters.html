Archiving Masters

If you archive a master when you ingest a video, you will be able to re-transcode the video later without ingesting it again. This topic explains your options for archiving your masters.

Archiving a master

To archive a master for your video, you need to include the following code in your ingest profile:

"archive_master": true

Source or rendition

You can choose to use your video source file as the master, or you can select a high-quality rendition to use as the master.

To use the video source file you ingest as the master, include the following code in your ingest profile:

"digital_master": {
    "rendition": "passthrough"
}

To use a rendition for the master instead of your source file, set the rendition field of the digital_master object to the reference id for the rendition:

"digital_master": {
	    "rendition": "av0"
	},
"keep_renditions": true,
"name": "nondrm-av0",
"packages": [],
"renditions": [
    {
        "media_type": "video",
        "reference_id": "av0", ...

If you set rendition to none, no master will be archived.

Distribution

You can choose whether the master is distributed or not. Distributed means that the master is pushed to the CDN and available as a playable rendition. If the master is not distributed, it is archived by Video Cloud but not pushed to the CDN.

Note: if you selected one of your renditions to be the master, and set distribution to false, that rendition will not be pushed to the CDN and will not be available for playback.
