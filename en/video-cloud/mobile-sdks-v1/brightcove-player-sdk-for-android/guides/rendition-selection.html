Rendition Selection

For apps using the Brightcove Native Player SDK for Android, there is a default rendition selection process when playing a video.

When you retrieve videos from the Brightcove catalog, the player has to decide which source (rendition) to play back. This is implemented in the DefaultSourceSelectionController component.

Brightcove provides a default implementation as follows:

  1. It looks for the first source that has a deliveryType of HLS.
  2. If none is found, it will fall back to the first source with a deliveryType of MP4.
  3. The source URL is provided to the MediaController for playback.
  4. When playback begins, the MediaController is responsible for selecting which bit rate and resolution to serve.
  5. The MediaController will start at the first item in the index and adjust accordingly.

When using the default transcode settings, the first HLS source should be the master m3u8 playlist that points to all of your individual renditions.

Overriding source

If you want to override which rendition gets provided to the MediaController, you can listen for the SELECT_SOURCE event and respond with the desired source (rendition).

There are some exceptions to this:

  • When using the default player, which is based on Android's MediaPlayer, rendition selection is not exposed by the OS, so there is no way to modify it.
  • When using the HLS player, which is based on VisualOn, you can use the VisualOn API's to select a rendition.
