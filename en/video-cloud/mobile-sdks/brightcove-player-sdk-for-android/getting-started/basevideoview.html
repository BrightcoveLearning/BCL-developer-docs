Understanding BaseVideoView

The BaseVideoView class provides the basic functionality for the video view, and its subclasses do the following:

  • They are designed to be dropped into your XML Layout file to replace Android's VideoView widget.
  • They provide public methods to mirror those from Android's VideoView widget.
  • They provide additional logic for playlists.

BaseVideoView subclasses

The BaseVideoView subclasses can be used outside of a BrightcovePlayer or BrightcovePlayerFragment, but require life cycle management and full screen support to be implemented by the developer.

The subclasses for the BaseVideoView can be divided into the following categories:

Basic video playback

  • BrightcoveVideoView

    This class is designed to replace Android's native VideoView. It provides public methods to mirror that of VideoView and is a Java collection-like API for managing the internal playlist. It uses the Event workflow and will emit events related to video playback.

    This class is designed to work with Android's MediaController class to provide a controller for video playback that matches the device and OS version.

HLS video playback

  • BrightcoveExoPlayerVideoView

    This class provides improved support for HLS playback by leveraging Google's ExoPlayer library.

    It is recommended to use this class for HLS playback. For more details, see the HLS Playback with the Player SDK document.

  • SeamlessVideoView

    This class provides support for HLS and seamlessly stitching videos together, but it does have the added cost and complexity of using the VisualOn plugin.

    The SeamlessVideoView is an add-on to the Player SDK for Android. Because this leverages a VisualOn player, it does require additional native libraries to be included in your app and has additional costs. Please contact your Brightcove account manager for pricing details.

Video playback with animation

  • BrightcoveTextureVideoView

    This class uses the TextureView for rendering. This view is used if you want to support animation, like having the video play while it is opening up from a thumbnail.

For more details about these classes, refer to the Brightcove Player SDK for Android Reference documents.
