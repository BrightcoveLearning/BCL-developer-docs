Managing Activity Lifecycle

As users interact with your Android app, the OS invokes callback methods in an Activity instance that correspond with different stages of its lifecycle.

If you cannot, or choose not to use the BrightcovePlayer, you will need to manage an Activity's lifecycle yourself.

Note: If you are using the BrightcovePlayer, you do not need to manage the lifecycle manually, and you can skip this topic.

If you are unfamiliar with Android's Activity lifecycle, get started with the Managing the Activity Lifecycle reference.

If you want to override which rendition gets provided to the MediaController, you can listen for the SELECT_SOURCE event and respond with the desired source (rendition).

There are some exceptions to this:

  • When using the default player, which is based on Android's MediaPlayer, rendition selection is not exposed by the OS, so there is no way to modify it.
  • When using the HLS player, which is based on VisualOn, you can use the VisualOn API's to select a rendition.
