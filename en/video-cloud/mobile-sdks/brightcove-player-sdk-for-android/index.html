Brightcove Player SDK for Android

Welcome to the Android SDK developer documentation. Here you will find the information needed to develop with the Android SDK.

At Brightcove, we strive to keep the documentation structured, clean and intuitive. As such, we leverage github READMEs for details on all our plugins and integrations. For your convenience, we've listed and linked to these plugins below.

Supported versions

Brightcove provides active support for the latest Android SDK on the following Android versions:

  • 5.0
  • 4.4
  • 4.3
  • 4.2
  • 4.1
  • 4.0
  • 2.3.3

Support is defined in terms of the [major version].[minor version] of an OS (for example: Android 4.1). There are two levels of support

  • Active: Brightcove will test and fix bugs on these OS versions.
  • Passive: Brightcove will test on these OS versions, but bug fixes will be handled on a case by case basis.

Generally, Brightcove provides active support for the versions of the OS that are used by 80% of the user base, and passive support for other versions used by at least 5% of the user base. Only the latest minor revision of a version is supported.

Core SDK installation

Installation of the core SDK is very straightforward.

  1. Simply download the SDK and unzip it. You will find the SDK archive file contains six other zip files.
  2. Unzip the android-sdk-4.x.x.zip file.
  3. From this archive add the android-sdk-4.x.x.jar file to your library.

SDK overview

The Brightcove Native SDK for Android simplifies integration of video playback with Brightcove Video Cloud and many popular advertising, analytics, and DRM solutions. It also provides closed captioning support for pre-KitKat devices and easily customizable player controls with similar look and feel to the Brightcove Web Player to provide a consistent user experience across platforms.

The BaseVideoView subclasses are intended to be drop-in replacements for Android's VideoView widget. They are:

  • BrightcoveVideoView

    • This uses the Android MediaPlayer, similar to Android's VideoView.

  • BrightcoveTextureVideoView

    • This uses the TextureView for rendering.

  • SeamlessVideoView

    • Also known as the Brightcove HLS Player, the SeamlessVideoView provides greatly improved support for HLS and seamlessly stitching videos together. This is where it gets it's name. The SeamlessVideoView internally uses a VisualOn player, which requires additional native libraries to be included in your app and has additional costs. Please contact your Brightcove account manager for pricing details.

  • BrightcoveExoPlayerVideoView

    • This uses Google's Exo Player and the SurfaceView for rendering.

Analytics

If you use the Brightcove player and the Catalog class included in the SDK to retrieve and play videos, analytics data will be automatically collected and will appear in your Video Cloud Analytics.

When not using the Catalog to create Video or Playlist objects for playback, the Analytics component's account property must be manually set.

For details about sending data to Video Cloud Analytics, see the Using Brightcove Analytics guide.

When using the HLS Player (SeamlessVideoView) on Amazon devices, which include a VisualOn license, Brightcove Analytics adjusts accordingly to avoid double billing.

Geo Restrictions

When you use Brightcove's CMS API to add geographic filtering properties to your videos, you can control which countries they can (or cannot) be viewed in. The Brightcove Player SDK for Android supports geo-restricted videos. Learn how to add geo-filtering using the CMS API.

SDK components

Below is a list of important architectural components to help you understand the SDK.

Brightcove Player

The BrightcovePlayer class is a basic Activity, which wraps a BrightcoveVideoView and sets up an EventLogger and default MediaController. It implements basic lifecycle management and full screen logic. It is meant to be subclassed.

At a minimum the onCreate() method should be overridden with logic to wire up the BrightcoveVideoView variable to the layout.xml file. For example:

brightcoveVideoView = (BrightcoveVideoView) findViewById(R.id.bc_video_view);

BrightcoveVideoView

The BaseVideoView subclasses, like BrightcoveVideoView, can be used outside a BrightcovePlayer or BrightcovePlayerFragment, but require life cycle management and full screen support to be implemented by the developer.

The BrightcoveVideoView class is designed to replace Android's native VideoView. It provides public methods to mirror that of VideoView and is a Java collection-like API for managing the internal playlist. It uses the Event workflow and will emit events related to video playback.

The BrightcoveVideoView class is designed to work with Android's MediaController class to provide a controller for video playback that matches the device and OS version.

Events

Events in the Brightcove Player SDK for Android represent a state or action that can be passed down through a chain of listeners. All events have a type and a map of properties associated with them. Events can be stopped from propagating down the chain of listeners, or can be instructed to prevent their default listeners in the video playback classes.

Events can represent state or an action occurring. For example, the event type DID_SEEK_TO represents the state of the action SEEK_TO.

EventEmitter

The EventEmitter is the public facing API for classes that wish to emit and listen to events. The event emitter can be obtained from the BrightcoveVideoView. It is also possible to create your own event emitter and instruct the BrightcoveVideoView to use it. This is useful if you want control of the chain of listeners.

Components

Components in the Brightcove Player SDK for Android are objects that can use events and the event emitter system. Using a subclass of AbstractComponent will allow you to quickly create a component that listens for and emits events. As a component, you must describe in annotations which events the component will listen for and emit.

Catalog

The Catalog provides access to Video Cloud videos and playlists. This component's methods allow retrieval of videos and playlists by ID and/or ReferenceID. The catalog will make the URL requests and parse the returned data.

Note: Access to the Video Cloud videos and playlists are not available to Brightcove Perform customers.
