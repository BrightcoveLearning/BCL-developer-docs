Brightcove Player SDK for Android Overview

The Brightcove Native SDK for Android simplifies integration of video playback with Brightcove Video Cloud and many popular advertising, analytics, and DRM solutions. It also provides closed captioning support for pre-KitKat devices and easily customizable player controls with similar look and feel to the Brightcove Web Player to provide a consistent user experience across platforms.

Understanding BaseVideoView

The BaseVideoView class provides the basic functionality for the video view, and its subclasses do the following:

  • They are designed to be dropped into your XML Layout file to replace Android's VideoView widget.
  • They provide public methods to mirror those from Android's VideoView widget.
  • They provide additional logic for playlists.

The subclasses for the BaseVideoView can be divided into the following categories:

Basic video playback

  • BrightcoveVideoView

    This uses the Android MediaPlayer, similar to Android's VideoView. This is the view you will want to use for basic video playback.

HLS video playback

  • BrightcoveExoPlayerVideoView

    This uses Google's ExoPlayer library, and is a good choice for playing HLS videos.

  • SeamlessVideoView

    This provides greatly improved support for HLS and seamlessly stitching videos together. This is where it gets it's name.

    The SeamlessVideoView is an add-on to the Player SDK for Android. Because this leverages a VisualOn player, it does require additional native libraries to be included in your app and has additional costs. Please contact your Brightcove account manager for pricing details.

Video playback with animation

  • BrightcoveTextureVideoView

    This uses the TextureView for rendering. This view is used if you want to support animation, like having the video play while it is opening up from a thumbnail.

For more details about these classes, refer to the Brightcove Player SDK for Android Reference documents.

Analytics

If you use the Brightcove player and the Catalog class included in the SDK to retrieve and play videos, analytics data will be automatically collected and will appear in your Video Cloud Analytics.

When not using the Catalog to create Video or Playlist objects for playback, the Analytics component's account property must be manually set.

For details about sending data to Video Cloud Analytics, see the Using Brightcove Analytics guide.

When using the HLS Player (SeamlessVideoView) on Amazon devices, which include a VisualOn license, Brightcove Analytics adjusts accordingly to avoid double billing.

Geo Restrictions

When you use Brightcove's CMS API to add geographic filtering properties to your videos, you can control which countries they can (or cannot) be viewed in. The Brightcove Player SDK for Android supports geo-restricted videos. Learn how to add geo-filtering using the CMS API.
