Working with App Transport Security (ATS)

In this topic, you will learn how to work with App Transport Security.

As of January 1, 2017, all App Store apps are required to use App Transport Security. You will no longer be able to disable ATS for App Store apps. For details, see the Apple will require HTTPS connections for iOS apps by the end of 2016 document.

With Apple's release of the iOS 9 SDK, a new feature called App Transport Security (ATS) enforces secure connections between an app and web services.

Because ATS restrictions do not apply for older versions of iOS, it will only affect applications targeting the iOS 9.0 SDK and later. Applications targeting early versions of the iOS SDK and those already in the App Store will not be affected.

Support for ATS within the Brightcove ecosystem is now available. You can either:

  • Build your apps with ATS enabled
  • Disable ATS requirements

Build Apps with ATS Enabled

The following steps need to be in place for ATS to work:

  1. Use secure communication to deliver your media content.

    For complete specifications, see the Requirements for Connecting Using ATS document.

    • Video Cloud customers (whether you are using the house CDN or a named CDN) should contact their Brightcove account managers to ensure accounts are configured to deliver over HTTPS.

    • For those using remote assets, you may need to configure your CDN appropriately.

  2. In your Xcode projects, update the source selection policy to use HTTPS.

    Here is a code example:

    - (void)setup
{
  BCOVPlayerSDKManager *manager = [BCOVPlayerSDKManager sharedManager];

  BCOVBasicSessionProviderOptions *options = [[BCOVBasicSessionProviderOptions alloc] init];
  options.sourceSelectionPolicy = [BCOVBasicSourceSelectionPolicy sourceSelectionHLSWithScheme:kBCOVSourceURLSchemeHTTPS];
  id<BCOVPlaybackSessionProvider> provider = [manager createBasicSessionProviderWithOptions:options];
  _playbackController = [manager createPlaybackControllerWithSessionProvider:provider viewStrategy:[manager defaultControlsViewStrategy]];

  _playbackController.analytics.account = kViewControllerAccountID;

  _playbackController.delegate = self;
  _playbackController.autoAdvance = YES;
  _playbackController.autoPlay = YES;

  _service = [[BCOVPlaybackService alloc] initWithAccountId:kViewControllerAccountID policyKey:kViewControllerPlaybackServicePolicyKey];
}

    For details, see the Source Selection section of the Brightcove Player SDK for iOS reference document.

    Brightcove recommends using the BCOVPlaybackService class to retrieve your media content from the Playback API. This is the latest and recommended API to use to retrieve your video and playlist data from your Video Cloud library. For details, see the Playback API Documentation.

Disable ATS requirements

When building applications that target the iOS 9 SDK or later, you may choose to disable ATS. Follow these steps to disable ATS requirements in your app:

  1. In Xcode, right click on the Info.plist file and open it as Source Code.

  2. Copy the following code: 

    <key>NSAppTransportSecurity</key>
<dict>
    <key>NSAllowsArbitraryLoads</key>
    <true/>
</dict>

  3. In the Info.plist file, before the closing </dict> tag, paste the code from above.

  4. In Xcode, when you open the Info.plist file as Property List, you should now see this entry in the plist dictionary.

Disabling ATS does not alter existing HTTPS connections. It simply prevents iOS from enforcing additional security requirements on your http communication.

You may also try whitelisting certain domains to disable ATS requirements. For details, see the NSAppTransportSecurity document.
