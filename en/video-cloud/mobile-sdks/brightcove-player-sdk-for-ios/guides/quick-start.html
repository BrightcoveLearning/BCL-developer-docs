Simple Video Playback App

This tutorial will familiarize you with the basics of working with the Brightcove Player SDK for iOS by walking you through the development of a simple video playback app.

This example uses videos from a playlist returned from the Brightcove catalog service.

This example plays videos from an internal array of video files.

There are two ways to try out this sample.

Follow the steps below to get familiar with setting up an app project that uses the Brightcove SDK.

  1. Follow the steps below to get familiar with setting up an app project that uses the Brightcove SDK.
    • You can view the entire code for the AppDelegate implementation.
    • You can view the entire code for the View Controller header.
    • You can also view the entire code for the View Controller implementation.
  2. Download the entire Xcode project for experimentation.
    1. Download the VideoCloudBasicPlayer sample to your local system.
    2. Run the pod install command.
    3. Open and run the newly created workspace.

This example is written in Objective C, but you can also experiment with a sample written in Swift. See the Swift version of the VideoCloudBasicPlayer sample.

Create the project

Set up a project in Xcode. Then add the SDK along with any dependencies to the project.

Set up the Xcode project

Create a new Xcode project for the app.

  1. Start the creation of a new iOS project in Xcode. Select Create a new Xcode project.

  2. Choose Single View Application for the template and select Next.

  3. Set the project information as follows:

    • Product Name: Simple Video Playback
    • Organization Name: YourCompanyName
    • Company Identifier: com.example-company

  4. Choose the location where you want to save your project and create it.

  5. Now close the project (yes, close it — this is important!)

Add the SDK and its dependencies to your project

The easiest way to add the SDK and its dependencies to your project is to use CocoaPods, by including ReactiveCocoa. If you do not have CocoaPods installed, see the installation instructions on the CocoaPods site.

Using CocoaPods (the pod tool) on an existing Xcode project will modify the project file. You may want to make a backup before doing this.

  1. In your project folder, create a plain text file called Podfile (no file extension).

  2. Using a text editor, add the following lines of code to the Podfile and save it. This code does the following:

    • Defines the iOS platform version.
    • Installs the current version of the Brightcove player SDK.

    platform :ios, '7.1' # (or whatever version you want to target)

pod 'Brightcove-Player-SDK'

  3. Open a Terminal session and navigate to your Simple Video Playback Xcode project folder.

  4. In the Terminal session, type the command

    5. pod install

    and press Return to run the it. You should see a series of messages in the terminal indicating that the ReactiveCocoa library has been integrated.

  5. Notice the last line, which is important — from this point on, you must open the Simple Video Playback.xcworkspace file in Xcode, not the Simple Video Playback.xcodeproj file. The pod install command creates this .xcworkspace file for your project.

If you don't want to use CocoaPods , you can download the latest version of the SDK and manually add it to your project.

Connect the storyboard view

Connect the Main.storyboard view with thevideoContainer property.

  1. In the Assistant Editor, open the Main.storyboard.

  2. In the companion view, expand the View Controller Scene and then the View Controller menu to expose the View object.

  3. Open the ViewController.m file in the Assistant Editor.
