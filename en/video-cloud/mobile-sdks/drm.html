Content Security (DRM) with the Native Player SDKs

Digital rights management (DRM) protects your video content by encrypting the video data and unlocking it based on license policies.

The Brightcove Native Player SDK platforms support the following DRM solutions:

Platform DRM Solution
Native Player SDK for iOS Apple's FairPlay Streaming
Native Player SDK for tvOS Apple's FairPlay Streaming
Native Player SDK for Android Google's Widevine Modular

Note: to use DRM encryption, your account must be enabled for these features. Contact your account manager for more information.

FairPlay

Apple's FairPlay Streaming technology is used with HTTP Live Streaming (HLS) video content.

Video Cloud customers

Create FairPlay renditions as follows:

  1. Review the Reference IDs section of the Content Security document. You will need to assign a reference_id to each rendition so that it can be assigned to a DRM package.
  2. Using Dynamic Ingest, create a FairPlay rendition for you HLS video.

Perform customers

Make sure your video content has FairPlay renditions on an internet accessible URL.

Widevine

Google's Widevine Modular technology is used with MPEG-DASH video content.

Note: Brightcove has deprecated the use of Widevine Classic renditions. Please start using Widevine Modular renditions.

Video Cloud customers

Create Widevine Modular renditions as follows:

  1. Review the Reference IDs section of the Content Security document. You will need to assign a reference_id to each rendition so that it can be assigned to a DRM package.
  2. Using Dynamic Ingest, create a Widevine Modular rendition for you MPEG-DASH video.

Perform customers

Make sure your video content has Widevine Modular renditions on an internet accessible URL.
