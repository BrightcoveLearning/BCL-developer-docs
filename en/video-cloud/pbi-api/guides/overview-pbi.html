Pull-Based Ingest API Overview

Brightcove's Pull-Based Ingest (PBI) API is based on functionality where video source files are downloaded from the customer's storage location. The platform is cloud-centric, globally-distributed and based on modern practices for delivery best in class consistency and speed.

Strengths of the PBI API are:

  • Predictability: Queue time and end-to-end processing time is determinate.
  • Speed: Brightcove's transcoding solution provides superior speed and scalability.
  • Features: Input format support, 99.9% transcoding success rate and a simplified usage workflow are just a few of the API's advantages.
  • Insight: Views and notifications into the progress of the transcoding status are provided.

Workflow overview

Something here with a high-level look at the technology used.

  • CMS API: Retrieve a video ID
  • Zencoder: Transcode the video creating multiple renditions
  • S3: Move the master and renditions to storage based on profile settings
  • Catalogue: Store requisite information associated with the video to the Catalogue
  • Delete: Delete, when needed, the master or all items
  • Re-transcode: Create new renditions when master is present (error if master is not present)
  • Replace: Point to a new master, or replace an existing master
