Quick Start to Pull-Based Ingest

In this quick start you will perform the following tasks:

  • Create a Node.js application to act as your OAuth security proxy. This proxy will use your access token to make calls to the needed APIs.
  • Created a video ID using the CMS API
  • Created a ingest request using the ID returned from the CMS API and a POST method call to the PBI API.
  • Verified the ingest request functioned correctly.

For details on the API used in this document you can view the Pull-Based Ingest API Overview document and the Pull-Based Ingest API reference.

Note: The steps in the document involve curl and other command-line tools that are native to Mac and Linux systems, but not part of the Windows operating system. If you are on Windows, you can get these tools by installing cygwin. During the install you will be prompted for packages to install. Open the Net package, then check the curl option and then finish the installation. Once you have done this you will be able to follow the Quick Start to Pull-Based Ingest successfully as written using the cygwin command line. Tools are available to run curl at the normal Windows command line, but not detailed in this document.

OAuth proxy

The security on the PBI API uses OAuth2. This requires some work to access the API. In this quick start it is recommended you set up a Node.js application to act as a proxy. The OAuth Quick Start guides you through the steps to create a basic proxy in Node.js that gets an access token for a Brightcove API, makes the API call, and then returns the response data.

Obtain credentials

To use the PBI API you will need proper credentials. You will use the credentials in this quick start in a form when submitting requests to APIs. To obtain proper credentials use your choice of these two options (Be sure to read the paragraph following the bullets to be sure you ask for the proper operations):

  • OAuth: Get Client Credentials Using CURL
  • OAuth: Get Client Credentials Using Postman

Obtain video id

Before you can create an ingest request you must create a place holder video information, which contains the master copy of the video, metadata, possible renditions and other information. To do this you will use the Content Management System (CMS) API. Of course, you will need correct permissions to the the API. By requesting the operations in the previous section, you do have the required permissions.
