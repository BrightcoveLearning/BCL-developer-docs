Standard Ingest Profiles

When creating a pull-based ingest request using the PBI API you must supply profile which determine the number and type of renditions created by the ingest. The standard profiles from which you can choose are:

  • Express Standard
  • audio-only
  • balanced-high-definition
  • balanced-standard-definition
  • high-bandwidth-devices
  • low-bandwidth-devices
  • mp4-only
  • screencast
  • single-rendition

Here are details of each of the profiles:

Name Min. Tier # HD Renditions # SD Renditions Comment Formats
Single Rendition VOD 0 1 A single good quality MP4 MP4
HD VOD 12 14 Focused on higher bandwidth devices and HD quality MP4, HLS
ScreenCast VOD 12 8 Limited renditions, larger frame size with lower bitrates MP4, HLS
Standard VOD 6 14 Good coverage across all devices MP4, HLS
Audio Only VOD 0 6 Audio tracks to cover all devices MP4. HLS
UGC VOD 0 2 User Generated Content MP4. HLS
           
Live - Standard Live 0 4 Small set that is cost effective MP4, HLS
Live - HD Live 2 6 Larger set without HD MP4, HLS,HD
Live - Premium HD Live 4 8 All up with HD MP4, HLS, HD
