Playback API Overview

The Playback API is used to fetch video and playlist data from Video Cloud. It is typically used to get video data to a player on a web page or in a mobile app.

General Information

Base URL

The base URL for the Playback API is:

https://edge.api.brightcove.com

Account path

In all cases, requests will be made for a specific Video Cloud Account. So, you will always add the term accounts followed by your account id to the base URL:

https://edge.api.brightcove.com/accounts/{account_id}

Authentication

Requests are authenticated by a policy key, which can be passed in one of three ways:

  1. In an argument to an Accept header: 
    Accept: application/json;pk={policy_key}
  2. In an Authorization header using the realm keyword âˆ‘ : 
    Authorization: BCOV-Policy {policy_key}
  3. In a BCOV-Policy header: 
    BCOV-Policy: {policy_key}

Methods

The API supports the following GET requests only, of the following types:

  • get video by id
  • get video by reference id</li>
  • get playlist by id</li>
  • get playlist by reference id

Video requests

A single video object can be retrieved using either the video id or reference id.

Get video by id

Endpoint

https://edge.api.brightcove.com/accounts/{account_id}/{video_id}

Example

https://edge.api.brightcove.com/accounts/57838016001/38467382999

Response/Error codes

Code Status name Description
200 SUCCESS The normal response is sent.
401 UNAUTHORIZED Must provide a BCOV-Policy header with a legal policy key.
403 FORBIDDEN The policy key provided does not permit this account or video.
404 NOT_FOUND The requested resource is not available.
405 METHOD_NOT_ALLOWED Only GET, HEAD and OPTIONS are allowed for this api.
500 INTERNAL_SERVER_ERROR Internal server error.
501 NOT_IMPLEMENTED Feature not supported, for example Nomic keys that reference policy ids.
502 BAD_GATEWAY Got a bad response from a backend server.
503 SERVICE_UNAVAILABLE Returned this response from a backend server.
504 GATEWAY_TIMEOUT Either a backend server or one of the servers they rely on timed out.
