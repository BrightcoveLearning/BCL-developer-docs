Brightcove Player System Overview

Welcome! Thanks for using the Brightcove player system. The two key components of the system are:

  • Player Management APIs
  • The Brightcove player built on Video.js

Player Management APIs

The player management API's features fall into two major categories. Those categories and detailed strengths are listed here:

  • Centralized management
    • Automatic version updates
    • Create, update and activate players via API
    • Update thousands of players at once
    • Player versioning and rollback
  • Optimized performance
    • Pre-compilation of player assets
    • Embed thumbnail or video URLs in player
    • Deliver plugins through the same CDN

Brightcove Player

Some of the strengths of the Brightcove player are listed here:

  • HTML5 first
  • HLS everywhere
  • Powerful plugin architecture
  • Native player SDKs

Overview

The Brightcove player and the associated APIs provide you with both a lightweight and highly customizable player and a system for deploying players in multiple ways. Below is a high-level workflow diagram to show how the pieces fit together. Following the diagram is an explanation of the parts of the system.

Managing players via the Player Management API

The Player Management API allows you to create player configurations, which are the specifications for players. You can not only create multiple players, but also can also use the configuration of one player in multiple ways by using child players of a parent player. You can assign different media, styles and plugins to each child player, while still maintaining the parent/child relationship to the original player, so changes made to that player are inherited by all child players.

Player configurations are created in preview mode, which gives you additional flexibility in choosing when to publish them. Once you are satisfied with changes, you publish the player via a separate API call. Publishing creates a separate branch of the player configuration while retaining the preview branch. This means that you make and test modifications to the preview player and publish it to production only when you are satisfied with the changes.

Players are usable via a URL, iframe or multiple lines of code placed in a web page. The recommended implementation is the iframe for the following reasons:

  • Sharing videos via social media is easy.
  • The player is responsive by default.
  • You don't have to worry about collisions between the JavaScript and CSS code for the player and other scripts and styles used in the page.

The Brightcove Player

The Brightcove player is a lightweight player that loads in HTML5 mode by default, with a fall back to Flash if HTML5 is not supported. Since the default player is HTML5, you can customize using familiar web tools: HTML, CSS and JavaScript. It also has an extensive API and a flexible plugin model.

The Delivery System API

You can simply use the player code returned by the Player Management API in your HTML code to publish your players, but there is also a Delivery System API that gives you some some additional flexibility.

The Delivery System API is an interface for a Git system. It is used by the Player Management API to set up repositories for your players, and in many cases, there will be no need to interact with the repos directly. However, you can use it to add additional HTML, CSS and JavaScript to the player embed and publish the whole code together.

Player plugins

To customize your player, you use plugins. Some plugins are provided by Brightcove, some from the open source video.js community or you can write them yourself. Get started with plugins from the Quick Start to Plugin Development.
