Configuring the Video Content Properties for a Player

This topic provides an overview of the video content properties that can be configured for a player.

Configuring video content settings

The video content settings control the method for associating video content to a player.

To configure the video content settings, follow these steps:

  1. Click Players in the navigation header to open the Players module.
  2. Click the link for the player you want to configure.
  3. Locate the Video Content section and click Edit.

The following methods can be used to associate video content to a player:

  • Video ID - A Video Cloud video ID will be specified
  • Ref ID - A Video Cloud reference ID will be specified
  • Media URL - The URL(s) of a video file will be provided ( the video is not in a Video Cloud account)

The video content properties that appear will change based upon the method used to associate video content to a player.

Using a Video Cloud ID

When the Video ID method is selected, the Video Cloud video with the associated video ID will appear in the player. To specify a Video Cloud video ID, follow these steps:

  1. Select Video ID.
  2. Enter the Video Cloud video ID for the video. You can get the video ID by opening the Media module and locating the video. The video ID will appear on the right side of the page next to the video thumbnail.

  3. Enter the media dimensions for one of the video renditions. The media dimensions are used to determine the aspect ratio of the video. You can get a rendition size by opening the Media module and locating the video. A list of rendition sizes will appear on the right side of the page as part of the video properties.

    Note: Brightcove recommends that the aspect ratio of the player match the aspect ratio of your video.

Using a reference ID

When the Ref ID method is selected, the Video Cloud video with the associated reference ID will appear in the player. To specify a Video Cloud reference ID, follow these steps:

  1. Select Ref ID.
  2. Enter the Video Cloud reference ID for the video. You can get the reference ID by opening the Media module and locating the video. The reference ID will appear on the right side of the page next as part of the video properties.
  3. Enter the media dimensions for one of the video renditions. The dimensions are used to determine the aspect ratio of the video. You can get a rendition size by opening the Media module and locating the video. A list of rendition sizes will appear on the right side of the page as part of the video properties.

    Note: Brightcove recommends that the aspect ratio of the player match the aspect ratio of your video.

Using a media URL

When the Media URL option is selected, a URL to a media asset outside of Video Cloud is provided. To specify a media URL, follow these steps:

  1. Select Media URL.
  2. Configure the Video URL properties. The Video URL is the URL of a video file to load into the player. Providing both HLS and MP4 assets will allow for the widest browser, platform, and device support. If multiple video assets are listed, the player will attempt to play the first one listed. If it cannot (like MP4 on iOS), then the second video is tried, and so on.

    Note: Rendition selection can occur with HLS (m3u8) files as the master file itself can contain multiple renditions. Rendition selection for non-HLS delivery types is not possible, therefore, multiple MP4 video URLs should not be added.

  3. Specify the Poster Image URL. The poster image URL is the URL to an image that will be displayed inside the player until the viewer plays the video.
  4. Enter the Media Dimenstions. The media dimensions are used by the player to determine the aspect ratio of the video. Enter the media dimensions for one of the video files that is provided in the list of video URLs.

    Note: To set the player size for an iframe player embed, edit the player settings and configure the Fixed Player Dimension properties.

Modify Publishing Code

You can also assign Video Cloud videos by modifying the player publishing code, using either the video ID or the reference ID. See Generating Player Publishing Code for details.
