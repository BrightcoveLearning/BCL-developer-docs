Notes

[1] Will include auto-advance with countdown.

[2] Includes expanded sharing options.

[3] Ability to create a custom message and/or links on the player end screen.

[4] The Brightcove Player is accessible by default.

[5] A plugin is available for setting up overlays.

[6] A plugin is available. Customers must generate and host their own thumbnails.

[7] VMAP supports pre-rolls, midrolls, companions and postrolls.

[8] Only pre-rolls are currently supported when using VAST without VMAP.

[9] With IMA plugin, desktop only.

[10] Supported on desktop and mobile. VPAID support is not yet available for ads delivered through Once.

[11] Player verification is a Flash desktop feature and is not compatible with HTML5.

[12] Open source plugin available in the community.

[13] Currently, captions must be hosted separately from Video Cloud and included via a <track> tag.

[14] The Brightcove player uses HLS on the desktop and supports encrypted HLS (HLSe).

[15] HLS playback is not supported on IE8 or IE9. These devices will fall back to MP4.

[16] Facebook does not currently support HTML5 playback in the desktop timeline. They have indicated that they will be adding support in the coming months.

[17] BEML is deprecated with the release of the Brightcove Player.

[18] All functionality is accessible via a single set of JavaScript APIs. Customization acrcoss mobile and desktop no longer requires separate Flash and JavaScript plugins.

[19] HLS playback is supported only on IE10 and above. IE8 and IE9 require MP4 fallback.
