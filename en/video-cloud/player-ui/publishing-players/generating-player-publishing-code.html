Generating Player Publishing Code

This topic covers how to generate publishing code for both preview and published players.

The Players module provides the ability to generate the player embed code and player URL for both the preview player and published player.

This topic is broken up into the following sections:

  • Preview player vs. published player
  • Types of publishing code
  • Generating code for a preview player
  • Generating code for a published player
  • Modifying publishing code
  • Using URL parameters

Preview vs. published

The preview player is used so you can preview changes before they are published. Once the changes are acceptable, the player can be published and the changes will be reflected in any published players. The preview player will display a Brightcove logo overlay indicating that it should not be used in production.

The published player has been optimized and is the player that will appear on your web pages. This player contains all of the changes that have been made to the player settings and then published by clicking the Publish button.

Types of publishing code

There are three types of publishing code that may be generated after clicking on the Embed Code & URL button.

URL - Provides a URL to the preview or published player. This is useful for previewing the player in a browser.

iframe Embed Code - Recommended for most cases. Places the player inside a <iframe> tag. If player dimensions are specified, the iframe will be sized to the player dimensions. If no player dimensions are specified, the iframe can be placed inside of a <div> tag and the responsive nature of the player will resize the player to occupy the entire space of the <div>.

In-Page Embed Code - Allows the player to exist in the the HTML page directly, not in an iframe. This offers developers the benefits of ease of accessing the player and associated properties and events. By using the in-page embed code, you lose the ease of use of the player as a single, contained unit in social media applications and sharing. This option is available only for the published player.

Code for a preview player

To generate the publishing code for a preview player:

  1. Open the Players module.
  2. Click the link for the player you want to generate the code for. This will open the player properties page.
  3. Click the Embed Code & URL button and select Preview Player.
  4. Select the appropriate publishing code.

Code for a published player

To generate the publishing code for a published player:

  1. Open the Players module.
  2. Click the link for the player you want to generate the code for. This will open the player properties page.
  3. Click the Embed Code & URL button and select Published Player.
  4. Select the appropriate publishing code.

Modifying publishing code

You can add a Video Cloud video to the player publishing code directly using the video ID or the reference ID.

Using a video ID

In-Page Embed Code

Add the attribute data-video-id="{yourVideoID}" to the <video> tag:

<video
  data-account="57838016001"
  data-player="6e44f16e-31e9-48b2-83b9-6edd71217540"
  data-embed="default"
  data-video-id="68143720001"

  class="video-js" controls></video>
<script src="//players.brightcove.net/57838016001/6e44f16e-31e9-48b2-83b9-6edd71217540_default/index.min.js"></script>

Note that if you have not assigned content to the player in the Players Module, the data-video-id attribute will not exist - in that case, add it to the code as shown above. If it does exist, simply change the value to video ID of the video you want to load.

iFrame Embed Code

To add a Video Cloud video, or load a different video than the one you assigned in the Players Module, just add ?videoId={yourVideoID} to the end of the URL in the src attribute for the iframe:

<iframe src='//players.brightcove.net/57838016001/6e44f16e-31e9-48b2-83b9-6edd71217540_default/index.html?videoId=68143720001' allowfullscreen webkitallowfullscreen mozallowfullscreen></iframe>

Using the reference ID

In-Page Embed Code

Add the attribute data-video-id="ref{yourReferenceID}" to the <video> tag:

<video
  data-account="57838016001"
  data-player="6e44f16e-31e9-48b2-83b9-6edd71217540"
  data-embed="default"
  data-video-id="ref:68143720001"

  class="video-js" controls></video>
<script src="//players.brightcove.net/57838016001/6e44f16e-31e9-48b2-83b9-6edd71217540_default/index.min.js"></script>

Note that if you have not assigned content to the player in the Players Module, the data-video-id attribute will not exist - in that case, add it to the code as shown above. If it does exist, simply change the value to ref:{referenceID} using the reference ID of the video you want to load.

iFrame Embed Code

To add a Video Cloud video, or load a different video than the one you assigned in the Players Module, just add ?videoId=ref:{your video ID} to the end of the URL in the src attribute for the iframe:

<iframe src='//players.brightcove.net/57838016001/6e44f16e-31e9-48b2-83b9-6edd71217540_default/index.html?videoId=ref:68143720001' allowfullscreen webkitallowfullscreen mozallowfullscreen></iframe>

Using URL parameters

You can also add a URL parameter to the player URL to load a Video Cloud video dynamically at runtime.

Using a video ID

Add the parameter videoId=yourVideoID to the player URL:

http://players.brightcove.net/1486906377/e001c3a9-3bc5-4a1f-bef1-69f04b277df3_default/index.html?videoId=3908955198001

Using the reference ID

Add the attribute videoId=ref:yourReferenceID to the player URL:

http://players.brightcove.net/1486906377/e001c3a9-3bc5-4a1f-bef1-69f04b277df3_default/index.html?videoId=ref:tigervideo
