Quick Start to the Policy API

In this quick start you will perform the following tasks:

  • Create a policy object
  • Retrieve a policy object

For details on the API used in this document you can view the Content Management System API Overview document and the Content Management System API reference.

API Tester

The Policy API's security is implemented using OAuth2. To permit you to get started using the Policy API as quickly as possible, the API calls in this quick start will use a shell script called api-tester.sh to get access tokens and make the API calls. The api-tester script uses cURL to make http calls. cURL is built in on Mac OS X and all Unix/Linux systems. To get cURL on Windows, see Setting up cURL. Then download the api-tester.sh script

You can see the full code for the shell script at Getting Access Tokens.

Running the shell script

After you download the shell script, unzip it wherever you like, and then go to a command line in the same folder. To run the script, type:

bash api-tester.sh

Each time you run the script, it will prompt you for 5 variables:

  • Client ID (see the next section on obtaining client credentials)
  • Client Secret (see the next section on obtaining client credentials)
  • The full API call (example: https://policy.api.brightcove.com/v1/accounts/12345678/policy_keys
  • The request verb - one of the following: GET | POST | PATCH | PUT | DELETE
  • Request data - used for POST, PATCH, and PUT; for GET and DELETE requests, you will leave this blank and just press return

For each part of this quick start, you will be given the values for each of these inputs to use with your account.

To use the Policy API you will need proper credentials. In this quick start, you will work on a Brightcove test account, so you will not need to complete this step, but be aware that you will need to get client credentials when you work on your own account.

Create a policy

One of the basic tasks you will want to create a policy. The fundamental call information is:

  • HTTP method: POST
  • Endpoint: https://policy.api.brightcove.com/v1/accounts/{YOUR_ACCOUNT_ID}/policy_keys

You will need to pass data defining the policy in the request body. The simplest policy looks like this:

{
"key-data": {
    "account-id": "{account_id}"
}
  1. You will now use the api-tester.sh script to retrieve all videos. Run bash api-tester.sh using the input values below:

    client_id:

    553d4903-4547-435d-944c-2c8e2f6abc5d

    client_secret:

    Request URL:

    {
"key-data": {
    "account-id": "57838016001"
}
  2. You should see an array of video objects in the response.

Create video

The fundamental call information for creating a video is:

  • HTTP method: POST
  • Endpoint: https://policy.api.brightcove.com/v1/accounts/YOUR_ACCOUNT_ID/videos

The video object created will eventually contain the master copy of the video, metadata, possible renditions and other information. In the call body you supply a simple JSON object specifying the name of the video to be created. To see the fields for which you can provide values during the POST, browse to the Policy API Overview document's Setting values section.

Using reference IDs

      
In the Policy API calls in the rest of this quick start, the video ID is used in the endpoint URL. You can also use the video's reference ID if you wish. If you use the reference ID you must prepend that identifier with ref:. For example, the first URL uses the video ID, whereas the second uses the reference ID.

      
https://policy.api.brightcove.com/v1/accounts/videos/123456768
        https://policy.api.brightcove.com/v1/accounts/videos/ref:xyzzy123

Search by tags

        
The Policy API has a rich searching implementation which you can leverage. Not only does the implementation provide many ways to search, but you can also manipulate the display of results by sorting, limiting result set size, etc. A few examples will be shown in this quick start. For complete documentation see the Policy API Overview document's Search section.

        
You will now search your videos for a certain keyword. These tags can be set via API, or in Video Cloud Studio where they are referred to as tags. The fundamental call information for searching video data is:

        
    
          
  • HTTP method: GET
    • 
          
  • Endpoint: https://policy.api.brightcove.com/v1/accounts/YOUR_ACCOUNT_ID/videosq=field:searchTerms
    • 
        

        
Note: Before completing this task you must perform the previous steps above to retrieve and assign account and credential information, and also provide a video id.

        
    
          
  1. You will now search for videos that have the tags quick or start (the ones you added to your new video earlier). To do this, you will use an API call that appends a question mark to the endpoint to create the query string, then a q:, and finally the syntax to look for specific tags, in this case tags:quick,start.
