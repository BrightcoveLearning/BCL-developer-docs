Getting Started with the Media Module

This topic provides an overview of the Media module which is used to manage videos in a Video Cloud account.

Note: The New Video Cloud Studio is currently in beta and is not supported by Brightcove Support. If you would like access to the beta, please contact your Brightcove Account Manager.

This topic contains the following sections:

  • Launching the Media module
  • Using quick view
  • Publishing a video

Launching the Media module

To access the Media module, login to Video Cloud Studio and click Media in the navigation header.

The Media module can also be accessed at: https://studio.brightcove.com/products/videocloud-pre-release/media

The body of the page will display the following:

  • Video thumbnail
  • Video name and ID
  • Last updated date and time
  • Video status
    • - indicates the video is active
    • - indicates the video is inactive and can't be played

Note: Videos uploaded using the Upload module will not display a thumbnail image by default. This limitation will be addressed in a future update. To create a thumbnail, you can use the image capture tool or upload your own image using the current Video Cloud Studio.

Using quick view

The Media module offers a quick view feature that can be used to quickly view metadata associated with a video. Open quick view by clicking on the row containing a video.

The video list will expand to display the following:

  • Preview player
  • Video name and duration
  • Video ID
  • Reference ID
  • Short description
  • Video tags
  • Last updated date and time
  • Video status

Known Issue: Since the preview player is being loaded over HTTPS within the Studio, if your content is being served over HTTP, the preview player will not playback the content and your browser will display an insecure scripts message. We are working on a fix to address this issue and will update beta testers on the progress.

Publishing a video

To get the publishing code for a video using the Media module:

  1. Select the video to publish and click Publish. The publishing dialog will open.

    Known Issue: Since the preview player is being loaded over HTTPS within the Studio, if your content is being served over HTTP, the preview player will not playback the content and your browser will display an insecure scripts message. We are working on a fix to address this issue and will update beta testers on the progress.

  2. Enter the Player ID to use. To get the player ID, open the Players module, select a player and the Player ID will appear on the right side of the page.

    Note that the preview will update to display the player that was selected. If the player ID is invalid, an error will be displayed.

  3. Select an aspect ratio for the player, 16 x 9 or 4 x 3. Select Custom to manually enter and player width and height.
  4. Click Get Embed Code.

    There are three type of publishing code to choose from:

    • iframe - Recommended for most cases. Places the player inside an <iframe> tag. If player dimensions are specified, the iframe will be sized to the player dimensions. If no player dimensions are specified, the iframe can be placed inside of a <div> tag and the responsive nature of the player will resize the player to occupy the entire space of the <div>.
    • In-Page - Allows the player to exist in the the HTML page directly, not in an iframe. This offers developers the benefits of ease of accessing the player and associated properties and events. By using the in-page embed code, you lose the ease of use of the player as a single, contained unit in social media applications and sharing. This option is available only for the published player.
    • URL - Provides a URL to the preview or published player. This is useful for previewing the player in a browser.
  5. When finished, click Done.
