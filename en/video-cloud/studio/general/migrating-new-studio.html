Migrating to the New Video Cloud Studio FAQ

Media Module

How do I upload videos?
Videos are now upload using the Upload module. The Media module is not used to upload videos.
I have a custom transcode profile in the current Video Cloud Studio. Is that profile available in the New Video Cloud Studio?

No. The profiles in your current/legacy Video Cloud Studio account are not carried over to the New Studio. There are a list of Brightcove recommended default profiles that you can choose from when you upload using the Upload Module. Custom profiles must be created using the new Profiles API. Once the custom profile is created for an account using the API, it will be available for selection in the Upload module (and will become the default selection in the Upload module).

Currently, changes to the default profiles used by the Upload module must be done by a Brightcove Sales Engineer. Contact your Brightcove Account Manager for more information.

Do I need to create any special types of renditions?

The new Brightcove Player defaults to HLS renditions on Desktop. Most Video Cloud customers have at least one (typically multiple) HLS renditions on their videos, but if you don’t have any HLS renditions, you will need to re-transcode or re-upload your videos and use a profile that transcodes to multiple HLS renditions.

Currently, changes to the default profiles used by the Upload module must be done by a Brightcove Sales Engineer. Contact your Brightcove Account Manager for more information.

Can I access all of my existing videos in the New Video Cloud Studio?
Yes. Your entire Video Cloud catalog is available/accessible in the new Media module. Thumbnails, video still images as well as all video properties and custom metadata fields will show up in the new studio.

Are captions supported?

Yes. The new Brightcove Player and new Media module support captions in WebVTT format. Old DFXP captions files will not work for videos published to new Brightcove Players and won’t be displayed in the new Media Module.

Note that currently the new Media module does not support the ability to host caption files. That functionality will be coming in a future release. For more information on setting captions in the Media module, see Configuring Text Tracks to Provide Captions.

Players Module

Can I manage and publish Smart Players in the New Studio?
No. We do not have any plans at this time to support Smart Player management or publishing in the New Video Cloud Studio. Smart Player management and publishing will remain in the legacy studio and the New Studio (Media and Players modules) will only display/support the New Brightcove Player. We encourage our customer base to migrate over to the New Brightcove Player and do not want to encourage the continued use of Smart Players in the New Studio.
Will my player customizations work?

Smart Player customization will not work with the new Brightcove Player. The new Brightcove Player has a new extension and customization model based on the HTML5 video standard and relies entirely on Javascript and CSS. Plug-ins can be written to change UI behavior, including player controls, overlays, menus, and integration with third-party tools. See Configuring Player Plugins and Styling Players for more details.

What features are available today and how do they differ from what’s in the Smart Player?
The new Brightcove Player has many new and different features compared to the Smart Player. Some popular Smart Player features are not yet available in the new Brightcove Player. Consult this table to see if your requirements match what’s available now and to get a sense for what’s coming soon. See the Player Feature Comparison for details.
How do I try it?

You can get started by creating a new player in the new Players Module. Once you’ve configured the player the way you want, you can associate a Video Cloud video with it by specifying a video ID or reference ID from the Media Module from your existing Video Cloud account.

This step-by-step guide will show you how you can quickly get started using an existing video with the new Brightcove Player.

How does the new Brightcove Player work with my custom CMS workflow?

With the new Brightcove Player you can now create new players and update existing player configurations without interactively logging-in to Video Cloud Studio. Using the new Player Management API you can add player creation and update steps to your custom workflow by calling out to a simple REST-based API. If you don’t need to update or create a new player configuration, you can easily insert video IDs or reference IDs of videos stored in Brightcove Video Cloud into a copied page template that includes the embed code generated by Video Cloud.

Refer to the Player Management API documentation to learn how to update your players programmatically as part of your custom publishing workflow.
