Brightcove Player Migration Guide

This guide offers information and guidance on migrating from the Flash-based Smart Player to the new Brightcove Player. There are two main sections for:

  • Video Cloud Studio Users
  • Integration Developers

Note: The new Brightcove Player is available today as an option for Video Cloud customers, but you can keep using the Flash-based Smart Player - it’s not going away. You can try the new Brightcove Player without disrupting any of your existing player sites or pages, and you can choose to update them one at a time or all at once - however you like.

Video Cloud Studio Users

If you use Video Cloud Studio to create your players and copy and paste the publishing code into you web pages, migrating to the new Brightcove Player will be straightforward. However, there are a few things to consider before you move forward.

Considerations

Player features

Although the new Brightcove Player is stable and fully ready for production, we are still adding features to it, and some features available in the old Smart Player are not yet available in the new player. See the Player Feature Comparison for a detailed comparison of current features.

Player customizations

If you have created customized players using BEML, styling, or plugins, you will need to re-create these customizations for the new Brightcove Player. Smart Player customization will not work with the new Brightcove Player. The new Brightcove Player has a new extension and customization model based on Javascript and CSS. Plug-ins can be written to change UI behavior, including player controls, overlays, menus, and integration with third-party tools.

Integration Developers

If you integrate Video Cloud with your CMS, the new Brightcove Player is going to make your job easier. With the new Brightcove Player you can now create new players and update existing player configurations without interactively logging-in to Video Cloud Studio. Using the new Player Management Service API you can add player creation and update steps to your custom workflow by calling out to a simple REST-based API. If you don’t need to update or create a new player configuration, you can easily insert video IDs or reference IDs of videos stored in Brightcove Video Cloud into a copied page template that includes the embed code generated by Video Cloud or the Player Management Service.

Now...or wait a little?

Before you begin, however, be aware that there are even better things to come. As we launch the new player, we are also launching a beta version of a new CMS API. This API combines operations from the Player Management API with others that allow you to ingest and interact with videos, providing one API for all your CMS integration needs.
