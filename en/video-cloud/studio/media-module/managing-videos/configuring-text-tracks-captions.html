Configuring Text Tracks to Provide Captions

In this topic you will learn how to configure the text track properties using the Media module.

The text track properties can be used to provide captions for a video. Captions provide additional or interpretive information to individuals who wish to access it. Captions are necessary so the deaf and hard of hearing can access the audio portion of video presentations. To display captions for a video in the Brightcove player, you need to complete the following tasks:

  • Create caption files
  • Configure the text track properties
  • Style captions

Creating caption files

To add captions to a video, you first need to create a captions file. The Brightcove player requires that captions be in Web Video Text Tracks Format (WebVTT) format. The WebVTT file format is in plain text and contains time text data followed by the desired caption or subtitle text. A portion of a WebVTT file is shown below.

WEBVTT

1
00:00:00.000 --> 00:00:03.530 align:middle line:84%

2
00:00:03.530 --> 00:00:06.510 align:middle line:90%
In this video we'll learn how
to log into and navigate around

3
00:00:06.510 --> 00:00:09.550 align:middle line:84%
inside of Video Cloud Studio.

4
00:00:09.550 --> 00:00:12.020 align:middle line:90%
Video Cloud Studio is
a web-based application

For more information on the WebVTT format, see this link. You should have a separate WebVTT file for each caption language you need to support. While there are many third-party tools to help create captions files, Brightcove has several partners that provide transcription, captioning and translation services. For a complete list of Brightcove partners, visit the Partners section of our website.

Configuring the text track properties

Before configuring the text track properties, the caption files should be placed at an internet accessible URL. Brightcove does not currently provide a location to store caption files. This is planned for a future release.

To view and edit the text track properties for a video, open the Media module, click the video title link and locate the Text Tracks section. To configure the text track properties, follow these steps:

  1. Click Edit.
  2. Add the URL to the caption file and the Language. The language is the two-letter language code that specifies the language of the track text data.
  3. Click Show Advanced Settings to view additional settings that can be configured for the text track:
    • Kind - Kind of text track. Options include:
      • Captions - Similar to subtitles, but may also include sound effects and other audio information
      • Subtitles - The transcription or translation of the dialogue
      • Descriptions - Intended to be a separate text file that describes the video through a screenreader
      • Chapters - Intended to help the user navigate through the video
      • Metadata - Information and content about the video which isnâ€™t intended to be displayed to the viewer by default (though you may wish to do so using JavaScript)
    • Label - Label to display inside player. If no label is supplied, it defaults to using the language code.
    • MIME Type
    • Default - Sets this language as the default language
  4. Click Add Text Track to add additional tracks.
  5. Click Save.

Note: When using the iFrame embed code to publish a player, you may need to add the <crossorigin> attribute to the <track> tag and set it to anonymous for captions to appear on iOS devices.

When a player with associated captions is published, a CC button will appear on the player chrome. Click the CC button to select a caption language and display the captions.

Styling captions

The default look and feel of the captions is subject to change so you may wish to style the captions. There are a few ways that captions can be styled:

  • By adding styling options to the WebVTT file
  • By creating a custom class in a CSS file
  • User styling options

Adding styling options to the WebVTT file

Style options can be applied directly to the WebVTT file that allow you to change the location and orientation of the captions as well as the text. For examples on how to do this, see the Styling Options section of the Video Subtitling and WebVTT document.

Creating a custom class in a CSS file

A custom class can be created as part of a CSS file and then applied to elements in the WebVTT file. For example, the class below changes the text color to white, background color to blue and bolds the font.

<style>
   .captionstyle {color:white; background-color:blue; font-weight:bold}
</style>

Then, the WebVTT file can be modified to apply the new captionstyle.

WEBVTT

1
00:00:00.000 --> 00:00:03.530 align:middle line:84%

2
00:00:03.530 --> 00:00:06.510 align:middle line:90%
<c.captionstyle>In this video we'll learn how
to login to and navigate around</c>

3
00:00:06.510 --> 00:00:09.550 align:middle line:84%
<c.captionstyle>inside of Video Cloud Studio.</c>

For more information on styling options for caption files, see the W3C WebVTT specification document or see Video Subtitling and WebVTT.

User styling of captions

Viewers on desktop and Android devices can change the appearance of the captions by using the captions settings menu.

Clicking on the captions settings menu will display an overlay which allows the following caption properties to be set:

  • Foreground color and opacity
  • Background color and opacity
  • Window color and opacity
  • Font size
  • Text edge style
  • Font family

Click Defaults to return the captions to their default styling.

For users on iOS devices, caption styling is controlled through the iOS Settings. To access the settings select Settings > General > Accessibility > Subtitles & Captioning > Style. Here you can select a predefined style or create new styles.

Limitations

Captions are supported on the following device types and operating systems.

  • iOS 7 & 8
  • Android 4.x
  • Internet Explorer 8+
  • The latest Chrome, Safari, and Firefox browsers on desktops
