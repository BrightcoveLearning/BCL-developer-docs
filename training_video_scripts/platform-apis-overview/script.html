[show insomnia request/response]

All the platform are REST APIs, meaning that they are accessed through HTTP requests, using different methods for different kinds of requests, sometimes including a request body in JSON format. Responses also come back in JSON format. Depending on the kind of request, the response may contain an elaborate data object, or, if the operation requested is asynchronous - a request to ingest a video, for example, the response may be just a job id you can use to track the progress of the operation.

And some requests, particularly requests to delete an asset of some kind, will generally return an HTTP 204 response with no content.

Authentication for API requests is generally managed by our OAuth API, which can be used to create client credentials that are then used to get access tokens for requests. This is explained in more detail in the videos and documentation for the OAuth API. A few of the APIs use an API key instead of access tokens for authentication.

There are several tools that you can use to test API requests, and we have provided instructions for setting up and using these. The simplest is a command line tool called CURL. To simplify the use of CURL, Brightcove Learning Services has created a shell script that prompts you for the necessary inputs, fetches an access token, makes the request, and displays a pretty-printed response. There are also several desktop apps you can use. The most popular are the cross-platform tools Insomnia and Postman, which make it easy to set up the authentication and save your requests in collections.

[show api samples]

We have also created an extensive collection of sample apps built around the platform API, all based on use cases suggested to us by our customers. The samples are built in CodePens to make it easy for you to duplicated them and experiment with the code. We also store the code for the samples in GitHub repositories, so that you can download it and modify it to meet your needs.
